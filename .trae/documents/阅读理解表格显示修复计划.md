# 阅读理解表格显示修复计划

## 问题描述

用户反馈：阅读理解题的阅读材料在答题页面不支持显示表格，表格内容以纯文本形式展示，影响阅读体验。

从截图可见，阅读材料中的表格数据（如游玩项目、注意事项等）没有正确渲染为表格格式。

---

## 问题分析

### 1. 需要定位的问题点

1. **阅读材料存储格式**
   - 检查数据库中阅读材料的存储格式（HTML/Markdown/纯文本）
   - 确认表格数据是如何存储的

2. **前端渲染组件**
   - 找到答题页面中显示阅读材料的组件
   - 检查是否使用了富文本渲染
   - 确认是否支持表格 HTML 标签

3. **后端接口返回**
   - 检查题目接口返回的数据格式
   - 确认阅读材料字段是否包含 HTML 标签

### 2. 可能的原因

1. 阅读材料使用纯文本存储，不支持 HTML 标签
2. 前端渲染时使用了 `{{ }}` 文本插值而非 `v-html`
3. 富文本编辑器（Quill）的表格功能未启用
4. CSS 样式中表格样式被覆盖或缺失
5. XSS 过滤规则过滤了表格标签

## 技术要点

### 1. Quill 编辑器表格支持

Quill 默认不支持表格，需要：
- 使用 `quill-better-table` 模块
- 或使用 HTML 直接插入表格
- 或在富文本模式下允许表格标签

### 2. Vue 中渲染 HTML

```vue
<!-- 错误：会显示 HTML 标签 -->
<div>{{ readingPassage }}</div>

<!-- 正确：渲染 HTML 内容 -->
<div v-html="readingPassage"></div>
```

### 3. 表格样式示例

```css
.reading-passage table {
  width: 100%;
  border-collapse: collapse;
  margin: 16px 0;
}

.reading-passage th,
.reading-passage td {
  border: 1px solid #d0d7de;
  padding: 8px 12px;
  text-align: left;
}

.reading-passage th {
  background-color: #f6f8fa;
  font-weight: 600;
}

.reading-passage tr:nth-child(even) {
  background-color: #f6f8fa;
}
```

### 4. XSS 过滤白名单

需要在 `xss-filter.js` 中添加：
```javascript
const allowedTags = [
  // ... 现有标签
  'table', 'thead', 'tbody', 'tr', 'th', 'td', 'caption', 'col', 'colgroup'
];

const allowedAttributes = {
  // ... 现有属性
  'colspan': true,
  'rowspan': true,
  'style': true // 谨慎使用
};
```
