# TRAE 个人规则（User Rules）
适用：所有项目 | 全局生效
版本：v1.0 | 最后更新：2026-03-29
优先级：低于项目规则

## 一、交互与输出规范
1. 所有回答必须使用中文，技术问题附带原理说明 + 可直接运行的代码示例
2. 代码输出必须格式化，语法高亮，关键逻辑加中文注释
3. 代码行数 > 20 行时，拆分函数/组件，保持颗粒度合理
4. 遇到模糊需求时，先确认细节，不自行假设
5. 提供 Windows 终端命令，适配本地开发环境

## 二、开发流程规范
1. 开发顺序：需求确认 → 接口设计 → 数据库设计 → 前端页面 → 测试
2. 提交前必须执行：npm run lint + npm run format
3. 所有代码必须可直接运行，无语法错误、无崩溃风险
4. 功能开发完成后，同步更新项目文档

## 三、代码质量规范
1. 前端：props 必须定义类型 + 默认值；空值安全使用 ?.、??
2. 后端：所有异步方法加 try/catch；接口返回统一格式 {code, msg, data}
3. 禁止硬编码，所有配置使用环境变量或配置文件，或储存在数据库中
4. 大数据场景：无内存泄漏、无卡顿、接口响应时间 < 500ms

## 四、性能与交付规范
1. 数据库：高频查询字段加索引；分页查询必加 LIMIT
2. 前端：组件按需加载，路由懒加载；大列表使用虚拟滚动
3. 后端：连接池复用；批量操作分批执行
4. 交付：代码可直接部署生产，无 BUG、无冗余、适配大数据
