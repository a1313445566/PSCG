## 🚀 部署执行步骤

### 方式一：MySQL 命令行（推荐）

```bash
# 1. 执行导航菜单迁移
mysql -u root -p pscg < migrations/add-navigation-menus.sql

# 2. 执行产品卡片迁移（新增）
mysql -u root -p pscg < migrations/add-product-cards.sql
```

### 方式二：MySQL 客户端工具

1. 打开 Navicat / DBeaver / MySQL Workbench
2. 连接到 `pscg` 数据库
3. 打开 SQL 编辑器
4. 复制 `migrations/add-product-cards.sql` 内容粘贴执行
5. 检查输出确认无错误

### 方式三：Node.js 脚本执行（自动化部署）

```javascript
// scripts/run-migration.js
const mysql = require('mysql2/promise')
const fs = require('fs')
const path = require('path')

async function runMigration() {
  const connection = await mysql.createConnection({
    host: 'localhost',
    user: process.env.DB_USER || 'root',
    password: process.env.DB_PASSWORD,
    database: process.env.DB_NAME || 'pscg'
  })

  const sqlPath = path.join(__dirname, '../migrations/add-product-cards.sql')
  const sql = fs.readFileSync(sqlPath, 'utf-8')
  
  await connection.query(sql)
  console.log('✅ 产品卡片迁移脚本执行成功')
  
  await connection.end()
}

runMigration().catch(console.error)
```

## ❓ 常见问题

### Q1: 执行报错 "Table already exists"？
**A**: 脚本使用了 `CREATE TABLE IF NOT EXISTS` 和 `INSERT IGNORE`，可以安全重复执行。

### Q2: 如何在已有数据的基础上重新初始化默认数据？
```sql
-- 先清空现有数据
TRUNCATE TABLE product_cards;

-- 再执行迁移脚本的 INSERT 部分
INSERT INTO product_cards (...) VALUES (...);
```

### Q3: 生产环境执行需要注意什么？
1. **备份数据库**：`mysqldump -u root -p pscg > backup_$(date +%Y%m%d).sql`
2. **低峰期执行**：避免业务高峰期
3. **验证数据**：执行后立即检查数据和功能
4. **保留日志**：保存迁移输出以便排查问题
