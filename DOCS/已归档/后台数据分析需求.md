## 🎯 需求概述

开发后台数据分析可视化界面，帮助管理员全面了解学生答题情况、学科表现、错题分布等数据，为教学决策提供数据支持。

### 核心功能
1. **数据概览面板** - 展示关键指标卡片（总用户数、答题次数、总题数、整体正确率）
2. **多维度分析图表** - 年级分析、学科分析、时间趋势、班级分析等可视化图表
3. **错题分析模块** - 错题率统计、易错题列表展示
4. **高级筛选功能** - 支持按学生、年级、班级、学科、时间范围筛选
5. **数据导出功能** - 导出分析报告为 Excel 文件

### 用户价值
- 管理员可以快速了解整体答题情况
- 教师可以发现教学薄弱环节
- 支持数据驱动的教学改进决策

### 8. ECharts 使用规范

#### 图表配置要求
```javascript
import * as echarts from 'echarts'
import { onMounted, onUnmounted, ref } from 'vue'

// 图表实例引用
const chartRef = ref(null)
let chartInstance = null

// 初始化图表
const initChart = () => {
  if (!chartRef.value) return
  
  chartInstance = echarts.init(chartRef.value)
  
  const option = {
    // 图表配置
    title: {
      text: '学科正确率分析'
    },
    tooltip: {
      trigger: 'axis'
    },
    xAxis: {
      type: 'category',
      data: [] // 从 API 获取
    },
    yAxis: {
      type: 'value',
      axisLabel: {
        formatter: '{value}%'
      }
    },
    series: [{
      name: '正确率',
      type: 'bar',
      data: [] // 从 API 获取
    }]
  }
  
  chartInstance.setOption(option)
}

// 更新图表数据
const updateChart = (data) => {
  if (!chartInstance) return
  
  chartInstance.setOption({
    xAxis: {
      data: data.map(item => item.subject)
    },
    series: [{
      data: data.map(item => item.accuracy)
    }]
  })
}

// 窗口大小变化时重新调整图表
const handleResize = () => {
  chartInstance?.resize()
}

// 生命周期
onMounted(() => {
  initChart()
  window.addEventListener('resize', handleResize)
})

onUnmounted(() => {
  chartInstance?.dispose()
  window.removeEventListener('resize', handleResize)
})
```

**⚠️ 注意**:
- 必须在 `onUnmounted` 时销毁图表实例，避免内存泄漏
- 必须监听窗口 `resize` 事件，确保图表自适应
- 使用 ECharts 5.x 版本

const DataAnalysis = defineAsyncComponent(() =>

<AdminLayout>
  <DashboardView v-if="activeMenu === 'dashboard'" />

### 📋 本项目的复用规范

- ❌ **禁止自己实现分页逻辑** - 不要在每个组件中重复实现分页

- ✅ **必须使用 useLoading** - 所有异步操作统一使用 useLoading Hook
- ❌ **禁止自己管理 loading 状态** - 不要手动创建 loading ref

- ✅ **必须使用 api 封装** - 所有 API 调用统一使用 api 工具
- ❌ **禁止使用原生 fetch/axios** - 确保统一的错误处理和认证

#### 消息提示
- ✅ **必须使用 message 封装** - 所有用户提示统一使用 message 工具
- ❌ **禁止直接使用 Element Plus** - 确保统一的提示样式和行为
