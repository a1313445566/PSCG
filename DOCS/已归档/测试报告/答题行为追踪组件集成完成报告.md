#### 修改点 3：添加组件引用和行为追踪逻辑

```javascript
// 答题行为追踪组件引用
const behaviorTracker = ref(null)

// 当前正在答题的题目ID（用于追踪）
const currentTrackingQuestionId = ref(null)

// 选择选项
const selectOption = (questionId, option, questionType = 'single') => {
  // 如果是新题目，开始追踪
  if (currentTrackingQuestionId.value !== questionId) {
    behaviorTracker.value?.startTracking(questionId)
    currentTrackingQuestionId.value = questionId
  }
  
  // 追踪首次答案
  const currentAnswer = userAnswers.value[questionId]
  if (!currentAnswer || (questionType === 'multiple' && currentAnswer.length === 0)) {
    behaviorTracker.value?.trackFirstAnswer(option)
  } else {
    // 如果已有答案，说明是修改
    behaviorTracker.value?.trackModification()
  }
  
  quizStore.submitAnswer(questionId, option, questionType)
}

// 处理题目悬停（犹豫追踪）
const handleQuestionHover = (questionId, isEnter) => {
  if (isEnter) {
    behaviorTracker.value?.trackHoverStart()
  } else {
    behaviorTracker.value?.trackHoverEnd()
  }
}
```

#### 修改点 4：提交答案时提交行为数据

  // ... 原有验证逻辑 ...
  
  try {

    const userId = localStorage.getItem('userId')
    if (userId && behaviorTracker.value) {
      // 为每个题目提交行为数据
      for (const question of currentQuestions.value) {
        const userAnswer = userAnswers.value[question.id]
        if (userAnswer) {
          const isCorrect = isAnswerCorrect(question, userAnswer)

            parseInt(userId),
            question.id,
            finalAnswer,
            isCorrect
          )
        }
      }
      
      // 刷新缓冲区，确保所有数据已提交

    }

## 🎯 追踪的数据

### 自动追踪的答题行为

| 数据项 | 说明 | 追踪方式 |
|--------|------|----------|
| **答题时间** | 从开始答题到提交的时间 | `startTracking()` 开始计时，`submitBehavior()` 计算总时长 |
| **修改次数** | 答案被修改的次数 | `trackModification()` 每次修改时调用 |
| **首次答案** | 第一次选择的答案 | `trackFirstAnswer()` 记录首次选择 |
| **犹豫时间** | 鼠标悬停总时长（>1秒） | `trackHoverStart/End()` 记录悬停时长 |
| **是否跳题** | 是否跳过题目又返回 | `trackSkip/Return()` 记录跳题行为 |
| **最终答案** | 提交时的答案 | `submitBehavior()` 提交时记录 |
| **是否正确** | 答案是否正确 | `submitBehavior()` 计算正确性 |
| **会话ID** | 答题会话唯一标识 | 组件挂载时自动生成 |

## 📈 数据流程

```
用户答题
    ↓
开始追踪答题时间
    ↓
用户悬停题目 → 记录犹豫时间
    ↓
用户选择答案 → 记录首次答案/修改次数
    ↓
用户修改答案 → 增加修改次数
    ↓
用户提交答案 → 提交行为数据到缓冲区
    ↓
缓冲区满（10条）或定时器触发（30秒） → 批量提交到后端
    ↓
后端保存到 answer_behavior 表
    ↓
数据积累 ≥10 条 → 可触发学习风格分析
    ↓
AI 自动分析学习风格 → 保存到 user_learning_style 表
```

SELECT 
  ab.id,
  u.name as student_name,
  ab.answer_time,
  ab.answer_modifications,
  ab.hesitation_time,
  ab.is_correct,
  ab.created_at
FROM answer_behavior ab
INNER JOIN users u ON ab.user_id = u.id
ORDER BY ab.created_at DESC
LIMIT 20;

  COUNT(*) as total_behaviors,
