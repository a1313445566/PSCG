# 📊 PSCG 系统性能监控功能集成指南

## 📋 概述

本文档详细描述了PSCG系统中已实现的性能监控功能，包括系统健康监控、API性能分析、数据库性能监控和缓存管理。这些功能已经通过中间件和API实现，只需要集成前端界面即可使用。

## 🏗️ 系统架构

### 核心组件
```
1. 中间件层
   ├── responseTime.js - API响应时间监控
   └── dbPerformance.js - 数据库性能监控

2. API层
   ├── /api/performance/db - 数据库性能数据
   ├── /api/performance/health - 系统健康状态
   └── /api/cache/stats - 缓存统计信息

3. 数据存储层
   ├── 实时监控数据（内存存储）
   └── 慢查询记录（内存缓存）
```

## 🔧 已实现的功能

### 1. API响应时间监控 (responseTime.js)

#### 功能特性
- ✅ **实时响应时间记录**：监控所有API请求的响应时间
- ✅ **端点统计分析**：按API端点统计平均响应时间、最大响应时间
- ✅ **慢请求告警**：自动识别超过500ms的慢请求
- ✅ **错误率监控**：统计每个端点的错误率（HTTP 4xx/5xx）
- ✅ **请求计数**：统计总请求数和每个端点的请求数

#### 数据收集
```javascript
// 监控数据结构
{
  endpoints: {
    "GET /api/questions": {
      requestCount: 123,
      totalResponseTime: 4567,
      avgResponseTime: 37.1,
      maxResponseTime: 250,
      errorCount: 5
    }
  },
  totalRequests: 1500,
  totalResponseTime: 45000,
  maxResponseTime: 800,
  slowRequests: [...],
  timestamp: "2025-03-23T18:30:00Z"
}
```

### 2. 数据库性能监控 (dbPerformance.js)

#### 功能特性
- ✅ **查询时间统计**：记录每次数据库查询的执行时间
- ✅ **慢查询检测**：自动识别超过200ms的慢查询
- ✅ **查询计数**：统计总查询数和慢查询数
- ✅ **性能报告**：生成数据库性能分析报告

#### 监控指标
```javascript
{
  averageQueryTime: 45.2,    // 平均查询时间(ms)
  queryCount: 234,           // 总查询次数
  slowQueryCount: 12,        // 慢查询次数
  slowQueries: [            // 慢查询详情列表
    {
      query: "SELECT * FROM questions WHERE subject_id = ?",
      time: 234,
      timestamp: "2025-03-23T18:25:00Z"
    }
  ]
}
```

### 3. 系统健康监控

#### 功能特性
- ✅ **内存使用监控**：实时监控Node.js进程内存使用情况
- ✅ **运行时间统计**：记录系统启动时间和运行时长
- ✅ **健康状态检测**：定期检查系统健康状态
- ✅ **进程信息收集**：收集进程PID、平台信息等

#### 健康检查API
```bash
# 获取系统健康状态
GET /api/performance/health

# 响应示例
{
  "status": "ok",
  "uptime": 86400,           // 运行时间(秒)
  "timestamp": "2025-03-23T18:30:00Z",
  "memoryUsage": {
    "rss": 150241280,        // Resident Set Size
    "heapTotal": 102760448,  // 堆内存总量
    "heapUsed": 75623424,    // 已使用堆内存
    "external": 16723256     // 外部内存
  }
}
```

### 4. 缓存管理

#### 功能特性
- ✅ **缓存命中率统计**：实时计算缓存命中率
- ✅ **缓存项管理**：查看和管理缓存中的项目
- ✅ **手动清理功能**：支持手动清空缓存
- ✅ **使用情况分析**：分析缓存空间使用情况

#### 缓存API
```bash
# 获取缓存统计信息
GET /api/cache/stats

# 手动清空缓存
POST /api/cache/clear
```

## 🔍 数据指标说明

### 1. 性能评分标准

#### API响应时间评分
| 响应时间 | 评分 | 状态 |
|---------|------|------|
| < 100ms | 🟢 优秀 | 极快响应 |
| 100-300ms | 🟡 良好 | 正常响应 |
| 300-500ms | 🟠 一般 | 可优化 |
| > 500ms | 🔴 较差 | 需要优化 |

#### 数据库查询时间评分
| 查询时间 | 评分 | 建议 |
|---------|------|------|
| < 50ms | 🟢 优秀 | 索引良好 |
| 50-100ms | 🟡 良好 | 正常性能 |
| 100-200ms | 🟠 一般 | 可优化查询 |
| > 200ms | 🔴 较差 | 需要优化索引或查询 |

#### 缓存命中率评分
| 命中率 | 评分 | 说明 |
|--------|------|------|
| > 90% | 🟢 优秀 | 缓存策略有效 |
| 70-90% | 🟡 良好 | 正常水平 |
| 50-70% | 🟠 一般 | 可优化缓存策略 |
| < 50% | 🔴 较差 | 需要重新设计缓存 |

### 2. 告警阈值设置

#### 建议的告警阈值
```javascript
const alertThresholds = {
  // API性能告警
  api: {
    avgResponseTime: 300,    // 平均响应时间超过300ms
    maxResponseTime: 800,    // 单次响应超过800ms
    errorRate: 5,            // 错误率超过5%
  },
  
  // 数据库性能告警
  database: {
    avgQueryTime: 100,       // 平均查询时间超过100ms
    slowQueryRate: 5,        // 慢查询率超过5%
    totalSlowQueries: 10,    // 累计慢查询超过10次
  },
  
  // 系统资源告警
  system: {
    memoryUsage: 80,         // 内存使用率超过80%
    uptime: 7 * 24 * 3600,   // 连续运行超过7天
  },
  
  // 缓存性能告警
  cache: {
    hitRate: 60,             // 命中率低于60%
    missRate: 40,            // 未命中率超过40%
  }
};
```

## 🎨 前端集成方案

### 1. 组件设计方案

#### 推荐组件结构
```vue
<template>
  <!-- 1. 系统健康概览 -->
  <HealthOverview :data="healthData" />
  
  <!-- 2. API性能分析 -->
  <ApiPerformance :metrics="apiMetrics" />
  
  <!-- 3. 数据库监控 -->
  <DatabaseMonitoring :stats="dbStats" />
  
  <!-- 4. 缓存管理 -->
  <CacheManagement :cache="cacheStats" />
</template>
```

### 2. 数据获取方案

#### 推荐的数据获取逻辑
```javascript
// 统一数据获取函数
const fetchPerformanceData = async () => {
  try {
    const [healthData, apiMetrics, dbStats, cacheStats] = await Promise.all([
      fetch('/api/performance/health'),
      fetch('/api/performance/api'),
      fetch('/api/performance/db'),
      fetch('/api/cache/stats')
    ]);
    
    return {
      health: await healthData.json(),
      api: await apiMetrics.json(),
      db: await dbStats.json(),
      cache: await cacheStats.json()
    };
  } catch (error) {
    console.error('获取性能数据失败:', error);
    return null;
  }
};
```

### 3. 实时刷新方案

#### 推荐的实时监控实现
```javascript
// 定时刷新数据
const setupRealTimeMonitoring = () => {
  // 默认30秒刷新一次
  const refreshInterval = setInterval(async () => {
    const data = await fetchPerformanceData();
    if (data) {
      updateDashboard(data);
    }
  }, 30000);
  
  return () => clearInterval(refreshInterval);
};
```

## 🐛 故障排查指南

### 1. 常见问题及解决方案

#### 问题1：API监控数据不更新
**可能原因：** 中间件未正确加载
**解决方案：** 
```bash
# 检查服务器日志
pm2 logs 0 --lines 20

# 重启服务器
pm2 restart 0
```

#### 问题2：数据库性能报告为空
**可能原因：** 数据库连接失败
**解决方案：**
```javascript
// 检查数据库连接配置
const connection = mysql.createConnection({
  host: 'localhost',
  user: 'root',
  password: 'password',
  database: 'quiz_db'
});
```

#### 问题3：缓存命中率异常低
**可能原因：** 缓存策略不合理
**解决方案：**
```javascript
// 调整缓存策略
const cacheConfig = {
  maxAge: 3600000,      // 缓存1小时
  maxSize: 1000,        // 最多1000个缓存项
  checkPeriod: 60000    // 每60秒检查一次
};
```

### 2. 监控指标异常处理

#### 监控项异常处理流程
```
1. 检测到异常指标
2. 记录异常日志
3. 发送告警通知（可选）
4. 触发自动恢复机制（可选）
5. 生成异常报告
```
