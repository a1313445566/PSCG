## 背景

随着题库数量增长，题目管理模块面临以下问题：

**性能问题：**
- 前端一次性加载所有题目数据（`limit=1000`），数据量大时页面卡顿
- 筛选和搜索在前端进行，无法利用数据库索引优化
- 后端 API 未返回 `total` 字段，无法显示总数

**用户体验问题：**
- 筛选操作需点击"应用筛选"按钮，操作繁琐
- 编辑题目需弹窗，操作步骤多
- 删除操作无撤销功能，误操作无法恢复
- 批量操作功能有限（仅删除）

**布局问题：**
- 列表视图和分类视图切换复杂，用户认知负担重
- 筛选条件和操作按钮混杂，功能分区不清晰
- 没有利用左侧空间做导航

### 2.4 新增批量操作接口

**文件**：`routes/questions.js`

**新增接口**：

```javascript
// 批量操作接口
router.post('/batch', async (req, res) => {
  try {
    const { action, ids, data } = req.body;

    if (!action || !ids || !Array.isArray(ids) || ids.length === 0) {
      res.status(400).json({ error: '参数错误' });
      return;
    }

    // 限制单次操作数量
    if (ids.length > 100) {
      res.status(400).json({ error: '单次最多操作100条记录' });
      return;
    }

    const placeholders = ids.map(() => '?').join(',');

    switch (action) {
      case 'delete':
        await db.run(`DELETE FROM questions WHERE id IN (${placeholders})`, ids);
        res.json({ success: true, affected: ids.length });
        break;

      case 'updateDifficulty':
        await db.run(
          `UPDATE questions SET difficulty = ? WHERE id IN (${placeholders})`,
          [data.difficulty, ...ids]
        );
        res.json({ success: true, affected: ids.length });
        break;

      case 'updateType':
        await db.run(
          `UPDATE questions SET type = ? WHERE id IN (${placeholders})`,
          [data.type, ...ids]
        );
        res.json({ success: true, affected: ids.length });
        break;

      case 'move':
        await db.run(
          `UPDATE questions SET subject_id = ?, subcategory_id = ? WHERE id IN (${placeholders})`,
          [data.subjectId, data.subcategoryId, ...ids]
        );
        res.json({ success: true, affected: ids.length });
        break;

      default:
        res.status(400).json({ error: '未知操作类型' });
    }
  } catch (error) {
    console.error('批量操作失败:', error);
    res.status(500).json({ error: '操作失败' });
  }
});
```

```javascript
// 修改 loadQuestions 方法（第 161-199 行）
async loadQuestions(params = {}) {
  try {
    this.isLoading = true
    this.error = null

    const {
      subjectId = null,
      subcategoryId = null,
      type = null,
      keyword = null,
      page = 1,
      limit = 50
    } = params

    let url = `${getApiBaseUrl()}/questions`
    const queryParams = []

    if (subjectId) queryParams.push(`subjectId=${subjectId}`)
    if (subcategoryId) queryParams.push(`subcategoryId=${subcategoryId}`)
    if (type) queryParams.push(`type=${type}`)
    if (keyword) queryParams.push(`keyword=${encodeURIComponent(keyword)}`)
    queryParams.push(`page=${page}`)
    queryParams.push(`limit=${limit}`)

    if (queryParams.length > 0) {
      url += `?${queryParams.join('&')}`
    }

    const response = await fetch(url)
    if (response.ok) {
      const result = await response.json()
      // 适配新格式
      this.questions = Array.isArray(result.data) ? result.data : (Array.isArray(result) ? result : [])
      this.pagination = {
        total: result.total || this.questions.length,
        page: result.page || page,
        limit: result.limit || limit
      }
      return result
    }
  } catch (error) {
    this.error = error.message
    this.questions = []
  } finally {
    this.isLoading = false
  }
}
```

```vue
<script setup>
import { ref, reactive, computed, watch, onMounted } from 'vue';
import { useQuestionStore } from '../../../stores/questionStore';

const questionStore = useQuestionStore();

// 分页状态
const pagination = reactive({
  page: 1,
  limit: 50,
  total: 0
});

// 筛选条件
const filterSubjectId = ref('');
const filterSubcategoryId = ref('');
const filterType = ref('');
const searchKeyword = ref('');

// 防抖搜索
let searchTimer = null;
const debouncedSearch = () => {
  clearTimeout(searchTimer);
  searchTimer = setTimeout(() => {
    loadQuestions();
  }, 300);
};

// 加载题目
const loading = ref(false);
const loadQuestions = async () => {
  loading.value = true;
  try {
    const result = await questionStore.loadQuestions({
      subjectId: filterSubjectId.value || null,
      subcategoryId: filterSubcategoryId.value || null,
      type: filterType.value || null,
      keyword: searchKeyword.value || null,
      page: pagination.page,
      limit: pagination.limit
    });
    pagination.total = result?.total || 0;
  } finally {
    loading.value = false;
  }
};

// 实时筛选：监听筛选条件变化
watch([filterSubjectId, filterSubcategoryId, filterType], () => {
  pagination.page = 1;
  loadQuestions();
});

// 搜索关键词变化（防抖）
watch(searchKeyword, () => {
  debouncedSearch();
});

// 分页变化
const handlePageChange = (page) => {
  pagination.page = page;
  loadQuestions();
};

const handleSizeChange = (size) => {
  pagination.limit = size;
  pagination.page = 1;
  loadQuestions();
};

// 初始加载
onMounted(() => {
  loadQuestions();
});
</script>
```

### 5.1 行内快速编辑（P0）

```vue
<el-table-column prop="content" label="题目内容" min-width="300">
  <template #default="{ row }">
    <div 
      v-if="editingId !== row.id"
      @dblclick="startInlineEdit(row)"
      class="editable-cell"
    >
      {{ truncate(row.content, 100) }}
    </div>
    <el-input
      v-else
      v-model="editingContent"
      @blur="saveInlineEdit(row)"
      @keyup.enter="saveInlineEdit(row)"
      @keyup.escape="cancelInlineEdit"
      autofocus
    />
  </template>
</el-table-column>

<script setup>
const editingId = ref(null);
const editingContent = ref('');

const startInlineEdit = (row) => {
  editingId.value = row.id;
  editingContent.value = row.content;
};

const saveInlineEdit = async (row) => {
  if (editingContent.value !== row.content) {
    // 调用更新接口
    await fetch(`/api/questions/${row.id}`, {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ ...row, content: editingContent.value })
    });
    ElMessage.success('修改成功');
  }
  editingId.value = null;
};

const cancelInlineEdit = () => {
  editingId.value = null;
};
</script>
```

### 5.2 删除撤销功能（P0）

```javascript
const deleteWithUndo = (row) => {
  // 临时从列表移除
  const index = questions.value.findIndex(q => q.id === row.id);

    message: '题目已删除',

    await fetch(`/api/questions/${row.id}`, { method: 'DELETE' });
  }, 3000);
};
```

### 9.5 已知问题修复

#### 学科导航题目数量显示
- **问题**：学科导航树没有显示学科和子分类下的题目数量
- **原因**：前端加载学科数据时没有获取子分类统计
- **解决**：在 `questionStore.fetchAndCacheCoreData` 中并行请求 `/questions/subcategories/stats`，合并统计到 subjects 数据中

#### 预览功能性能优化
- **问题**：点击预览按钮有时响应缓慢
- **原因**：每次预览都要发送 API 请求获取题目详情
- **解决**：
  1. 添加预览缓存机制（`previewCache` Map），最多缓存50条
  2. 添加加载状态显示（`previewLoading`）
  3. 重复预览同一题目直接使用缓存数据

#### 富文本内容处理
- **问题**：行内编辑会破坏富文本内容；预览弹窗中选项和解析不显示富文本
- **解决**：
  1. 添加 `isRichText()` 检测富文本内容
  2. 富文本内容禁止行内编辑，提示用户使用完整编辑功能
  3. 预览弹窗中使用 `v-html` 渲染选项、答案、解析
