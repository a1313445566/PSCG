# 学科独立题目数量配置

## 需求概述

**需求名称**：答题设置 - 学科独立题目数量控制  
**需求类型**：功能增强  
**优先级**：中  
**创建日期**：2026年3月24日  

### 背景

当前系统的答题设置中，题目数量控制是全局配置，所有学科使用相同的题目数量设置。实际使用中，不同学科的题目特点不同，需要不同的题目数量配置。例如：
- 语文题目较长，可能需要较少的题目数量
- 数学题目较短，可以设置更多的题目数量
- 新增学科时，应自动使用全局默认配置

## 功能需求

### 1. 全局默认配置（保留现有）

保持现有的全局题目数量设置，作为未单独配置学科的默认值：

```js
{
  fixedQuestionCount: false,      // 是否固定题目数量
  minQuestionCount: 3,            // 随机最小值
  maxQuestionCount: 5,            // 随机最大值
  fixedQuestionCountValue: 5      // 固定数量值
}
```

### 2. 学科独立配置（新增）

新增按学科独立配置的功能：

```js
{
  subjectQuestionCounts: {
    // 学科ID: 配置对象
    1: { enabled: true, fixed: false, min: 5, max: 10, value: 5 },   // 语文
    2: { enabled: true, fixed: true, min: 3, max: 5, value: 8 },     // 数学
    // 未配置或 enabled: false 的学科使用全局默认
  }
}
```

### 3. 配置优先级

```
学科独立配置（已启用） > 全局默认配置
```

### 4. 配置逻辑

```js
function getQuestionCountForSubject(subjectId) {
  const subjectConfig = settings.subjectQuestionCounts?.[subjectId]
  
  // 如果学科有独立配置且已启用
  if (subjectConfig?.enabled) {
    if (subjectConfig.fixed) {
      return subjectConfig.value
    }
    return random(subjectConfig.min, subjectConfig.max)
  }
  
  // 否则使用全局默认
  if (settings.fixedQuestionCount) {
    return settings.fixedQuestionCountValue
  }
  return random(settings.minQuestionCount, settings.maxQuestionCount)
}
```

## 技术方案

### 1. 数据存储

在现有的 settings 中扩展，存储于数据库 `settings` 表：

| 字段名 | 值 | 说明 |
|--------|-----|------|
| `fixedQuestionCount` | 'false' | 全局默认：是否固定 |
| `minQuestionCount` | '3' | 全局默认：随机最小值 |
| `maxQuestionCount` | '5' | 全局默认：随机最大值 |
| `fixedQuestionCountValue` | '5' | 全局默认：固定值 |
| `subjectQuestionCounts` | JSON字符串 | 学科独立配置 |

`subjectQuestionCounts` JSON 结构：
```json
{
  "1": { "enabled": true, "fixed": false, "min": 5, "max": 10, "value": 5 },
  "2": { "enabled": true, "fixed": true, "min": 3, "max": 5, "value": 8 }
}
```

### 2. 修改文件清单

| 文件路径 | 修改内容 |
|----------|----------|
| `src/stores/questionStore.js` | 1. settings 增加 subjectQuestionCounts<br>2. 新增 getQuestionCountForSubject 方法<br>3. loadSettings 解析新字段 |
| `src/components/admin/basic-settings/AnswerSetting.vue` | 1. 新增 props: subjects, subjectQuestionCounts<br>2. 界面增加学科配置区域<br>3. 保存时提交新配置 |
| `src/views/AdminView.vue` | 传递 subjects 和 subjectQuestionCounts 给组件 |
| `src/views/QuizView.vue` | 使用 getQuestionCountForSubject 获取题目数量 |

### 3. 代码示例

#### questionStore.js

```js
// state
settings: {
  // 保留现有
  fixedQuestionCount: false,
  minQuestionCount: 3,
  maxQuestionCount: 5,
  fixedQuestionCountValue: 5,
  // 新增
  subjectQuestionCounts: {}
}

// getters
getQuestionCountForSubject: (state) => (subjectId) => {
  const config = state.settings.subjectQuestionCounts?.[subjectId]
  
  if (config?.enabled) {
    if (config.fixed) {
      return config.value
    }
    return Math.floor(Math.random() * (config.max - config.min + 1)) + config.min
  }
  
  // 使用全局默认
  if (state.settings.fixedQuestionCount) {
    return state.settings.fixedQuestionCountValue
  }
  return Math.floor(Math.random() * 
    (state.settings.maxQuestionCount - state.settings.minQuestionCount + 1)
  ) + state.settings.minQuestionCount
}

// loadSettings 中解析
this.settings.subjectQuestionCounts = 
  JSON.parse(settings.subjectQuestionCounts?.replace(/'/g, '') || '{}')
```

#### QuizView.vue

```js
// 修改前
const { fixedQuestionCount, minQuestionCount, maxQuestionCount, fixedQuestionCountValue } = 
  settingsStore.settings
let questionCount
if (fixedQuestionCount) {
  questionCount = fixedQuestionCountValue
} else {
  questionCount = Math.floor(Math.random() * (maxQuestionCount - minQuestionCount + 1)) + minQuestionCount
}

// 修改后
const questionCount = settingsStore.getQuestionCountForSubject(subjectId.value)
```

## 技术约束

### 1. 技术栈约束

| 约束项 | 版本要求 | 说明 |
|--------|----------|------|
| 前端框架 | Vue 3.5.x (`^3.5.25`) | 必须使用 `<script setup>` 语法 |
| 状态管理 | Pinia 3.x (`^3.0.4`) | 遵循现有 store 结构 |
| UI组件库 | **Element Plus 2.x (`^2.13.3`)** | ⚠️ 版本敏感，不同版本 API 差异较大 |
| 后端框架 | Express 4.x (`^4.18.2`) | 使用 `require`/`module.exports` |
| 数据库 | MySQL 8.0 | 使用现有 `settings` 表 |
| 构建工具 | Vite 5.x (`^5.2.0`) | - |

**Element Plus 版本注意事项**：
- 当前版本 `2.13.3`，需确保开发环境一致
- `el-input-number` 组件：v2.x 使用 `:min` / `:max` 属性
- `el-button` 组件：v2.x 使用 `:class` 动态绑定样式
- 若升级至 v3.x，部分 API 可能不兼容，需评估后再升级

### 2. 数据库约束

**现有 `settings` 表结构**：
```sql
CREATE TABLE settings (
  setting_key VARCHAR(100) PRIMARY KEY,
  setting_value TEXT
);
```

**约束条件**：
- 不能修改表结构，使用 `setting_value` 存储 JSON 字符串
- `subjectQuestionCounts` 字段值需 JSON 序列化后存储
- 存储格式：`'{"1":{"enabled":true,"fixed":false,"min":5,"max":10,"value":5}}'`
- 读取时需处理单引号转义：`.replace(/'/g, '')`

### 3. 现有代码约束

**必须保留的逻辑**：
- 全局默认配置的 getter `getQuestionCount`（现有代码可能依赖）
- 现有 props 定义（`randomizeAnswers`, `fixedQuestionCount` 等）
- `update-settings` 事件签名

**禁止修改**：
- 数据库表结构
- 后端 API 接口签名（`/api/settings` 返回格式）
- 其他页面使用 `settingsStore.settings` 的方式

### 4. 兼容性约束

| 场景 | 处理方式 |
|------|----------|
| 旧版本数据（无 `subjectQuestionCounts`） | 默认为 `{}`，使用全局配置 |
| `subjectQuestionCounts` 为空字符串 | 解析为 `{}` |
| 学科 ID 不存在于配置中 | 使用全局默认配置 |
| 学科被删除 | 配置保留，不影响其他学科 |

### 5. 数值约束

| 参数 | 最小值 | 最大值 | 默认值 |
|------|--------|--------|--------|
| `min` | 1 | 50 | 3 |
| `max` | 1 | 50 | 5 |
| `value` | 1 | 50 | 5 |

**业务规则**：
- `max` 必须 ≥ `min`（前端自动纠正）
- 最大值设为 50，满足部分学科需要整套题目练习的场景
- 题目数量不能 ＞ 题库实际题目数（后端校验）

### 6. 性能约束

| 约束项 | 要求 |
|--------|------|
| 配置加载 | 在 `loadSettings()` 中一次性解析，避免重复 JSON.parse |
| getter 性能 | 纯函数，无副作用，可缓存 |
| 界面渲染 | 学科列表使用虚拟滚动（学科数量 > 20 时） |

### 7. 安全约束

- 管理员权限验证（已有，无需额外处理）
- 前端输入校验 + 后端二次校验
