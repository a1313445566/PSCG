## 功能测试流程

### 测试场景1：开启选项打乱

#### 步骤：
1. **登录后台管理系统**
   - 访问：http://your-domain/admin
   - 使用管理员账号登录

2. **设置选项打乱为"开启"**
   - 进入"系统设置"
   - 找到"答题设置"
   - 点击"开启"按钮
   - 点击"保存设置"
   - 确认显示：`当前状态：开启`

3. **开始答题测试**
   - 前端学生账号登录
   - 选择任意学科和题库
   - 开始答题

4. **观察选项顺序**
   - 记录第一道题的选项顺序
   - 例如：A.苹果 B.香蕉 C.橙子 D.葡萄

5. **重新开始答题**
   - 提交或退出当前答题
   - 重新选择同一题库
   - 观察同一题目的选项顺序

6. **验证结果**
   - ✅ 选项顺序应该不同
   - ✅ 每次答题选项顺序随机变化
   - ✅ 选择答案后提交正确

#### 预期结果：
```
第一次答题：A.苹果 B.香蕉 C.橙子 D.葡萄
第二次答题：A.橙子 B.葡萄 C.苹果 D.香蕉  ← 顺序改变
第三次答题：A.葡萄 B.苹果 C.香蕉 D.橙子  ← 又改变
```

## 数据库验证

### 检查打乱后的选项是否保存
```sql
-- 查看最近的答题记录
SELECT 
  qat.id,
  qat.question_id,
  q.content,
  q.options as original_options,
  qat.shuffled_options,
  qat.user_answer,
  qat.correct_answer,
  qat.is_correct,
  qat.created_at
FROM question_attempts qat
JOIN questions q ON qat.question_id = q.id
ORDER BY qat.created_at DESC
LIMIT 5;
```

### 验证点：
- ✅ `shuffled_options` 字段有值
- ✅ 开启打乱时：`shuffled_options` ≠ `original_options`
- ✅ 关闭打乱时：`shuffled_options` = `original_options`

## 常见问题排查

### 问题1：选项顺序没有变化
**检查**：
- 后台设置是否为"开启"
- 是否点击了"保存设置"
- 清除浏览器缓存重试

### 问题2：答案验证错误
**检查**：
- 查看浏览器控制台错误
- 检查后端日志：`pm2 logs PSCG`
- 验证数据库中的 shuffled_options 字段

### 问题3：设置不生效
**检查**：
- 清除浏览器 localStorage
- 重新登录
- 检查 settingsStore 是否正确加载
