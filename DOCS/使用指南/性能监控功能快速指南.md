# 🚀 性能监控功能快速指南

## 📋 核心功能概览

### ✅ **已实现的后端功能**
1. **API响应时间监控** - 监控所有API请求的响应时间
2. **数据库性能监控** - 记录查询时间和慢查询
3. **系统健康检查** - 监控内存使用和运行状态
4. **缓存管理** - 统计缓存命中率和清理缓存

### 🎯 **可用API端点**

#### 1. 数据库性能
```bash
GET /api/performance/db
```
**返回：** 平均查询时间、慢查询统计、查询总数

#### 2. 系统健康
```bash
GET /api/performance/health
```
**返回：** 内存使用、运行时间、健康状态

#### 3. 缓存统计
```bash
GET /api/cache/stats
```
**返回：** 命中率、缓存项数量、使用统计

#### 4. 清空缓存
```bash
POST /api/cache/clear
```
**返回：** 操作结果

## 🎨 前端集成建议

### 1. 最小化组件（推荐先实现）
```vue
<template>
  <div class="performance-dashboard">
    <!-- 健康状态卡片 -->
    <HealthCard :data="healthData" />
    
    <!-- API性能表格 -->
    <ApiTable :metrics="apiMetrics" />
    
    <!-- 数据库监控 -->
    <DbMonitor :stats="dbStats" />
    
    <!-- 缓存管理 -->
    <CacheManager :cache="cacheStats" />
  </div>
</template>

<script setup>
// 统一数据获取
const fetchData = async () => {
  const urls = [
    '/api/performance/health',
    '/api/performance/db',
    '/api/cache/stats'
  ];
  
  return Promise.all(urls.map(url => fetch(url).then(r => r.json())));
};
</script>
```

### 2. 数据刷新策略
```javascript
// 简单的定时刷新
let refreshTimer = null;

const startMonitoring = () => {
  refreshTimer = setInterval(async () => {
    const [health, db, cache] = await fetchData();
    // 更新界面数据
  }, 30000); // 30秒刷新一次
};

const stopMonitoring = () => {
  if (refreshTimer) clearInterval(refreshTimer);
};
```

## 📊 关键指标说明

### 1. 性能评分标准
| 指标 | 🟢 优秀 | 🟡 良好 | 🟠 一般 | 🔴 较差 |
|------|---------|---------|---------|---------|
| API响应时间 | <100ms | 100-300ms | 300-500ms | >500ms |
| 数据库查询 | <50ms | 50-100ms | 100-200ms | >200ms |
| 缓存命中率 | >90% | 70-90% | 50-70% | <50% |
| 内存使用率 | <60% | 60-80% | 80-90% | >90% |

### 2. 告警阈值建议
```javascript
const thresholds = {
  api: { slow: 500, errorRate: 5 },      // API慢响应500ms，错误率5%
  db: { slow: 200, slowRate: 5 },        // 数据库慢查询200ms，慢查询率5%
  system: { memory: 80, uptime: 604800 } // 内存80%，运行7天
};
```

## ⚠️ 注意事项

### 1. 权限控制
- 性能监控功能应限制为管理员访问
- 建议添加认证中间件
- 记录访问日志

### 2. 性能影响
- 监控数据收集会占用少量系统资源
- 建议合理设置监控频率
- 定期清理历史数据

### 3. 数据安全
- 敏感查询信息需要脱敏处理
- 监控API应限制访问频率
- 定期备份重要监控数据

## 📞 快速支持

### 常见问题排查
1. **API返回404**：检查服务器是否运行，中间件是否正确加载
2. **数据不更新**：检查定时刷新是否正常工作
3. **权限问题**：确认用户有管理员权限
