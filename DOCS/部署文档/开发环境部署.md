# 开发环境部署指南

> PSCG 智能学习系统 | 版本：v1.0 | 最后更新：2026-04-05
> 适用于 Windows / macOS / Linux 本地开发环境

## 一、Windows 开发环境

### 1.1 软件安装

#### Node.js 安装

1. 访问 https://nodejs.org/
2. 下载 **LTS 版本**（推荐 v20.x）
3. 运行安装程序，勾选 **"Add to PATH"**
4. 验证安装：

```powershell
node -v    # 应显示 v20.x.x
npm -v     # 应显示 10.x.x
```

#### MySQL 安装

1. 访问 https://dev.mysql.com/downloads/installer/
2. 下载 **MySQL Installer**（Windows 版）
3. 选择 **"Server only"** 或 **"Developer Default"**
4. 设置 root 密码（请记住！）
5. 选择 **"Development Computer"** 配置类型
6. 完成安装后验证：

```powershell
mysql -u root -p -e "SELECT VERSION();"
```

#### Git 安装

1. 访问 https://git-scm.com/download/win
2. 下载并运行安装程序
3. 使用默认选项即可
4. 验证：

```powershell
git --version
```

### 1.2 VSCode 插件推荐

打开 VSCode，安装以下插件：

| 插件名称 | 用途 | 必须 |
|----------|------|------|
| **Volar** (Vue Language Features) | Vue3 语法高亮/智能提示 | 是 |
| **ESLint** | 代码规范实时检查 | 是 |
| **Prettier** | 代码格式化 | 是 |
| **SCSS IntelliSense** | SCSS 语法支持 | 推荐 |
| **Error Lens** | 行内错误显示 | 推荐 |
| **GitLens** | Git 增强工具 | 推荐 |
| **Thunder Client** | API 测试（替代 Postman） | 可选 |

### 1.3 首次运行步骤

```powershell
# 1. 进入项目目录
cd E:\PSCG

# 2. 安装依赖
npm install

# 3. 配置环境变量
copy .env.example .env
notepad .env   # 编辑配置（见下方说明）

# 4. 初始化数据库
# 先确保 MySQL 正在运行，然后：
mysql -u root -p < migrations/migrate_to_aittest.sql

# 5. 启动后端服务（终端窗口 1）
npm run server

# 6. 新开终端窗口，启动前端开发服务器（终端窗口 2）
npm run dev
```

### 1.4 Windows 特殊注意事项

**Sharp 模块问题**（常见于 Windows）：

```powershell
# 如果 sharp 安装失败
npm install sharp --ignore-scripts
npm rebuild sharp
```

**路径分隔符**：项目统一使用正斜杠 `/`，Git Bash 和 PowerShell 都能正确处理。

**防火墙提示**：首次运行时 Windows Defender 可能弹出防火墙提示，选择 **"允许访问"**。

## 二、macOS 开发环境

### 2.1 软件安装（使用 Homebrew）

```bash
# 安装 Homebrew（如果未安装）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装 Node.js
brew install node@20

# 安装 MySQL
brew install mysql

# 启动 MySQL 服务
brew services start mysql

# 安装 Git（通常 macOS 自带）
brew install git

# 验证安装
node -v
npm -v
mysql --version
git --version
```

### 2.2 MySQL 初始配置

```bash
# 安全配置（设置 root 密码等）
mysql_secure_installation

# 登录测试
mysql -u root -p
```

### 2.3 VSCode 插件

与 Windows 相同，在 VSCode Extensions 中搜索安装 Volar、ESLint、Prettier。

### 2.4 首次运行

```bash
# 1. 克隆/进入项目
cd /path/to/PSCG

# 2. 安装依赖
npm install

# 3. 配置环境变量
cp .env.example .env
vim .env   # 或用 nano .env

# 4. 初始化数据库
mysql -u root -p < migrations/migrate_to_aittest.sql

# 5. 启动后端（终端 1）
npm run server

# 6. 启动前端（终端 2，新开标签页）
npm run dev
```

### 2.5 macOS 特殊注意事项

**权限问题**：如果遇到 `EACCES` 错误，避免使用 `sudo npm install`，改为修复 npm 权限：

```bash
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

## 三、Linux 开发环境（Ubuntu/Debian）

### 3.1 软件安装

```bash
# 更新包列表
sudo apt update

# 安装 Node.js（通过 NodeSource，获取最新 LTS 版本）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 安装 MySQL
sudo apt install -y mysql-server

# 安装 Git
sudo apt install -y git

# 安装构建工具（某些原生模块需要）
sudo apt install -y build-essential python3

# 验证安装
node -v    # v20.x.x
npm -v     # 10.x.x
mysql --version
```

### 3.2 MySQL 初始化

```bash
# 安全配置
sudo mysql_secure_installation

# 以 root 身份登录（Ubuntu 的 MySQL 默认使用 auth_socket）
sudo mysql

# 在 MySQL 中执行：
CREATE USER 'PSCG'@'localhost' IDENTIFIED BY 'your-password';
CREATE DATABASE pscg DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
GRANT ALL PRIVILEGES ON pscg.* TO 'PSCG'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 3.3 VSCode + Remote-SSH（推荐）

如果开发机是远程 Linux 服务器：

1. 本地安装 VSCode
2. 安装 **Remote - SSH** 扩展
3. 连接远程服务器后自动同步插件和环境

### 3.4 首次运行

```bash
# 进入项目目录
cd ~/PSCG

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
nano .env

# 初始化数据库
```

## 六、热更新（HMR）说明

本项目使用 Vite 作为构建工具，支持极速热更新：

### 6.1 HMR 工作原理

```
修改 .vue/.js/.scss 文件
    ↓
Vite Dev Server 检测变更（通过 WebSocket）
    ↓
只重新编译修改的模块（非全量重载）
    ↓
推送更新到浏览器
    ↓
页面无刷新更新（保留状态）
```

### 6.2 触发全量 reload 的情况

以下情况会导致页面完全刷新（丢失状态）：
- 修改 `vite.config.js`
- 修改 `index.html`
- 修改路由配置 (`src/router/index.js`)
- 某些 CSS 变更（取决于具体改动）

### 6.3 HMR 不生效排查

1. **确认终端无报错**：查看 `npm run dev` 的输出
2. **检查端口**：确认 5173 端口未被占用
3. **清除浏览器缓存**：硬刷新 `Ctrl+Shift+R`
4. **检查防火墙**：确保未阻止 WebSocket 连接
5. **重启开发服务器**：`Ctrl+C` 后重新 `npm run dev`

## 七、开发代理配置详解

### 7.1 当前代理规则

[vite.config.js](file:///e:/PSCG/vite.config.js) 中配置了三个代理规则：

```javascript
// vite.config.js
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:3001',  // 转发目标
      changeOrigin: true                  // 修改请求头 Origin
    },
    '/images': {
      target: 'http://localhost:3001',
      changeOrigin: true
    },
    '/audio': {
      target: 'http://localhost:3001',
      changeOrigin: true
    }
  }
}
```

### 7.2 代理工作流

```
浏览器发起: GET /api/questions?page=1
    ↓
Vite Dev Server (:5173) 拦截 /api/*
    ↓
转发到: http://localhost:3001/api/questions?page=1
    ↓
Express 返回响应
    ↓
原路返回给浏览器
```

### 7.3 为什么需要代理？

1. **解决跨域**：前端 `:5173` 与后端 `:3001` 不同源
2. **模拟生产环境**：生产环境中 API 也是通过 `/api/...` 访问
3. **统一请求地址**：前端代码中写 `/api/...` 无需区分环境

### 7.4 自定义代理（高级）

如需添加更多代理规则：

```javascript
// vite.config.js
server: {
  proxy: {
    // 示例：代理 WebSocket
    '/ws': {
      target: 'ws://localhost:3001',
      ws: true
    },
    // 示例：重写路径
    '/old-api': {
      target: 'http://localhost:3001',
      rewrite: (path) => path.replace(/^\/old-api/, '/api')
    }
  }
}
```

## 八、常见开发问题速查

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| `EADDRINUSE :3001` | 端口被占用 | `netstat -ano \| findstr :3001` → `taskkill /PID xxx /F` |
| `ECONNREFUSED 127.0.0.1:3306` | MySQL 未启动 | 启动 MySQL 服务 |
| `Cannot find module` | 依赖缺失 | `rm -rf node_modules && npm install` |
| HMR 不更新 | WebSocket 被阻 | 检查防火墙/杀毒软件 |
| ESLint 报错 | 代码不规范 | `npm run lint` 自动修复 |
| sharp 编译失败 | 缺少 Python/构建工具 | 安装 build-tools 或 `npm rebuild sharp` |
| 页面空白白屏 | 路由/编译错误 | 查看 F12 Console 错误信息 |

## 九、开发效率技巧

### 9.1 并行启动前后端（Windows PowerShell 脚本）

创建 `dev.ps1`：

```powershell
# 同时启动后端和前端开发服务器
Start-Process -NoNewWindow npm -ArgumentList "run","server"
Start-Sleep -Seconds 3
Start-Process -NoNewWindow npm -ArgumentList "run","dev"
Write-Host "前后端服务已启动" -ForegroundColor Green
```

### 9.2 常用开发命令速查

```bash
# 日常开发循环
npm run dev              # 启动前端
npm run server           # 启动后端
npm run lint             # 修复代码风格
npm run format           # 格式化代码

# 构建与预览
npm run build            # 构建生产版本
npm run preview          # 预览构建结果

# 数据库相关
mysql -u PSCG -p pscg < migrations/xxx.sql   # 执行迁移
mysqldump -u PSCG -p pscg > backup.sql       # 备份数据库

# Git 操作
git status               # 查看状态
git add .                # 暂存所有
git commit -m "feat(xxx): 描述"  # 提交
git push                 # 推送
```
