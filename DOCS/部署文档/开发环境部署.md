# 开发环境部署

## 系统要求

### 硬件要求
- **CPU**: 至少 2 核处理器
- **内存**: 至少 4GB RAM
- **存储空间**: 至少 10GB 可用空间

### 软件要求
- **操作系统**: Windows 10/11, macOS, Linux
- **Node.js**: 18.x 或更高版本（推荐 v20+）
- **npm**: 8.0 或更高版本
- **MySQL**: 8.0 或更高版本
- **Git**: 2.0 或更高版本

## 安装 Node.js

### Windows 系统
1. 访问 [Node.js 官网](https://nodejs.org/)
2. 下载 LTS 版本的 Node.js 安装包
3. 运行安装包，按照提示完成安装
4. 打开命令提示符，运行 `node -v` 和 `npm -v` 验证安装成功

### macOS 系统
1. 访问 [Node.js 官网](https://nodejs.org/)
2. 下载 LTS 版本的 Node.js 安装包
3. 运行安装包，按照提示完成安装
4. 打开终端，运行 `node -v` 和 `npm -v` 验证安装成功

### Linux 系统
1. 使用包管理器安装 Node.js
   - Ubuntu/Debian: `sudo apt install nodejs npm`
   - CentOS/RHEL: `sudo yum install nodejs npm`
2. 或使用 nvm 安装
   ```bash
   curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.3/install.sh | bash
   nvm install --lts
   ```
3. 运行 `node -v` 和 `npm -v` 验证安装成功

## 安装 MySQL

### Windows 系统
1. 访问 [MySQL 官网](https://dev.mysql.com/downloads/installer/)
2. 下载 MySQL Installer
3. 运行安装程序，选择「Developer Default」安装类型
4. 按照提示完成安装，设置 root 密码
5. 启动 MySQL 服务

### macOS 系统
1. 访问 [MySQL 官网](https://dev.mysql.com/downloads/mysql/)
2. 下载 macOS 版本的 MySQL 安装包
3. 运行安装包，按照提示完成安装
4. 启动 MySQL 服务

### Linux 系统
1. 使用包管理器安装 MySQL
   - Ubuntu/Debian: `sudo apt install mysql-server`
   - CentOS/RHEL: `sudo yum install mysql-server`
2. 启动 MySQL 服务: `sudo systemctl start mysql`
3. 运行安全配置: `sudo mysql_secure_installation`

## 配置数据库

1. 打开 MySQL 客户端
2. 创建数据库：
   ```sql
   CREATE DATABASE pscg DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
3. 创建用户并授权（使用 PSCG 大写）：
   ```sql
   CREATE USER 'PSCG'@'localhost' IDENTIFIED BY 'your_password';
   GRANT ALL PRIVILEGES ON pscg.* TO 'PSCG'@'localhost';
   FLUSH PRIVILEGES;
   ```

## 配置环境变量

1. 复制 `.env.example` 文件为 `.env`：
   ```bash
   cp .env.example .env
   ```

2. 编辑 `.env` 文件，配置以下内容：
   ```
   # 服务器配置
   SERVER_PORT=3001

   # API 地址配置（开发环境使用 /api 代理）
   VITE_API_URL=/api

   # 签名密钥（请替换为实际密钥）
   SIGNATURE_SECRET=your-secret-key-here
   VITE_SIGNATURE_SECRET=your-secret-key-here

   # 数据库配置
   DB_HOST=127.0.0.1
   DB_PORT=3306
   DB_USER=PSCG
   DB_PASSWORD=your_password
   DB_NAME=pscg
   ```

**注意**：
- `SIGNATURE_SECRET` 用于 API 签名验证，请替换为安全的随机密钥
- 生成密钥：`node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"`
- 数据库用户名应与 `config/database.js` 中的配置一致（默认为 `PSCG`）

## 初始化数据库

**方法 1：手动导入 SQL 文件**
```bash
mysql -u PSCG -p pscg < scripts/db-migration.sql
```

**方法 2：使用迁移脚本（migrations 目录）**
```bash
mysql -u PSCG -p pscg < migrations/migrate_to_aittest.sql
```

**注意**：数据库表结构会在后端服务首次启动时自动创建（通过 `services/database.js` 中的初始化逻辑）

## 常见问题

### 端口被占用
- **问题**: 端口 3001 或 5173 被占用
- **解决方案**: 更改 `.env` 文件中的 `PORT` 配置或停止占用端口的进程

### 数据库连接失败
- **问题**: 无法连接到 MySQL 数据库
- **解决方案**: 检查数据库配置是否正确，确保 MySQL 服务正在运行

### 依赖安装失败
- **问题**: `npm install` 失败
- **解决方案**: 清理 npm 缓存，重新安装依赖
  ```bash
  npm cache clean --force
  npm install
  ```

### 启动失败
- **问题**: 前端或后端启动失败
- **解决方案**: 查看错误信息，检查配置文件和依赖是否正确

## 开发流程

1. **创建分支**: 从 main 分支创建新的开发分支
2. **编写代码**: 按照编码规范编写代码
3. **运行测试**: 确保代码通过测试
4. **提交代码**: 提交代码到远程仓库
5. **创建 PR**: 创建 Pull Request 进行代码审查
6. **合并代码**: 代码审查通过后合并到 main 分支

## 注意事项

- **环境变量**: 不要将包含敏感信息的 .env 文件提交到版本控制系统
- **依赖管理**: 安装新依赖时，使用 `npm install` 命令并提交 package.json 和 package-lock.json 文件
- **代码风格**: 遵循项目的编码规范，使用 ESLint 和 Prettier 保持代码风格一致
- **数据库操作**: 操作数据库时要注意备份，避免数据丢失
- **安全问题**: 注意防止 SQL 注入、XSS 攻击等安全问题
