# 📦 PSCG 系统部署配置指南

**文档版本**: v2.1.1  
**最后更新**: 2026-03-28

本文档描述将 PSCG 部署到生产环境的推荐流程，包含环境变量、安装、构建、PM2、Nginx、SSL、备份与故障排除。

## 0. 环境变量基线（示例）
在项目根目录创建 `.env`：
```bash
DB_HOST=localhost
DB_PORT=3306
DB_NAME=pscg_db
DB_USER=pscg_user
DB_PASSWORD=your_secure_password
DB_CHARSET=utf8mb4
PORT=3001
JWT_SECRET=your_jwt_secret_key
SESSION_SECRET=your_session_secret_key
SIGNATURE_SECRET=your_signature_secret       # 与前端 VITE_SIGNATURE_SECRET 一致
NODE_ENV=production
```
前端 `.env.production`：
```bash
VITE_API_URL=https://your-domain.com/api
VITE_SIGNATURE_SECRET=your_signature_secret   # 与后端 SIGNATURE_SECRET 一致
```

## 1. 环境要求
- OS：Ubuntu 20.04 LTS 或更高
- 内存：≥2GB；磁盘：≥10GB；公网 IP 或域名
- 软件：Node.js 18.x（推荐）、MySQL 8.0+、Nginx 1.18+、PM2 5.x+

## 2. 安装步骤
```bash
# 系统更新与基础工具
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl wget git build-essential

# Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
node -v && npm -v

# MySQL 8
sudo apt install -y mysql-server
sudo systemctl enable --now mysql
sudo mysql_secure_installation

# Nginx
sudo apt install -y nginx
sudo systemctl enable --now nginx
```

## 3. 数据库配置
```sql
CREATE DATABASE pscg_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'pscg_user'@'localhost' IDENTIFIED BY 'your_secure_password';
GRANT ALL PRIVILEGES ON pscg_db.* TO 'pscg_user'@'localhost';
FLUSH PRIVILEGES;
```

## 4. 项目部署
```bash
# 获取代码
git clone https://your-repo/pscg.git /var/www/pscg
cd /var/www/pscg

# 安装依赖并构建前端
npm install
npm run build

# (可选) 测试数据库连接
node -e "require('./services/database.js').testConnection()"
```

## 5. PM2 运行
`ecosystem.config.js` 示例：
```javascript
module.exports = {
  apps: [{
    name: 'pscg-server',
    script: 'server.cjs',
    instances: 'max',
    exec_mode: 'cluster',
    env: { NODE_ENV: 'production', PORT: 3001 }
  }]
};
```
启动：
```bash
pm2 start ecosystem.config.js
pm2 startup
pm2 save
pm2 status
```

## 6. Nginx 反向代理
`/etc/nginx/sites-available/pscg`：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        root /var/www/pscg/dist;
        try_files $uri $uri/ /index.html;
        expires 1h;
        add_header Cache-Control "public, immutable";
    }

    location /api {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        root /var/www/pscg/dist;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;

    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css text/xml text/javascript application/javascript application/xml+rss application/json image/svg+xml;
}
```
启用并重载：
```bash
sudo ln -s /etc/nginx/sites-available/pscg /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl restart nginx
```

## 7. SSL（可选）
```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com -d www.your-domain.com
sudo certbot renew --dry-run
```

## 8. 备份与维护
- 数据库：`mysqldump -u pscg_user -p'pwd' pscg_db > /var/backups/pscg/pscg_db.sql`
- 配置/上传：备份 `config/`、`.env`、`uploads/`；可用 crontab 每日 2:00，保留 7 天。
- 监控：`pm2 status / pm2 logs pscg-server`，`tail -f /var/log/nginx/error.log`。
- 更新：`sudo apt upgrade -y`，`npm update`，`npm install -g pm2@latest && pm2 update`。

## 9. 常见问题
- 502：检查 PM2 进程、3001 监听、Nginx 反代配置。
- DB 连接失败：核对 `.env`，确认 MySQL 运行、用户权限、端口开放。
- 静态资源 404：检查 `dist/` 路径与 Nginx root/try_files。
- 签名/CSRF 失败：确保前后端 SIGNATURE_SECRET 一致、时间同步；前端携带 `X-CSRF-Token`。
- 队列堆积（AI/语义/行为）：检查 `aiQueueService` 并发/重试配置与错误日志。
