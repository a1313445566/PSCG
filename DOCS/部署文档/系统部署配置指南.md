# 📦 PSCG 系统部署配置指南

本文档详细介绍了如何部署 PSCG 智能题库系统到生产环境。

## 1. 环境要求

### 1.1 系统要求
- **操作系统**：Ubuntu 20.04 LTS 或更高版本
- **内存**：至少 2GB RAM
- **存储**：至少 10GB 可用空间
- **网络**：公网IP地址或域名

### 1.2 软件依赖
| 软件 | 版本要求 | 安装方法 |
|------|----------|----------|
| Node.js | 16.x 或更高 | `apt install nodejs` |
| NPM | 8.x 或更高 | `apt install npm` |
| MySQL | 8.0 或更高 | `apt install mysql-server` |
| Nginx | 1.18+ | `apt install nginx` |
| PM2 | 5.x+ | `npm install -g pm2` |

## 2. 安装步骤

### 2.1 系统更新
```bash
# 更新系统包
sudo apt update
sudo apt upgrade -y

# 安装基础工具
sudo apt install -y curl wget git build-essential
```

### 2.2 安装 Node.js
```bash
# 使用 NodeSource 安装 Node.js 18.x
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# 验证安装
node --version
npm --version
```

### 2.3 安装 MySQL
```bash
# 安装 MySQL
sudo apt install -y mysql-server

# 启动 MySQL
sudo systemctl start mysql
sudo systemctl enable mysql

# 运行安全配置
sudo mysql_secure_installation
```

### 2.4 安装 Nginx
```bash
# 安装 Nginx
sudo apt install -y nginx

# 启动 Nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

## 3. 数据库配置

### 3.1 创建数据库和用户
```sql
-- 登录 MySQL
sudo mysql

-- 创建数据库
CREATE DATABASE pscg_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户
CREATE USER 'pscg_user'@'localhost' IDENTIFIED BY 'your_secure_password';

-- 授予权限
GRANT ALL PRIVILEGES ON pscg_db.* TO 'pscg_user'@'localhost';

-- 刷新权限
FLUSH PRIVILEGES;

-- 退出
EXIT;
```

### 3.2 配置数据库连接
在项目根目录创建 `.env` 文件：
```bash
DB_HOST=localhost
DB_PORT=3306
DB_NAME=pscg_db
DB_USER=pscg_user
DB_PASSWORD=your_secure_password
DB_CHARSET=utf8mb4
JWT_SECRET=your_jwt_secret_key
SESSION_SECRET=your_session_secret_key
NODE_ENV=production
```

## 5. PM2 进程管理

### 5.1 安装 PM2
```bash
npm install -g pm2
```

### 5.2 配置 PM2
```bash
# 创建 ecosystem.config.js
cat > ecosystem.config.js << EOF
module.exports = {
  apps: [{
    name: 'pscg-server',
    script: 'server.cjs',
    instances: 'max',
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'production',
      PORT: 3001
    },
    error_file: '/var/log/pm2/pscg-error.log',
    out_file: '/var/log/pm2/pscg-out.log',
    log_file: '/var/log/pm2/pscg-combined.log',
    time: true
  }]
};
EOF
```

### 5.3 启动服务
```bash
# 启动 PM2 服务
pm2 start ecosystem.config.js

# 设置开机启动
pm2 startup
pm2 save

# 查看状态
pm2 status
pm2 logs pscg-server
```

## 6. Nginx 反向代理配置

### 6.1 创建 Nginx 配置文件
```bash
sudo nano /etc/nginx/sites-available/pscg
```

添加以下配置：
```nginx
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;
    
    # 前端静态文件
    location / {
        root /var/www/pscg/dist;
        try_files $uri $uri/ /index.html;
        expires 1h;
        add_header Cache-Control "public, immutable";
    }
    
    # API 代理
    location /api {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
    
    # 静态资源
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        root /var/www/pscg/dist;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    
    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types
        text/plain
        text/css
        text/xml
        text/javascript
        application/javascript
        application/xml+rss
        application/json
        image/svg+xml;
}
```

### 6.2 启用站点
```bash
# 启用配置
sudo ln -s /etc/nginx/sites-available/pscg /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

### 8.1 监控命令

```bash
htop

```

### 8.2 备份脚本
创建备份脚本 `/usr/local/bin/backup-pscg.sh`：
```bash
#!/bin/bash
BACKUP_DIR="/var/backups/pscg"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR/$DATE

# 备份数据库
mysqldump -u pscg_user -p'your_secure_password' pscg_db > $BACKUP_DIR/$DATE/pscg_db.sql

# 备份配置文件
cp -r /var/www/pscg/config $BACKUP_DIR/$DATE/
cp /var/www/pscg/.env $BACKUP_DIR/$DATE/

# 备份上传文件
cp -r /var/www/pscg/uploads $BACKUP_DIR/$DATE/

# 压缩备份
tar -czf $BACKUP_DIR/pscg_backup_$DATE.tar.gz -C $BACKUP_DIR $DATE

# 清理旧备份（保留最近7天）
find $BACKUP_DIR -name "*.tar.gz" -mtime +7 -delete
find $BACKUP_DIR -type d -mtime +7 -exec rm -rf {} \;

echo "Backup completed: pscg_backup_$DATE.tar.gz"
```

### 8.3 设置定时备份
```bash
# 编辑 crontab
sudo crontab -e

# 添加以下行（每天凌晨2点备份）
0 2 * * * /bin/bash /usr/local/bin/backup-pscg.sh
```

## 9. 故障排除

### 9.1 常见问题

#### 问题1：502 Bad Gateway
```bash
# 检查后端服务是否运行
pm2 status pscg-server

# 检查端口是否监听
netstat -tlnp | grep :3001

# 重启服务
pm2 restart pscg-server
```

#### 问题2：数据库连接失败
```bash
# 检查 MySQL 服务
sudo systemctl status mysql

# 测试连接
mysql -u pscg_user -p'your_secure_password' -e "SHOW DATABASES;"

# 检查 .env 配置
cat /var/www/pscg/.env
```

#### 问题3：静态文件无法加载
```bash
# 检查文件权限
ls -la /var/www/pscg/dist/

# 修复权限
sudo chown -R www-data:www-data /var/www/pscg/dist
sudo chmod -R 755 /var/www/pscg/dist
```

### 10.2 定期更新
```bash
# 系统更新
sudo apt update
sudo apt upgrade -y

# Node.js 模块更新
cd /var/www/pscg
npm update

# PM2 更新
npm install -g pm2@latest
pm2 update
```

### 10.3 日志轮转
编辑 `/etc/logrotate.d/nginx`：
```nginx
/var/log/nginx/*.log {
    daily
    missingok
    rotate 14
    compress
    delaycompress
    notifempty
    create 0640 www-data adm
    sharedscripts
    postrotate
        /etc/init.d/nginx reload > /dev/null 2>&1 || true
    endscript
}
```

## 11. 性能优化

### 11.1 Nginx 优化
在 `/etc/nginx/nginx.conf` 中添加：
```nginx
worker_processes auto;
worker_rlimit_nofile 65535;

events {
    worker_connections 4096;
    multi_accept on;
    use epoll;
}

http {
    # 其他配置...
    client_max_body_size 100M;
    keepalive_timeout 65;
    keepalive_requests 1000;
}
```

### 11.2 Node.js 优化
在 `ecosystem.config.js` 中添加：
```javascript
module.exports = {
  apps: [{
    // ... 其他配置

  }]
};
```
