# 部署指南

## 环境要求

### 软件要求
- **操作系统**: Linux (Ubuntu 20.04+ 推荐) 或 Windows Server
- **Node.js**: 18.x 或更高版本（推荐 v20+）
- **MySQL**: 8.0 或更高版本
- **Nginx**: 1.18 或更高版本（生产环境）
- **PM2**: 5.0 或更高版本（生产环境）

## 开发环境部署

### 1. 安装依赖
```bash
npm install
```

### 2. 配置数据库
创建 `.env` 文件：
```
DB_HOST=localhost
DB_PORT=3306
DB_USER=PSCG
DB_PASSWORD=your_password
DB_NAME=pscg
SERVER_PORT=3001
```

**注意**: 
- 数据库用户名默认为 `PSCG`（根据 config/database.js 配置）
- 数据库名称默认为 `pscg`
- 需要配置 `SIGNATURE_SECRET` 用于 API 签名验证

### 4. 初始化数据库
```bash
# 方法 1: 手动导入 SQL 文件
mysql -u PSCG -p pscg < scripts/db-migration.sql

# 方法 2: 使用迁移脚本
mysql -u PSCG -p pscg < migrations/migrate_to_aittest.sql
```

**注意**: 数据库表结构会在后端服务首次启动时自动创建（通过 services/database.js 中的初始化逻辑）

### 5. 启动开发服务器
```bash
# 启动前端开发服务器（Vite，默认端口 5173）
npm run dev

# 启动后端服务器（Express，默认端口 3001）
npm run server
```

访问地址：
- 前端：http://localhost:5173
- 后端：http://localhost:3001

**注意**: 
- 前端开发服务器支持热重载
- 后端服务器会自动创建数据库表结构
- 首次启动时查看控制台日志确认数据库连接成功

## 生产环境部署

### 1. 服务器准备
```bash
# 更新系统
sudo apt update
sudo apt upgrade -y

# 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_16.x | sudo -E bash -
sudo apt install -y nodejs

# 安装 MySQL
sudo apt install -y mysql-server

# 安装 Nginx
sudo apt install -y nginx
```

### 2. 配置 MySQL
```bash
sudo mysql_secure_installation

# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE pscg DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 创建用户（使用 PSCG 大写）
CREATE USER 'PSCG'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON pscg.* TO 'PSCG'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 3. 部署项目
```bash
# 克隆项目
cd /var/www
git clone https://github.com/a1313445566/PSCG.git
cd PSCG

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
nano .env
```

**注意**: 环境变量配置应与 `.env.example` 保持一致，主要包括：
- `SERVER_PORT`: 后端服务器端口（默认 3001）
- `VITE_API_URL`: API 地址（生产环境使用 `/api`）
- `SIGNATURE_SECRET`: API 签名密钥（需替换为安全随机密钥）
- `DB_USER`: 数据库用户名（使用 `PSCG`）
- `DB_PASSWORD`: 数据库密码

### 4. 构建前端
```bash
npm run build
```

### 5. 配置 Nginx
创建配置文件：
```bash
sudo nano /etc/nginx/sites-available/pscg
```

配置内容：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/PSCG/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # 文件上传目录
    location /images {
        alias /var/www/PSCG/images;
    }

    location /audio {
        alias /var/www/PSCG/audio;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/pscg /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 9. 使用 PM2 启动后端
```bash
# 安装 PM2
sudo npm install -g pm2

# 启动服务
pm2 start server.cjs --name "pscg-app"

# 保存配置
pm2 save
pm2 startup

# 查看状态
pm2 status

# 查看日志
pm2 logs pscg-app
```

### 10. 配置 SSL（可选）
```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo systemctl enable certbot.timer
```

## 数据备份策略

### 数据库备份
```bash
# 创建备份脚本
nano /var/www/PSCG/scripts/backup.sh
```

脚本内容：
```bash
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/var/www/PSCG/backups"
mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u PSCG -p'your_password' pscg > "$BACKUP_DIR/pscg_$DATE.sql"

# 压缩备份
gzip "$BACKUP_DIR/pscg_$DATE.sql"

# 删除 7 天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete
```

设置定时任务：
```bash
crontab -e
# 添加：0 2 * * * /var/www/PSCG/scripts/backup.sh
```

## 常见问题与解决方案

### 1. 端口被占用

**Linux 系统**：
```bash
# 查看端口占用
sudo lsof -i :3001
# 或
sudo netstat -tulpn | grep 3001

# 结束进程
sudo kill -9 <PID>
```

**Windows 系统**：
```powershell
# 查看端口占用
netstat -ano | findstr :3001

# 结束进程
taskkill /PID <PID> /F
```

### 2. 数据库连接失败

**Linux 系统**：
- 检查数据库配置是否正确（DB_HOST, DB_USER, DB_PASSWORD, DB_NAME）
- 确认 MySQL 服务正在运行：`systemctl status mysql`
- 检查防火墙设置：`ufw status`
- 验证数据库用户权限：`SHOW GRANTS FOR 'PSCG'@'localhost';`

**Windows 系统**：
- 检查数据库配置是否正确
- 确认 MySQL 服务正在运行：`services.msc` 中查看 MySQL 服务
- 检查 Windows 防火墙设置
- 验证数据库用户权限

### 3. 文件上传失败

**Linux 系统**：
- 检查 upload 目录权限：`ls -ld images audio`
- 确认目录存在：`mkdir -p images audio`
- 检查 Nginx 配置中的 alias 路径是否正确
- 查看 Nginx 错误日志：`tail -f /var/log/nginx/error.log`

**Windows 系统**：
- 检查目录权限：`icacls images` 和 `icacls audio`
- 确认目录存在：`mkdir images` 和 `mkdir audio`
- 检查 Nginx 配置中的 alias 路径
- 查看 Nginx 错误日志

### 4. 前端 404 错误

**Linux 系统**：
- 确认 dist 目录存在：`ls -la dist/`
- 检查 Nginx 配置中的 root 路径是否正确
- 确认 try_files 配置：`try_files $uri $uri/ /index.html;`
- 清除浏览器缓存后重试

**Windows 系统**：
- 确认 dist 目录存在：`dir dist`
- 检查 Nginx 配置中的 root 路径
- 确认 try_files 配置
- 清除浏览器缓存
