## 三、源码部署步骤

### 3.1 克隆代码

```bash
# 克隆项目仓库
git clone <your-repository-url> /opt/PSCG
cd /opt/PSCG

# 切换到生产分支（如 main 或 production）
git checkout main

# 查看当前版本
git log -1 --oneline
```

### 3.2 安装依赖

```bash
cd /opt/PSCG

# 安装生产依赖（跳过 devDependencies）
npm install --production=false
# 注意：本项目需要 devDependencies 用于构建，所以用完整安装
npm install
```

### 3.3 构建前端

```bash
# 构建生产版本
npm run build

# 构建产物输出到 dist/ 目录
ls -la dist/
# 应包含 index.html, assets/ 目录等
```

### 3.4 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑配置
nano .env
```

**生产环境 `.env` 配置示例**：

```env
# ============================================
# 服务器配置
# ============================================
SERVER_PORT=3001

# ============================================
# API 地址（生产环境使用相对路径）
# 支持反向代理、Cloudflare Tunnel、HTTPS 自动适配
# ============================================
VITE_API_URL=/api

# ============================================
# 签名密钥（必须替换为强随机值！）
# 生成命令:
# node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
# ============================================
SIGNATURE_SECRET=<生成的128位十六进制随机密钥>
VITE_SIGNATURE_SECRET=<同上>

# ============================================
# 数据库配置
# ============================================
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=PSCG
DB_PASSWORD=<你的数据库密码>
DB_NAME=pscg
```

> **安全警告**：`SIGNATURE_SECRET` 必须是强随机值，泄露将导致 JWT Token 可被伪造。

### 3.5 数据库初始化

```bash
# 创建数据库（如尚未创建）
mysql -u root -p -e "
CREATE DATABASE IF NOT EXISTS pscg DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER IF NOT EXISTS 'PSCG'@'localhost' IDENTIFIED BY '你的密码';
GRANT ALL PRIVILEGES ON pscg.* TO 'PSCG'@'localhost';
FLUSH PRIVILEGES;
"

# 执行迁移脚本
mysql -u PSCG -p pscg < migrations/migrate_to_aittest.sql

# 执行额外迁移（按需）
mysql -u PSCG -p pscg < scripts/migrate-add-judgment-type.sql
```

### 3.6 启动服务

#### 方式一：直接启动（测试用）

```bash
cd /opt/PSCG
npm run server
```

服务启动后访问 `http://<服务器IP>:3001/api/health` 验证。

#### 方式二：PM2 进程管理（推荐）

```bash
# 安装 PM2（全局）
npm install -g pm2

# 启动应用
pm2 start server.cjs --name "pscg-api"

# 设置开机自启
pm2 startup
pm2 save

# 常用 PM2 命令
pm2 list              # 查看进程列表
pm2 logs pscg-api     # 查看日志
pm2 restart pscg-api  # 重启
pm2 stop pscg-api     # 停止
pm2 monit             # 监控面板
```

#### 方式三：Systemd 服务

创建 `/etc/systemd/system/pscg.service`：

```ini
[Unit]
Description=PSCG Learning System API Server
After=network.target mysql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/PSCG
ExecStart=/usr/bin/node server.cjs
Restart=always
RestartSec=10
Environment=NODE_ENV=production
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

```bash
# 启用并启动服务
systemctl daemon-reload
systemctl enable pscg
systemctl start pscg

# 查看状态
systemctl status pscg

# 查看日志
journalctl -u pscg -f
```

## 四、Nginx 反向代理配置

### 4.1 基础配置

编辑 `/etc/nginx/sites-available/pscg`：

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名或 IP

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml image/svg+xml;

    # 前端静态文件
    location / {
        root /opt/PSCG/dist;
        index index.html;
        try_files $uri $uri/ /index.html;  # Vue Router Hash 模式支持

        # 静态资源缓存
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 30d;
            add_header Cache-Control "public, immutable";
        }
    }

    # API 反向代理到 Node.js
    location /api/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;

        # WebSocket 支持（如需要）
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';

        # 代理头设置
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 图片资源代理
    location /images/ {
        proxy_pass http://127.0.0.1:3001;
        expires 7d;
    }

    # 音频资源代理
    location /audio/ {
        proxy_pass http://127.0.0.1:3001;
        expires 7d;
    }
}
```

### 4.2 启用站点

```bash
# 创建软链接启用站点
ln -s /etc/nginx/sites-available/pscg /etc/nginx/sites-enabled/

# 测试配置语法
nginx -t

# 重载 Nginx
nginx -s reload
```

### 4.3 SSL / HTTPS 配置（Let's Encrypt）

```bash
# 安装 Certbot
apt install certbot python3-certbot-nginx  # Debian/Ubuntu

# 获取 SSL 证书（自动配置 Nginx）
certbot --nginx -d your-domain.com

# 自动续期（Certbot 通常已配置 cron）
certbot renew --dry-run  # 测试续期
```

SSL 配置会自动添加到 Nginx 配置中。完整的 HTTPS 配置示例：

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    # SSL 优化参数
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # ... 其余配置与 HTTP 相同 ...
}

# HTTP → HTTPS 重定向
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

## 五、Caddy 反向代理配置（替代方案）

Caddy 配置更简洁，自动处理 SSL：

编辑 `/etc/caddy/Caddyfile`：

```
your-domain.com {
    # 自动 HTTPS（Let's Encrypt）
    tls your-email@example.com

    # Gzip 压缩
    encode gzip

    # 前端静态文件
    root * /opt/PSCG/dist
    try_files {path} /index.html

    # 静态资源缓存
    @static path *.js *.css *.png *.jpg *.svg *.woff2
    header @static Cache-Control "public, immutable, max-age=2592000"

    # API 反向代理
    handle_path /api/* {
        reverse_proxy localhost:3001
    }

    handle_path /images/* {
        reverse_proxy localhost:3001
    }

    handle_path /audio/* {
        reverse_proxy localhost:3001
    }

    # 安全头
    header {
        X-Frame-Options "SAMEORIGIN"
        X-Content-Type-Options "nosniff"
        -Server
    }
}
```

启动 Caddy：

```bash
caddy validate --config /etc/caddy/Caddyfile
systemctl restart caddy
```

## 六、PM2 集群模式配置（高可用）

对于生产环境，建议使用 PM2 集群模式充分利用多核 CPU：

**ecosystem.config.js**：

```javascript
module.exports = {
  apps: [
    {
      name: 'pscg-api',
      script: 'server.cjs',
      instances: 'max',         // 根据 CPU 核心数自动决定实例数
      exec_mode: 'cluster',     // 集群模式
      cwd: '/opt/PSCG',
      env: {
        NODE_ENV: 'production',
        SERVER_PORT: 3001
      },
      // 日志配置
      error_file: '/var/log/pscg/error.log',
      out_file: '/var/log/pscg/out.log',
      log_date_format: 'YYYY-MM-DD HH:mm:ss Z',

      // 内存限制（超过后重启）
      max_memory_restart: '500M',

      // 心跳检测
      autorestart: true,
      watch: false,
      max_restarts: 10,

      // 优雅关闭
      kill_timeout: 5000,
      wait_ready: true,
      listen_timeout: 10000
    }
  ]
}
```

启动集群：

```bash
pm2 start ecosystem.config.js
pm2 save
```

## 七、备份策略

### 7.1 数据库备份

创建 `scripts/backup-db.sh`：

```bash
#!/bin/bash
# PSCG 数据库自动备份脚本

BACKUP_DIR="/opt/backups/pscg"
DATE=$(date +%Y%m%d_%H%M%S)
RETENTION_DAYS=30

# 创建备份目录
mkdir -p "$BACKUP_DIR"

# 执行备份（压缩）
mysqldump -u PSCG -p'你的密码' --single-transaction --routines --triggers pscg \
  | gzip > "$BACKUP_DIR/pscg_$DATE.sql.gz"

# 删除过期备份
find "$BACKUPDIR" -name "*.sql.gz" -mtime +$RETENTION_DAYS -delete

echo "[$(date)] Backup completed: pscg_$DATE.sql.gz"
```

添加 Cron 定时任务（每天凌晨 2 点）：

```bash
crontab -e

# 添加以下行：
0 2 * * * /opt/PSCG/scripts/backup-db.sh >> /var/log/pscg-backup.log 2>&1
```

### 7.2 文件备份

```bash
#!/bin/bash
# 备份上传的图片和音频文件
BACKUP_DIR="/opt/backups/pscg"
DATE=$(date +%Y%m%d_%H%M%S)

tar -czf "$BACKUP_DIR/uploads_$DATE.tar.gz" \
  /opt/PSCG/images/ \
  /opt/PSCG/audio/
```

### 7.3 一键备份恢复

```bash
# 恢复数据库
gunzip < pscg_20260405_020000.sql.gz | mysql -u PSCG -p pscg

# 恢复上传文件
tar -xzf uploads_20260405_020000.tar.gz -C /
```

## 八、监控与日志

### 8.1 日志位置

| 日志类型 | 路径 | 说明 |
|----------|------|------|
| 应用日志 | PM2: `~/.pm2/logs/` 或 Systemd: `journalctl -u pscg` | Express 应用日志 |
| Nginx 日志 | `/var/log/nginx/access.log`, `/var/log/nginx/error.log` | 访问/错误日志 |
| MySQL 日志 | `/var/log/mysql/error.log` | 数据库错误 |
| 备份日志 | `/var/log/pscg-backup.log` | 备份执行记录 |

### 8.2 健康检查

系统提供内置健康检查接口：

```bash
# 检查 API 服务是否正常
curl -f http://localhost:3001/api/health || echo "API 服务异常"

# 检查外部访问是否正常
curl -f https://your-domain.com/api/health || echo "外部访问异常"
```

### 8.3 监控脚本示例

```bash
#!/bin/bash
# 简单的服务监控脚本
HEALTH_CHECK=$(curl -sf http://localhost:3001/api/health)
if [ $? -ne 0 ]; then
    echo "[ALERT] $(date): PSCG API is down! Restarting..."
    pm2 restart pscg-api || systemctl restart pscg
fi
```

## 九、维护操作

### 9.1 更新部署

```bash
cd /opt/PSCG

# 1. 拉取最新代码
git pull origin main

# 2. 安装新依赖（如有变化）
npm install

# 3. 重新构建前端

# 4. 重启服务
pm2 restart pscg-api

# 5. 重载 Nginx（通常不需要）
```

### 9.2 回滚操作

```bash
cd /opt/PSCG

# 查看历史提交
git log --oneline -10

# 回滚到指定版本
git checkout <commit-hash>

# 重新构建
npm run build

# 重启服务
pm2 restart pscg-api
```

### 9.3 数据库迁移

当有新的数据库变更时：

```bash
# 先备份数据库
mysqldump -u PSCG -p pscg > backup_before_migration.sql

# 执行新迁移脚本
mysql -u PSCG -p pscg < migrations/new_migration.sql

mysql -u PSCG -p pscg -e "SELECT COUNT(*) FROM questions;"
```

## 十、安全加固清单

- [ ] `SIGNATURE_SECRET` 已替换为强随机值
