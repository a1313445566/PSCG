## 一、服务器初始化

### 1.1 基础安全配置（以 Ubuntu 为例）

```bash
# ===== 1. 系统更新 =====
sudo apt update && sudo apt upgrade -y

# ===== 2. 创建专用运行用户 =====
sudo useradd -m -s /bin/bash pscg
sudo usermod -aG sudo pscg   # 如需 sudo 权限

# ===== 3. SSH 安全加固 =====
sudo nano /etc/ssh/sshd_config
```

修改以下配置项：

```
PermitRootLogin no                     # 禁止 root 登录
PasswordAuthentication no              # 禁止密码登录（仅允许密钥）
PubkeyAuthentication yes               # 允许公钥认证
Port 22                                # 可改为非常见端口（如 2222）
MaxAuthTries 3                         # 最大尝试次数
LoginGraceTime 30                      # 登录超时秒数
```

```bash
# 重启 SSH 服务
sudo systemctl restart sshd

# 上传本地公钥到服务器
ssh-copy-id pscg@your-server-ip
```

### 1.2 防火墙配置

```bash
# 安装 UFW 防火墙
sudo apt install ufw -y

# 默认策略
sudo ufw default deny incoming
sudo ufw default allow outgoing

# 开放必要端口
sudo ufw allow 22/tcp          # SSH
sudo ufw allow 80/tcp          # HTTP
sudo ufw allow 443/tcp         # HTTPS

# 启用防火墙
sudo ufw enable

# 查看状态
sudo ufw status verbose
```

### 1.3 Fail2Ban 防暴力破解

```bash
sudo apt install fail2ban -y

# 创建本地配置
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local

sudo nano /etc/fail2ban/jail.local
```

配置内容：

```ini
[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 3
bantime = 3600
findtime = 600

[nginx-http-auth]
enabled = true
filter = nginx-http-auth
port = http,https
logpath = /var/log/nginx/error.log
maxretry = 3
bantime = 600
```

```bash
sudo systemctl enable fail2ban
sudo systemctl start fail2ban
```

## 二、Node.js 生产配置

### 2.1 PM2 集群模式部署

**安装 PM2**：

```bash
sudo npm install -g pm2
```

**创建生态系统配置文件** `ecosystem.config.cjs`：

```javascript
module.exports = {
  apps: [
    {
      name: 'pscg-api',
      script: 'server.cjs',
      cwd: '/opt/PSCG',

      // 集群模式：利用多核 CPU
      instances: 'max',
      exec_mode: 'cluster',

      // 环境变量
      env: {
        NODE_ENV: 'production',
        SERVER_PORT: 3001
      },

      // 内存限制：超过 500MB 自动重启
      max_memory_restart: '500M',

      // 日志配置
      error_file: '/var/log/pscg/error.log',
      out_file: '/var/log/pscg/out.log',
      log_date_format: 'YYYY-MM-DD HH:mm:ss Z',
      merge_logs: true,

      // 高可用配置
      autorestart: true,
      watch: false,
      max_restarts: 10,
      min_uptime: '10s',

      // 优雅关闭
      kill_timeout: 5000,
      wait_ready: true,
      listen_timeout: 10000,

      // 心跳检测
      health_check: true,
      health_check_grace_period: 5000
    }
  ]
}
```

**启动与管理**：

```bash
cd /opt/PSCG

# 启动集群
pm2 start ecosystem.config.cjs

# 保存进程列表（开机自启）
pm2 save

# 设置 PM2 开机自启
pm2 startup systemd -u pscg --hp /home/pscg
# 执行输出的命令...

# 常用管理命令
pm2 list                          # 查看所有进程
pm2 logs pscg-api --lines 100     # 查看最近 100 行日志
pm2 monit                         # 实时监控面板
pm2 restart pscg-api              # 平滑重启（依次重启每个 worker）
pm2 reload pscg-api               # 零停机重载
pm2 stop pscg-api                 # 停止服务
pm2 delete pscg-api               # 移除进程
pm2 info pscg-api                 # 详细信息
```

### 2.2 PM2 日志轮转

安装日志轮转插件：

```bash
pm2 install pm2-logrotate
```

配置（默认已足够好，也可自定义）：

```bash
pm2 set pm2-logrotate:max_size 10M      # 单个日志最大 10MB
pm2 set pm2-logrotate:retain 30         # 保留 30 个日志文件
pm2 set pm2-logrotate:compress true     # 压缩旧日志
pm2 set pm2-logrotate:interval "0 0 * * *"  # 每天午夜轮转
```

### 2.3 Systemd 方案（替代 PM2）

如果不想使用 PM2，可以用 Systemd：

**创建服务文件** `/etc/systemd/system/pscg.service`：

```ini
[Unit]
Description=PSCG Learning System API
Documentation=https://github.com/your-repo/pscg
After=network.target mysql.service
Wants=mysql.service

[Service]
Type=forking
User=pscg
Group=pscg
WorkingDirectory=/opt/PSCG
ExecStart=/usr/bin/pm2 start ecosystem.config.cjs
ExecStop=/usr/bin/pm2 stop ecosystem.config.cjs
ExecReload=/usr/bin/pm2 reload ecosystem.config.css
PIDFile=/home/pscg/.pm2/pm2.pid
Restart=on-failure
RestartSec=10
LimitNOFILE=65535
LimitNPROC=4096

# 环境文件
EnvironmentFile=-/opt/PSCG/.env

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable pscg
sudo systemctl start pscg
```

## 三、MySQL 生产配置

### 3.1 性能优化配置

编辑 `/etc/mysql/mysql.conf.d/mysqld.cnf`：

```ini
[mysqld]

# ===== 基础配置 =====
user                    = mysql
pid-file                = /var/run/mysqld/mysqld.pid
socket                  = /var/run/mysqld/mysqld.sock
port                    = 3306
basedir                 = /usr
datadir                 = /var/lib/mysql

# ===== 连接配置 =====
max_connections         = 200           # 最大连接数
max_connect_errors      = 100
wait_timeout            = 600           # 连接空闲超时（秒）
interactive_timeout     = 600

# ===== InnoDB 缓冲池（关键性能参数）=====
# 设为物理内存的 50%-70%（假设 4GB 内存）
innodb_buffer_pool_size = 2G
innodb_buffer_pool_instances = 4       # 拆分为多个实例（<= CPU核心数）
innodb_log_file_size     = 256M        # redo log 大小
innodb_log_buffer_size   = 64M
innodb_flush_log_at_trx_commit = 2     # 性能与安全的平衡（1=最安全，2=折中）
innodb_flush_method      = O_DIRECT     # 绕过 OS 缓存

# ===== 查询缓存（MySQL 8.0 已移除，忽略）=====

# ===== 临时表 =====
tmp_table_size          = 64M
max_heap_table_size     = 64M

# ===== 慢查询日志（重要！）=====
slow_query_log          = 1
slow_query_log_file     = /var/log/mysql/slow-query.log
long_query_time         = 2             # 超过 2 秒记录为慢查询
log_queries_not_using_indexes = 1       # 记录未使用索引的查询

# ===== 二进制日志（用于备份恢复/主从复制）=====
log_bin                 = /var/log/mysql/mysql-bin.log
binlog_format           = ROW
expire_logs_days        = 7
max_binlog_size         = 200M

# ===== 字符集 =====
character-set-server    = utf8mb4
collation-server        = utf8mb4_unicode_ci

# ===== 安全 =====
skip-name-resolve                       # 禁止 DNS 解析（提升连接速度）
bind-address            = 127.0.0.1     # 仅本地访问

[client]
default-character-set   = utf8mb4

[mysql]
default-character-set   = utf8mb4
```

## 四、Nginx 生产调优

### 4.1 完整生产配置

`/etc/nginx/sites-available/pscg`：

```nginx
# ===== 全局配置优化（/etc/nginx/nginx.conf）=====
# worker_processes auto;              # 自动匹配 CPU 核心数
# worker_rlimit_nofile 65535;         # 文件描述符上限
# worker_connections 4096;            # 每个 worker 的最大连接数
# use epoll;                          # 使用 epoll 事件模型（Linux）
# multi_accept on;                    # 一次接受多个连接

# ===== PSCG 站点配置 =====
upstream pscg_backend {
    server 127.0.0.1:3001;
    keepalive 32;                     # 保持长连接
}

# 速率限制定义（全局）
limit_req_zone $binary_remote_addr zone=api_limit:10m rate=30r/s;
limit_conn_zone $binary_remote_addr zone=conn_limit:10m;

server {
    listen 80;
    server_name your-domain.com www.your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com www.your-domain.com;

    # ========== SSL 配置 ==========
    ssl_certificate     /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;
    ssl_session_cache   shared:SSL:10m;
    ssl_session_timeout 1d;
    ssl_session_tickets off;
    ssl_stapling on;
    ssl_stapling_verify on;

    # ========== 安全头 ==========
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline'; img-src 'self' data: blob:; font-src 'self'; connect-src 'self'; media-src 'self'" always;

    # ========== Gzip 压缩 ==========
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_min_length 1024;
    gzip_types
        text/plain
        text/css
        text/xml
        text/javascript
        application/json
        application/javascript
        application/xml+rss
        application/atom+xml
        image/svg+xml;

    # ========== 访问日志格式 ==========
    access_log /var/log/nginx/pscg_access.log combined buffer=32k flush=5m;
    error_log  /var/log/nginx/pscg_error.log warn;

    # ========== 前端静态资源 ==========
    root /opt/PSCG/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;

        # HTML 不缓存
        location ~* \.html$ {
            add_header Cache-Control "no-cache, no-store, must-revalidate";
            add_header Pragma "no-cache";
            add_header Expires "0";
        }

        # 静态资源长期缓存（带 hash 的文件名不会变）
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 365d;
            add_header Cache-Control "public, immutable";
            access_log off;
        }
    }

    # ========== API 反向代理 ==========
    location /api/ {
        limit_req zone=api_limit burst=50 nodelay;
        limit_conn conn_limit 20;

        proxy_pass http://pscg_backend;
        proxy_http_version 1.1;

        # WebSocket 支持
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection '';

        # 传递真实 IP
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时配置
        proxy_connect_timeout 10s;
        proxy_send_timeout 30s;
        proxy_read_timeout 60s;

        # 缓冲配置
        proxy_buffering on;
        proxy_buffer_size 4k;
        proxy_buffers 8 4k;

        # 禁止对 API 响应进行缓存
        add_header Cache-Control "no-store";
        add_header X-No-Cache "1";

        # 错误页面
        proxy_intercept_errors off;
    }

    # ========== 图片资源 ==========
    location /images/ {
        proxy_pass http://pscg_backend;
        proxy_set_header Host $host;
        expires 7d;
        add_header Cache-Control "public";
        access_log off;
    }

    # ========== 音频资源 ==========
    location /audio/ {
        proxy_pass http://pscg_backend;
        proxy_set_header Host $host;
        expires 7d;
        add_header Cache-Control "public";
        access_log off;
    }

    # ========== 禁止访问敏感文件 ==========
    location ~ /\. {
        deny all;
        access_log off;
        log_not_found off;
    }

    location ~* \.(env|git|sql|bak|conf|md)$ {
        deny all;
        access_log off;
        log_not_found off;
    }

    # ========== 健康检查（不限流）==========
    location /api/health {
        proxy_pass http://pscg_backend;
        access_log off;
    }
}
```

### 4.2 Nginx 全局优化

编辑 `/etc/nginx/nginx.conf`：

```nginx
user www-data;
worker_processes auto;
worker_rlimit_nofile 65535;
pid /run/nginx.pid;

events {
    worker_connections 4096;
    use epoll;
    multi_accept on;
}

http {
    # 基础设置
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    types_hash_max_size 2048;
    keepalive_timeout 65;
    keepalive_requests 1000;

    # 隐藏版本号
    server_tokens off;

    # MIME 类型
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    # 日志格式
    log_format main '$remote_addr - $remote_user [$time_local] '
                    '"$request" $status $body_bytes_sent '
                    '"$http_referer" "$http_user_agent" '
                    '$request_time $upstream_response_time';

    # 包含站点配置
    include /etc/nginx/sites-enabled/*;
}
```

### 4.3 性能调优参数说明

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `worker_processes` | Worker 进程数 | `auto`（等于 CPU 核心数） |
| `worker_connections` | 每个 Worker 最大连接数 | `4096` |
| `keepalive` | 后端长连接池 | `32` |
| `gzip_comp_level` | 压缩级别（1-9） | `6`（平衡CPU与带宽） |
| `proxy_buffer_size` | 代理缓冲大小 | `4k`-`16k` |
| `limit_req` | API 限流 | `30r/s` burst `50` |

### 5.3 自动备份方案

创建 `/opt/PSCG/scripts/auto-backup.sh`：

```bash
#!/bin/bash
# ============================================================
# PSCG 自动备份脚本 - 生产版
# 功能：数据库全量备份 + 上传文件备份 + 日志清理
# ============================================================

set -euo pipefail

# ===== 配置 =====
BACKUP_BASE="/opt/backups/pscg"
DATE=$(date +%Y%m%d_%H%M%S)
DB_USER="PSCG"
DB_PASS="<你的密码>"
DB_NAME="pscg"
RETENTION_DAYS=30
PROJECT_DIR="/opt/PSCG"

# 创建备份目录
mkdir -p "$BACKUP_BASE/database"
mkdir -p "$BACKUP_BASE/uploads"
mkdir -p "$BACKUP_BASE/logs"

echo "[$(date '+%Y-%m-%d %H:%M:%S')] 开始备份..."

# ===== 1. 数据库备份 =====
echo "  [1/3] 备份数据库..."
mysqldump -u"$DB_USER" -p"$DB_PASS" \
  --single-transaction \
  --routines \
  --triggers \
  --events \
  "$DB_NAME" \
  | gzip > "$BACKUP_BASE/database/pscg_db_${DATE}.sql.gz"

DB_SIZE=$(du -h "$BACKUP_BASE/database/pscg_db_${DATE}.sql.gz" | cut -f1)
echo "  数据库备份完成: $DB_SIZE"

# ===== 2. 上传文件备份 =====
echo "  [2/3] 备份上传文件..."
tar -czf "$BACKUP_BASE/uploads/uploads_${DATE}.tar.gz" \
  -C "$PROJECT_DIR" \
  images/ audio/ 2>/dev/null || true

UPLOAD_SIZE=$(du -h "$BACKUP_BASE/uploads/uploads_${DATE}.tar.gz" | cut -f1)
echo "  上传文件备份完成: $UPLOAD_SIZE"

# ===== 3. 清理过期备份 =====
echo "  [3/3] 清理过期备份..."
find "$BACKUP_BASE" -name "*.sql.gz" -mtime +$RETENTION_DAYS -delete
find "$BACKUP_BASE" -name "*.tar.gz" -mtime +$RETENTION_DAYS -delete

EXPIRED_COUNT=$(find "$BACKUP_BASE" -mtime +$RETENTION_DAYS | wc -l)
echo "  清理了 $EXPIRED_COUNT 个过期文件"

# ===== 完成 =====
TOTAL_SIZE=$(du -sh "$BACKUP_BASE" | cut -f1)
echo "[$(date '+%Y-%m-%d %H:%M:%S')] 备份完成！总大小: $TOTAL_SIZE"
```

**设置定时任务**：

```bash
# 赋予执行权限

# 添加 Cron 任务（每天凌晨 2 点整点备份，凌晨 3 点增量备份）

```

添加内容：

```cron

0 3 * * 0 /opt/PSCG/scripts/auto-backup.sh >> /var/log/pscg-backup.log 2>&1  # 周日额外备份
```

### 6.2 性能指标参考值

### 6.3 性能瓶颈定位

```bash
# 1. 检查 CPU 使用情况
```

### 7.1 垂直扩容（升级硬件）

### 7.2 水平扩容（多节点）

架构演进：

```
                    ┌──────────────┐
                    │   负载均衡    │
                    │  (Nginx/LVS)  │
                    └──────┬───────┘
                      ┌─────┼─────┐
                      ▼     ▼     ▼
                  ┌──────┐┌──────┐┌──────┐
                  │Node 1││Node 2││Node 3│
                  │:3001 ││:3001 ││:3001 │
                  └──┬───┘└──┬───┘└──┬───┘
                     └──────┼──────┘
                            ▼
                    ┌──────────────┐
                    │  MySQL 主从   │
                    │ (或 ProxySQL) │
                    └──────────────┘
```

Nginx 负载均衡配置：

```nginx
upstream pscg_cluster {
    server 192.168.1.10:3001 weight=3;  # 主节点，权重更高
    server 192.168.1.11:3001 weight=2;
    server 192.168.1.12:3001 weight=1 backup;  # 备用节点

    # 长连接
    keepalive 32;
}

location /api/ {
    proxy_pass http://pscg_cluster;
    # ... 其他配置同上
}
```

### 7.3 数据库扩容

- **读写分离**：主库写入，从库读取
- **分库分表**：按学科/年级拆分
- **引入 Redis**：缓存热点数据（排行榜、学科列表等）

## 八、故障排查手册

### 8.1 常见故障及解决方案

| 故障现象 | 可能原因 | 排查命令 | 解决方法 |
|----------|----------|----------|----------|
| 502 Bad Gateway | Node 进程崩溃 | `pm2 logs` | `pm2 restart pscg-api` |
| 504 Gateway Timeout | API 响应过慢 | 检查慢查询日志 | 优化 SQL / 增加超时时间 |
| 数据库连接拒绝 | 连接池耗尽 | `show processlist` | 调整 `max_connections` / 检查泄漏 |
| 内存溢出 OOM | 内存泄漏 | `pm2 monit` / `dmesg` | `max_memory_restart` / 排查泄漏点 |
| 磁盘满 | 日志/备份过多 | `df -h` | 清理日志 / 调整备份保留天数 |
| SSL 证书过期 | Let's Encrypt 续期失败 | `certbot certificates` | 手动续期 `certbot renew` |
| CPU 100% | 死循环/密集计算 | `top` / `pm2 monit` | 排查具体代码 |

### 8.2 故障排查流程图

```
用户报告无法访问
    │
    ▼
检查 Nginx 状态: systemctl status nginx
    ├── 异常 → 重启 Nginx: systemctl restart nginx
    └── 正常 ↓
            检查 PM2 状态: pm2 list
            ├── 异常 → 重启应用: pm2 restart pscg-api
            └── 正常 ↓
                    检查 MySQL: systemctl status mysql
                    ├── 异常 → 重启 MySQL + 检查磁盘空间
                    └── 正常 ↓
                            检查应用日志: pm2 logs pscg-api --lines 50
                            → 根据错误信息定位问题
```

### 8.3 紧急恢复流程

```bash
# 1. 一键恢复脚本
/opt/PSCG/scripts/emergency-restore.sh

# 2. 手动恢复步骤
pm2 restart all                    # 重启所有服务
systemctl restart nginx            # 重启 Nginx
systemctl restart mysql            # 重启 MySQL

# 3. 如需回滚代码
cd /opt/PSCG
git log --oneline -5               # 查看最近的提交
git checkout <上一个稳定的commit-hash>
npm run build
pm2 restart pscg-api

# 4. 如需恢复数据库
gunzip < /opt/backups/pscg/database/pscg_db_最新.sql.gz | mysql -u PSCG -p pscg
```
