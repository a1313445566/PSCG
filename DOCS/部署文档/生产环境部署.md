# 生产环境部署

## 系统要求

### 硬件要求
- **CPU**: 至少 4 核处理器
- **内存**: 至少 8GB RAM
- **存储空间**: 至少 20GB 可用空间
- **带宽**: 至少 1Mbps 上传/下载速度

### 软件要求
- **操作系统**: Ubuntu 20.04 LTS 或更高版本
- **Node.js**: 18.x 或更高版本（推荐 v20+）
- **npm**: 8.0 或更高版本
- **MySQL**: 8.0 或更高版本
- **PM2**: 5.0 或更高版本
- **Nginx**: 1.18 或更高版本
- **Git**: 2.0 或更高版本

## 数据库配置

### 1. 启动 MySQL 服务
```bash
sudo systemctl start mysql
sudo systemctl enable mysql
```

### 2. 安全配置
```bash
sudo mysql_secure_installation
```

### 3. 创建数据库和用户
```bash
sudo mysql -u root -p
```

```sql
CREATE DATABASE pscg DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'PSCG'@'localhost' IDENTIFIED BY 'your_strong_password';
GRANT ALL PRIVILEGES ON pscg.* TO 'PSCG'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

**注意**：数据库用户名应使用 `PSCG`（大写），与 `config/database.js` 配置保持一致

### 3. 配置环境变量

```
SERVER_PORT=3001

# API 地址配置（生产环境使用 /api）

# 签名密钥（请替换为实际密钥）

VITE_SIGNATURE_SECRET=your-secret-key-here

DB_HOST=localhost
DB_PORT=3306
DB_USER=PSCG
DB_PASSWORD=your_strong_password
DB_NAME=pscg
```

**注意**：
- `SIGNATURE_SECRET` 用于 API 签名验证，请替换为安全的随机密钥
- 数据库用户名使用 `PSCG`（大写）

## Nginx 配置

### 1. 创建 Nginx 配置文件
```bash
sudo nano /etc/nginx/sites-available/pscg
```

### 2. 配置 Nginx
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/PSCG/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # 文件上传目录
    location /images {
        alias /var/www/PSCG/images;
    }

    location /audio {
        alias /var/www/PSCG/audio;
    }
}
```

### 3. 启用配置
```bash
sudo ln -s /etc/nginx/sites-available/pscg /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## 系统服务配置

### 1. 创建 Systemd 服务文件
```bash
sudo nano /etc/systemd/system/pscg.service
```

### 2. 配置服务
```ini
[Unit]
Description=PSCG Smart Quiz System
After=network.target mysql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/PSCG
ExecStart=/usr/local/bin/node server.cjs
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

### 3. 启用服务
```bash
sudo systemctl daemon-reload
sudo systemctl enable pscg.service
sudo systemctl start pscg.service
```

## 备份策略

### 1. 数据库备份
```bash
# 创建备份目录
mkdir -p /var/www/PSCG/backups

# 定时备份脚本
nano /var/www/PSCG/scripts/backup.sh
```

```bash
#!/bin/bash
DATE=$(date +%Y-%m-%d_%H-%M-%S)
BACKUP_DIR="/var/www/PSCG/backups"

# 备份数据库
mysqldump -u pscg -p'your_strong_password' pscg > "$BACKUP_DIR/pscg_$DATE.sql"

# 压缩备份文件
gzip "$BACKUP_DIR/pscg_$DATE.sql"

# 删除7天前的备份
find "$BACKUP_DIR" -name "*.sql.gz" -mtime +7 -delete
```

### 2. 设置定时任务
```bash
crontab -e
```

添加以下内容：
```
# 每天凌晨2点备份数据库
0 2 * * * /var/www/PSCG/scripts/backup.sh
```

## 常见问题

### 1. 服务启动失败
- **问题**: 服务无法启动
- **解决方案**: 查看日志，检查配置文件和依赖

### 2. 数据库连接失败
- **问题**: 无法连接到数据库
- **解决方案**: 检查数据库配置，确保 MySQL 服务正在运行

### 3. 前端页面无法访问
- **问题**: 前端页面显示 404
- **解决方案**: 检查 Nginx 配置，确保前端构建文件存在

### 4. API 接口无法访问
- **问题**: API 接口返回 502 错误
- **解决方案**: 检查后端服务是否正在运行，端口是否正确

## 部署流程

1. **准备服务器**: 配置服务器环境，安装必要软件
2. **数据库配置**: 创建数据库和用户，初始化数据
3. **项目部署**: 克隆代码，安装依赖，构建前端
4. **服务配置**: 配置 Nginx，启动服务
5. **SSL 配置**: 获取 SSL 证书，启用 HTTPS
6. **监控配置**: 配置服务监控，设置备份策略
7. **安全加固**: 配置防火墙，禁用 root 登录

## 注意事项

- **环境变量**: 生产环境的环境变量应使用强密码和安全的密钥
- **备份**: 定期备份数据库和重要文件
- **监控**: 监控服务状态，及时发现和解决问题
- **安全**: 定期更新系统和依赖，防止安全漏洞
- **性能**: 根据实际负载调整服务器配置
