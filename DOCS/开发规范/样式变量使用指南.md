### 2.1 间距变量

| 变量名 | 值 | 使用场景 |
|--------|-----|----------|
| `$spacing-2xs` | `2px` | 极小间距 |
| `$spacing-xs` | `4px` | 小间距 |
| `$spacing-sm` | `8px` | 中小间距 |
| `$spacing-md` | `16px` | 标准间距 |
| `$spacing-lg` | `24px` | 大间距 |
| `$spacing-xl` | `32px` | 超大间距 |
| `$spacing-2xl` | `48px` | 区块间距 |
| `$spacing-3xl` | `64px` | 章节间距 |
| `$spacing-compact` | `12px` | 高频中间值 |
| `$spacing-comfortable` | `20px` | 舒适间距 |
| `$spacing-section` | `40px` | 区块容器 |

**使用示例**：

```scss
// ❌ 错误：硬编码间距
.card {
  padding: 20px;
  margin-bottom: 16px;
}

// ✅ 正确：使用间距变量
.card {
  padding: $spacing-comfortable; // 20px
  margin-bottom: $spacing-md;    // 16px
}
```

  }
}

## 一、快速开始

### 1.1 无需手动导入

配置 Vite 全局注入后，所有 `.vue` 文件的 `<style lang="scss">` 块可直接使用变量：

```vue
<style scoped lang="scss">
.my-button {
  color: $primary-color;  // ✅ 直接使用，无需 @use
  background: $bg-gradient-primary;
}
</style>
```

### 1.2 变量命名规范

| 变量类型 | 命名规则 | 示例 |
|---------|---------|------|
| 颜色变量 | `$xxx-color` | `$primary-color`, `$secondary-color` |
| 背景变量 | `$bg-xxx` | `$bg-gradient-page`, `$card-background` |
| 阴影变量 | `$shadow-xxx` | `$shadow-sm`, `$shadow-btn-primary` |
| 文字变量 | `$text-xxx` | `$text-primary`, `$text-shadow-base` |
| Admin变量 | `$admin-xxx` | `$admin-bg`, `$admin-text-primary` |

## 七、工具函数

### 7.1 颜色调整函数（v2.0 更新）

> ⚠️ **重要**：`lighten()` 和 `darken()` 已被 Sass 官方弃用，请使用 `color.adjust()` 替代。

#### 7.1.1 color.adjust() 函数

**模块导入**（必需）：

```scss
@use 'sass:color';
```

**语法**：

```scss
// 增加亮度
color.adjust($color, $lightness: x%);

// 降低亮度
color.adjust($color, $lightness: -x%);

// 调整饱和度
color.adjust($color, $saturation: x%);

// 调整透明度
color.adjust($color, $alpha: x%);
```

**使用示例**：

```scss
@use 'sass:color';

// ❌ 旧语法（已弃用，Dart Sass 3.0 将移除）
.card {
  background: lighten($bg-base, 2%);
  border-color: darken($border-color, 10%);
}

// ✅ 新语法（推荐）
.card {
  background: color.adjust($bg-base, $lightness: 2%);
  border-color: color.adjust($border-color, $lightness: -10%);
}
```

**替换速查表**：

| 旧语法 | 新语法 | 效果 |
|--------|--------|------|
| `lighten($color, x%)` | `color.adjust($color, $lightness: x%)` | 加亮颜色 |
| `darken($color, x%)` | `color.adjust($color, $lightness: -x%)` | 加深颜色 |

### 7.2 颜色混合函数

| 函数名 | 用途 | 示例 |
|--------|------|------|
| `shade($color, $amount)` | 加深颜色（混黑） | `shade($primary-color, 15%)` → `#e85555` |
| `tint($color, $amount)` | 加亮颜色（混白） | `tint($primary-color, 12%)` → `#ff8787` |
| `set-alpha($color, $opacity)` | 设置透明度 | `set-alpha($primary-color, 30)` → `rgba(255, 107, 107, 0.3)` |
| `gradient($from, $to)` | 生成线性渐变 | `gradient($primary-color, $secondary-color)` |

**使用示例**：

```scss
// ❌ 错误：硬编码rgba
.overlay {
  background: rgba(255, 107, 107, 0.3);
}

// ✅ 正确：使用函数
.overlay {
  background: set-alpha($primary-color, 30);
}
```

### 7.2 函数使用注意事项

1. **需要显式导入函数文件**：

```vue
<style scoped lang="scss">
@use "@/styles/scss/abstracts/_functions" as *;

.my-element {
  background: set-alpha($primary-color, 30);
}
</style>
```

2. **或者使用CSS变量**：

```scss
// 如果不需要动态计算，直接使用CSS变量
:root {
  --primary-color-30: rgba(255, 107, 107, 0.3);
}

.my-element {
  background: var(--primary-color-30);
}
```

## 九、最佳实践

### 9.1 优先级

1. **优先使用SCSS变量**：`$primary-color`（编译时替换，性能最优）
2. **其次使用CSS变量**：`var(--primary-color)`（运行时可修改，支持主题切换）
3. **避免硬编码**：`#ff6b6b`（维护困难，无法统一管理）

### 9.2 命名规范

```scss
// ✅ 好的命名：语义清晰
$primary-color
$bg-gradient-page
$shadow-btn-primary
$admin-text-primary

// ❌ 不好的命名：过于通用
$color1
$bg1
$shadow1
$text1
```

### 9.3 变量组合

```scss
// ✅ 推荐：使用组合变量
.game-button {
  background: $bg-gradient-primary;
  color: $text-white;
  box-shadow: $shadow-btn-primary;
  text-shadow: $text-shadow-primary;
}

// ❌ 不推荐：手动组合
.game-button {
  background: linear-gradient(135deg, $primary-color 0%, lighten($primary-color, 12%) 100%);
  color: #ffffff;
  box-shadow: 0 4px 0 shade($primary-color, 15%);
  text-shadow: 2px 2px 4px set-alpha($primary-color, 30);
}
```

## 十、检查清单

开发新功能时，请检查：

- [ ] 是否使用了SCSS变量而非硬编码？
- [ ] 是否选择了正确的变量类型（颜色/背景/阴影）？
- [ ] **是否使用了间距/尺寸变量（$spacing-*, $font-size-*, $border-radius-*）？**（v2.0 新增）
- [ ] Admin组件是否使用了`$admin-*`变量？
- [ ] 渐变是否使用了预定义的渐变变量？
- [ ] 阴影是否使用了预定义的阴影变量？
- [ ] **是否需要使用 color.adjust() 替代已弃用的 lighten()/darken()？**（v2.0 新增）
- [ ] **使用 color.adjust() 前是否导入了 @use 'sass:color'？**（v2.0 新增）
- [ ] 是否需要使用颜色函数（shade/tint/set-alpha）？
