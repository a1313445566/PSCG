## 二、常见AI幻觉陷阱

### 陷阱1: 先入为主

**错误示例**：
```
❌ 基于常见模式假设：
   "注册流程：填写账号、密码、确认密码"
```

**正确做法**：
```
✅ 验证实际实现：
   1. 搜索前端代码 → 确认无注册表单
   2. 检查后端API → 确认无注册接口
   3. 验证登录方式 → 学号+年级+班级
```

---

### 陷阱2: 过度推断

**错误示例**：
```
❌ 没有读取文件就重构：
   直接写"学生可以修改密码"
```

**正确做法**：
```
✅ 先读取，再重构：
   1. 读取原文件 → 确认无密码相关内容
   2. 搜索代码 → 确认无修改密码功能
   3. 基于实际 → 不写密码相关内容
```

---

### 陷阱3: 缺乏验证

**错误示例**：
```
❌ 假设后端有某个API：
   "学生可以通过邮箱找回密码"
```

**正确做法**：
```
✅ 搜索验证API是否存在：
   1. Grep "forgot-password" → 无结果
   2. Grep "reset-password" → 无结果
   3. 确认功能不存在 → 不写相关内容
```
