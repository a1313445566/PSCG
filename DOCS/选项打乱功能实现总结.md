## 核心逻辑

### 映射关系

```
reverseMapping (打乱后位置 -> 原始位置):
{
  0: 3,  // 打乱后位置 0 对应原始位置 3
  1: 0,  // 打乱后位置 1 对应原始位置 0
  2: 2,  // 打乱后位置 2 对应原始位置 2
  3: 1   // 打乱后位置 3 对应原始位置 1
}
```

### 数据流程

```
1. 前端加载题目（原始选项）
2. 前端打乱选项，生成 reverseMapping
3. 用户选择答案（基于打乱后的选项）
4. 前端提交答案 + reverseMapping
5. 后端将用户答案映射回原始位置
6. 后端验证答案
7. 后端将正确答案映射到打乱后位置
8. 后端返回结果（打乱后的选项和正确答案）
9. 前端显示结果
```

## 后台设置控制 🆕

### 设置界面
- 位置：后台管理系统 → 系统设置 → 答题设置
- 选项："答案随机排序" 开关
- 状态：开启/关闭

### 使用方法
1. **开启选项打乱**
   - 后台设置"答案随机排序"为"开启"
   - 每次答题选项会随机打乱
   - 防止学生记忆答案位置

2. **关闭选项打乱**
   - 后台设置"答案随机排序"为"关闭"
   - 选项保持原始顺序
   - 适用于调试或特定场景

### 实现逻辑
```javascript
// 前端根据设置决定是否打乱
if (randomizeAnswers) {
  // 打乱选项
  const { shuffledOptions, reverseMapping } = shuffleOptions(options);
} else {
  // 不打乱选项
  shuffleMapping = null;
}
```
