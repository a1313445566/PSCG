## 📝 文档维护规范

### 1. 新增文档流程
1. 根据文档类型选择对应目录
2. 创建新的Markdown文档
3. 更新 `目录结构说明.md`
4. 更新 `README.md` 导航链接
5. 更新文档状态和版本信息

### 2. 更新文档流程
1. 修改对应文档内容
2. 更新文档版本或修改日期
3. 如有重大变更，在CHANGELOG中记录

### 3. 命名规范
- 使用中文命名，便于理解
- 文件名应清晰描述文档内容
- 技术文档使用 `.md` 扩展名
- 脚本文件使用 `.sh` 扩展名
