## 一、命名规范

### 1.1 文件命名

| 类型 | 规范 | 示例 |
|------|------|------|
| Vue 组件 | PascalCase.vue | `HomeView.vue`, `QuestionCard.vue`, `AdminLayout.vue` |
| JavaScript/JS | camelCase.js | `useLoading.js`, `api.js`, `chatService.js` |
| SCSS 样式 | kebab-case.scss 或 `_kebab-case.scss` | `_variables.scss`, `admin-layout.scss` |
| 配置文件 | kebab-case | `database.js`, `.eslintrc.cjs` |

### 1.2 组件命名

```vue
<!-- 正确：PascalCase -->
<template>
  <SubjectCard />
  <QuestionList />
  <BatchAddQuestion />
</template>

<!-- 错误：禁止其他格式 -->
<template>
  <!-- ✗ subject-card -->
  <!-- ✗ Subject_card -->
</template>
```

### 1.3 变量与函数（camelCase）

```javascript
// 正确
const currentUser = ref(null)
const isLoading = computed(() => ...)
const handleSubmit = async () => { }
const fetchQuestionList = () => { }

// 错误
const CurrentUser = ref(null)      // ✗ PascalCase
const is_loading = ref(null)       // ✗ snake_case
const handle_submit = () => {}     // ✗ snake_case
```

### 1.4 常量（UPPER_SNAKE_CASE）

```javascript
// 正确
const MAX_RETRY_COUNT = 2
const TOKEN_EXPIRY_MS = 24 * 60 * 60 * 1000
const API_TIMEOUT = 30000
const SUPPORTED_IMAGE_TYPES = ['image/jpeg', 'image/png', 'image/webp']

// 错误
const maxRetryCount = 2           // ✗ 非全大写
const MaxRetryCount = 2           // ✗ 非 UPPER_SNAKE_CASE
```

### 1.5 CSS 类名（BEM 规范）

```scss
// Block__Element--Modifier 格式
.question-card {
  &__title { }
  &__options { }
  &--active { }
  &--disabled { }
}

// 正确示例
.subject-card__header
.admin-sidebar__menu-item--selected
.option-list__item--correct
```

## 二、Vue 组件规范

### 2.1 必须使用 `<script setup>`

```vue
<script setup>
// 正确：使用 Composition API + script setup
import { ref, computed, onMounted } from 'vue'
import { ElMessage } from 'element-plus'

const props = defineProps({
  title: {
    type: String,
    required: true
  },
  count: {
    type: Number,
    default: 0
  }
})

const emit = defineEmits(['update', 'delete'])

const loading = ref(false)
const dataList = ref([])

onMounted(() => {
  fetchData()
})
</script>

<!-- 禁止：Options API 或非 setup 写法 -->
<script>
export default {
  // ✗ 禁止这种写法
}
</script>
```

### 2.2 Props 必须定义类型

```javascript
// 正确：完整的 props 定义
const props = defineProps({
  questionId: {
    type: Number,
    required: true
  },
  options: {
    type: Array,
    default: () => []
  },
  disabled: {
    type: Boolean,
    default: false
  },
  callback: {
    type: Function,
    default: null
  }
})

// 错误：缺少类型定义
const props = defineProps(['id', 'name'])
```

### 2.3 Template 组织顺序

```vue
<template>
  <!-- 1. 根元素（仅一个） -->
  <div class="component-name">
    <!-- 2. 条件渲染 / 加载状态 -->
    <el-skeleton v-if="loading" :rows="5" />

    <!-- 3. 主要内容区 -->
    <div v-else class="content">
      <!-- 4. 列表/循环 -->
      <div v-for="item in list" :key="item.id" class="item">
        {{ item.name }}
      </div>
    </div>

    <!-- 5. 空状态 -->
    <el-empty v-if="!loading && list.length === 0" description="暂无数据" />

    <!-- 6. 分页 -->
    <el-pagination
      v-if="total > 0"
      :total="total"
      :page-size="pageSize"
      @current-change="handlePageChange"
    />
  </div>
</template>
```

### 2.4 样式必须 scoped

```vue
<style lang="scss" scoped>
/* 正确：scoped 隔离样式 */
.question-card {
  padding: 16px;

  &__title {
    font-size: 18px;
    font-weight: 600;
  }
}
</style>

<!-- 错误：全局样式污染 -->
<style>
.question-card { /* 会影响全局 */ }
</style>
```

### 2.5 导入顺序

```vue
<script setup>
// 1. Vue 核心
import { ref, reactive, computed, onMounted, watch } from 'vue'
```

### 3.2 BEM 命名规范

```scss
// Block: 独立的组件单元
// Element: Block 的组成部分（双下划线 __）
// Modifier: 状态或变体（双横线 --）

.question-card {                     // Block
  &__header {                        // Element
    &--highlighted { }               // Modifier
  }

  &__body {                          // Element
    padding: 16px;
  }

  &__option-list {                   // Element
    &__item {                        // 嵌套 Element
      &--selected { }                // Modifier
      &--correct { }
      &--wrong { }
    }
  }

  &--completed { }                   // Block Modifier
  &--skipped { }
}
```

### 3.3 CSS 变量使用

```scss
// _variables.scss 中定义
:root {
  // 颜色体系
  --color-primary: #409eff;
  --color-success: #67c23a;
  --color-warning: #e6a23c;
  --color-danger: #f56c6c;

  // 间距
  --spacing-xs: 4px;
  --spacing-sm: 8px;
  --spacing-md: 16px;
  --spacing-lg: 24px;
  --spacing-xl: 32px;

  // 圆角
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 12px;
}

// 组件中使用
.question-card {
  border-radius: var(--radius-md);
  padding: var(--spacing-lg);
  border: 1px solid var(--color-primary);
}
```

### 3.4 嵌套深度限制

```scss
// 正确：嵌套不超过 4 层
.question-card {
  &__header {
    &__title {
      font-size: 18px;  // 第 4 层，允许
    }
  }
}

// 错误：超过 4 层嵌套
.question-card {
  &__body {
    &__content {
      &__list {
        &__item {  // 第 5 层，禁止
          &__text { }  // 第 6 层，严重违规
        }
      }
    }
  }
}

// 解决方案：拆分为独立的 BEM 元素
.question-card {
  &__body { }
}
.question-list {
  &__item { }
}
.question-text { }
```

### 3.5 缩进与格式

- 缩进：**2 空格**
- 大括号：换行
- 选择器与括号之间：空格
- 属性声明后：分号（Prettier 配置为无分号，以 `.prettierrc` 为准）

## 四、API 调用规范

### 4.1 必须使用封装的 api.js

```javascript
// 正确：使用统一封装
import { apiGet, apiPost, apiPut, apiDelete } from '@/utils/api'

// GET 请求
const questions = await apiGet('/questions', { page: 1, pageSize: 20 })

// POST 请求
const result = await apiPost('/quiz/submit', { answers: userAnswers })

// PUT 请求
await apiPut(`/questions/${id}`, updatedData)

// DELETE 请求
await apiDelete(`/questions/${id}`)
```

### 4.2 禁止原生 fetch / axios

```javascript
// 错误：禁止直接使用 fetch
const res = await fetch('/api/questions')
const data = await res.json()

// 错误：禁止直接使用 axios
import axios from 'axios'
const res = await axios.get('/api/questions')
```

### 4.3 消息提示规范

```javascript
// 正确：使用统一消息工具
import { showSuccess, showError, showWarning, showInfo } from '@/utils/message'

showSuccess('操作成功')
showError('操作失败，请重试')
showWarning('请注意：此操作不可撤销')

// 错误：直接使用 ElMessage
import { ElMessage } from 'element-plus'
ElMessage.success('...')  // 禁止
```

### 4.4 错误处理规范

```javascript
// 所有异步请求必须 try/catch
const fetchData = async () => {
  const { loading, startLoading, stopLoading } = useLoading()
  startLoading()

  try {
    const data = await apiGet('/questions', { page: 1 })
    questionList.value = data.list
    total.value = data.total
  } catch (error) {
    console.error('获取题目列表失败:', error)
    showError('加载失败，请稍后重试')
  } finally {
    stopLoading()
  }
}
```

### 4.5 分页规范

```javascript
// 必须使用 usePagination composable
import { usePagination } from '@/composables/usePagination'

const { currentPage, pageSize, total, handlePageChange } = usePagination(totalRef)

// 在模板中使用
<el-pagination
  :current-page="currentPage"
  :page-size="pageSize"
  :total="total"
  @current-change="handlePageChange"
/>
```

## 五、Git 提交规范

### 5.1 Commit Message 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 5.2 Type 类型

| Type | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 文档更新 |
| `style` | 代码格式调整（不影响功能） |
| `refactor` | 重构（非新增功能、非修复Bug） |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具/依赖变更 |
| `build` | 构建系统变更 |

### 5.3 Scope 范围

常用 scope：`login`, `quiz`, `admin`, `question`, `user`, `analysis`, `chat`, `deploy`, `ci`

### 5.4 示例

```
feat(quiz): 添加判断题题型支持

- 新增 judgment 类型题目渲染逻辑
- 判断题选项固定为对/错，不打乱
- 更新答题卡支持判断题显示

Closes #123
```

```
fix(question): 修复选项打乱时判断题选项被错误打乱的bug

问题：判断题选项["对","错"]被打乱为["错","对"]
原因：shuffleOptions未检查题型
修复：添加type !== 'judgment'前置条件
```

## 六、ESLint 规则要点

### 6.1 配置文件

- ESLint 配置：[`.eslintrc.cjs`](file:///e:/PSCG/.eslintrc.cjs)
- 使用 `eslint-config-prettier` 避免 ESLint 与 Prettier 冲突

### 6.2 核心规则

| 规则 | 说明 |
|------|------|
| `vue/multi-word-component-names` | 组件名必须是多单词 |
| `no-console` | 允许 console.warn/error，禁止 console.log（调试时可临时放开） |
| `no-debugger` | 禁止 debugger 语句 |
| `prefer-const` | 优先使用 const |
| `no-unused-vars` | 禁止未使用的变量 |
| `vue/no-unused-vars` | Vue 模板中禁用未使用变量 |

### 6.3 运行检查

```bash
# 自动修复
npm run lint

# 仅检查（CI 用）
npm run lint:check
```

## 十、关键约束总结

1. **Vue 文件必须 `<script setup>`**，props 必须带类型定义
2. **API 调用必须用 `src/utils/api.js`**，禁止原生 fetch/axios
3. **消息提示必须用 `@/utils/message`**，禁止直接用 ElMessage
4. **样式必须 scoped**，SCSS 采用 7-1 架构 + BEM 命名
5. **图标必须显式导入** `@element-plus/icons-vue`
6. **加载状态用 useLoading**，分页用 usePagination
7. **提交前必须通过 `npm run lint && npm run format`**
8. **所有异步请求必须 try/catch**
