```vue
  fetchData()

<!-- 禁止：Options API 或非 setup 写法 -->
<script>

  // ✗ 禁止这种写法
```


### 2.3 Template 组织顺序

```vue
<template>
  <!-- 1. 根元素（仅一个） -->
  <div class="component-name">
    <!-- 2. 条件渲染 / 加载状态 -->
    <el-skeleton v-if="loading" :rows="5" />

    <!-- 3. 主要内容区 -->
    <div v-else class="content">
      <!-- 4. 列表/循环 -->
      <div v-for="item in list" :key="item.id" class="item">

      </div>

    <el-empty v-if="!loading && list.length === 0" description="暂无数据" />

      :total="total"
      :page-size="pageSize"

  </div>

```

## 三、CSS / SCSS 规范（v2.0 重大更新）

> ⚠️ **2026-04-07 重要更新**：项目已完成样式变量化改造，所有新代码必须遵守以下规范

### 3.0 核心原则（强制）

1. **禁止硬编码**：颜色、间距、字号、圆角、边框宽度必须使用 SCSS 变量
2. **变量优先级**：SCSS 变量 > CSS 变量 > 硬编码（仅允许透明黑等极少数例外）
3. **函数替代**：使用 `color.adjust()` 替代已弃用的 `lighten()/darken()`
4. **样式抽离**：`.vue` 文件中 `<style>` 超过 200 行且占比 > 30% 时，必须抽离为独立 `.scss` 文件
5. **全局注入**：Vite 已配置 `additionalData`，无需手动导入 `_variables.scss`

### 3.1 7-1 架构模式

本项目 SCSS 采用 7-1 架构，目录结构如下：

```
│   └── _mixins.scss       # Mixin 混入
```

### 3.5 color.adjust() 函数（强制使用）

> ⚠️ **重要**：`lighten()` 和 `darken()` 已被 Sass 官方弃用，必须使用 `color.adjust()` 替代。

```scss
@use 'sass:color';

// ❌ 旧语法（已弃用，Dart Sass 3.0 将移除）
.old-element {
  background: lighten($primary-color, 10%);
  border-color: darken($border-color, 10%);
}

// ✅ 新语法（推荐）
.new-element {
  background: color.adjust($primary-color, $lightness: 10%);
  border-color: color.adjust($border-color, $lightness: -10%);
}
```

**替换速查表**：

| 旧语法（已弃用） | 新语法（推荐） | 效果 |
|-----------------|---------------|------|
| `lighten($color, x%)` | `color.adjust($color, $lightness: x%)` | 加亮颜色 |
| `darken($color, x%)` | `color.adjust($color, $lightness: -x%)` | 加深颜色 |

### 3.6 颜色工具函数

在独立 `.scss` 文件中使用时，需显式导入：

```scss
@use "@/styles/scss/abstracts/_functions" as *;

.overlay {
  // ✅ 使用 set-alpha 函数
  background: set-alpha($primary-color, 30);  // rgba(255, 107, 107, 0.3)
  
  // ✅ 使用 shade/tint 函数
  border-color: shade($primary-color, 15%);   // 加深 15%
  background: tint($secondary-color, 20%);    // 加亮 20%
}
```

**可用函数**：

| 函数名 | 用途 | 示例 |
|--------|------|------|
| `set-alpha($color, $opacity)` | 设置透明度 | `set-alpha(#fff, 50)` → `rgba(255,255,255,0.5)` |
| `shade($color, $amount)` | 加深（混黑） | `shade($primary-color, 15%)` |
| `tint($color, $amount)` | 加亮（混白） | `tint($primary-color, 20%)` |
| `gradient($from, $to)` | 生成线性渐变 | `gradient($primary-color, $secondary-color)` |

### 3.8 硬编码禁止清单

以下情况**禁止硬编码**，必须使用变量：

| 类型 | 禁止示例 | 正确做法 |
|------|---------|---------|
| 十六进制颜色 | `#ff6b6b` | `$primary-color` |
| RGB/RGBA | `rgba(255,107,107,0.3)` | `set-alpha($primary-color, 30)` |
| 渐变 | `linear-gradient(135deg, ...)` | `$bg-gradient-page` |
| 阴影 | `0 4px 15px rgba(0,0,0,0.15)` | `$shadow-md` |
| 间距 | `padding: 16px` | `padding: $spacing-md` |
| 字号 | `font-size: 14px` | `font-size: $font-size-base` |
| 圆角 | `border-radius: 8px` | `border-radius: $border-radius-sm` |
| 边框宽度 | `border: 1px solid` | `border: $border-width solid` |

**允许的例外**（需注释说明）：

```scss
// ✅ 允许：透明纯黑（过于通用）
.overlay {
  background: rgba(0, 0, 0, 0.5);  // 通用遮罩层
}

// ✅ 允许：SVG 图形颜色数据（非 CSS 样式）
.icon-svg {
  fill: #2d3748;  // SVG 内联图形色
}

// ✅ 允许：Element Plus 深度覆盖（遵循库格式）
:deep(.el-button--primary) {
  --el-button-bg-color: #409eff;  // Element Plus 规定格式
}
```

### 3.9 样式文件抽离规范

当 `.vue` 文件满足以下条件时，**必须**将 `<style>` 抽离为独立 `.scss` 文件：

**触发条件**（满足任一即可）：
- `<style>` 行数 > 200 行 且 样式占比 > 30%
- 文件总行数 > 600 行

**抽离方式**：

```vue
<!-- 抽离前：QuestionList.vue（800 行，其中 style 400 行） -->
<style lang="scss" scoped>
.question-list { /* 400 行样式 */ }
</style>

<!-- 抽离后 -->
<style lang="scss" src="./styles/QuestionList.scss" scoped></style>
```

```
src/components/quiz/
  styles/
    QuestionList.scss    ← 新建独立文件
  QuestionList.vue       ← 仅保留 template + script（~400 行）
```

### 3.10 嵌套深度限制

```scss
// 正确：嵌套不超过 4 层
.question-card {
  &__header {
    &__title {
      font-size: $font-size-lg;  // 第 4 层，允许
    }
  }
}

// 错误：超过 4 层嵌套
.question-card {
  &__body {
    &__content {
      &__list {
        &__item {  // 第 5 层，禁止
          &__text { }  // 第 6 层，严重违规
        }
      }
    }
  }
}

// 解决方案：拆分为独立的 BEM 元素
.question-card {
  &__body { }
}
.question-list {
  &__item { }
}
.question-text { }
```

## 四、API 调用规范

### 4.1 必须使用封装的 api.js

```javascript
// 正确：使用统一封装
import { apiGet, apiPost, apiPut, apiDelete } from '@/utils/api'

await apiPut(`/questions/${id}`, updatedData)
```

```javascript
// 错误：禁止直接使用 fetch

// 错误：禁止直接使用 axios
```

### 4.3 消息提示规范

```javascript
// 正确：使用统一消息工具
import { showSuccess, showError, showWarning, showInfo } from '@/utils/message'

showSuccess('操作成功')
```

### 4.4 错误处理规范

```javascript
// 所有异步请求必须 try/catch
const fetchData = async () => {
  const { loading, startLoading, stopLoading } = useLoading()
  startLoading()

  try {
    const data = await apiGet('/questions', { page: 1 })
    questionList.value = data.list
    total.value = data.total
  } catch (error) {
    console.error('获取题目列表失败:', error)
    showError('加载失败，请稍后重试')
  } finally {
    stopLoading()
  }
}
```

### 4.5 分页规范

## 十、关键约束总结

1. **Vue 文件必须 `<script setup>`**，props 必须带类型定义
2. **API 调用必须用 `src/utils/api.js`**，禁止原生 fetch/axios
3. **消息提示必须用 `@/utils/message`**，禁止直接用 ElMessage
4. **样式必须 scoped**，SCSS 采用 7-1 架构 + BEM 命名
5. **图标必须显式导入** `@element-plus/icons-vue`
6. **加载状态用 useLoading**，分页用 usePagination
7. **提交前必须通过 `npm run lint && npm run format`**
8. **所有异步请求必须 try/catch**
9. **✨ 样式必须使用 SCSS 变量**（颜色/间距/字号/圆角/边框宽度），禁止硬编码
10. **✨ 使用 `color.adjust()` 替代已弃用的 `lighten()/darken()`**
11. **✨ 样式超过 200 行必须抽离为独立 `.scss` 文件**
