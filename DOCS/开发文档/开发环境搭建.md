npm -v

# 检查 MySQL 是否运行
mysql -u root -p -e "SELECT VERSION();"

## 三、配置环境变量

### 3.1 创建 .env 文件

在项目根目录创建 `.env` 文件，复制 `.env.example` 并修改：

```bash
cp .env.example .env
```

### 3.2 配置项说明

```env
# ============================================
# 服务器配置
# ============================================
SERVER_PORT=3001

# ============================================
# API 地址配置
# 开发环境: http://localhost:3001/api
# 生产环境: /api（推荐，使用相对路径自动适配域名）
# ============================================
VITE_API_URL=/api

# ============================================
# 签名密钥（必须替换为实际密钥）
# 生成命令: node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
# ============================================
SIGNATURE_SECRET=your-secret-key-here
VITE_SIGNATURE_SECRET=your-secret-key-here

# ============================================
# 数据库配置
# ============================================
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=PSCG
DB_PASSWORD=your-password
DB_NAME=pscg
```

### 3.3 关键配置说明

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `SERVER_PORT` | 否 | 后端服务端口，默认 3001 |
| `VITE_API_URL` | 是 | 前端请求的 API 基础路径，开发用 `http://localhost:3001/api`，生产用 `/api` |
| `SIGNATURE_SECRET` | 是 | JWT 签名密钥，**生产环境必须替换为强随机值** |
| `DB_HOST` | 是 | MySQL 地址 |
| `DB_USER` | 是 | 数据库用户名 |
| `DB_PASSWORD` | 是 | 数据库密码 |
| `DB_NAME` | 是 | 数据库名称 |

## 四、数据库初始化

### 4.1 创建数据库

登录 MySQL 并创建数据库：

```sql
-- 登录 MySQL
mysql -u root -p

-- 创建数据库（字符集使用 utf8mb4）
CREATE DATABASE pscg DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建专用用户（推荐）
CREATE USER 'PSCG'@'localhost' IDENTIFIED BY 'your-password';
GRANT ALL PRIVILEGES ON pscg.* TO 'PSCG'@'localhost';
FLUSH PRIVILEGES;
```

### 4.2 执行迁移脚本

项目提供了多个迁移脚本，位于 `migrations/` 和 `scripts/` 目录：

```bash
# 主迁移脚本（初始化表结构）
mysql -u PSCG -p pscg < migrations/migrate_to_aittest.sql

# 判断题类型迁移（如需要）
mysql -u PSCG -p pscg < scripts/migrate-add-judgment-type.sql

# 其他迁移脚本按需执行
```

### 4.3 验证数据库连接

启动后端服务后访问健康检查接口：

```bash
curl http://localhost:3001/api/health
```

或浏览器打开 `http://localhost:3001/api/health`。

## 五、启动开发服务器

本项目采用前后端分离架构，需要分别启动前端和后端服务。

### 5.1 启动后端服务（端口 3001）

```bash
# 方式一：直接启动
npm run server

# 方式二：Windows PowerShell
.\node server.cjs
```

后端服务启动成功后会输出类似以下日志：

```
Server running on port 3001
Database connected successfully
```

### 5.2 启动前端开发服务器（端口 5173）

**新开一个终端窗口**执行：

```bash
npm run dev
```

Vite 开发服务器启动后会显示：

```
  VITE v5.2.0  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

### 5.3 Vite 开发代理配置

前端开发服务器已配置代理（[vite.config.js](file:///e:/PSCG/vite.config.js)），自动将以下路径转发到后端：

| 代理路径 | 目标 | 说明 |
|----------|------|------|
| `/api/*` | `http://localhost:3001` | 所有 API 请求 |
| `/images/*` | `http://localhost:3001` | 图片资源 |
| `/audio/*` | `http://localhost:3001` | 音频资源 |

因此开发环境下前端可以直接使用 `/api/...` 格式的请求地址，无需跨域处理。

## 七、常见问题排查

### 7.1 端口占用

**现象**：`Error: listen EADDRINUSE :::3001` 或 `:::5173`

**解决**：

```bash
# Windows PowerShell - 查找并结束进程
netstat -ano | findstr :3001
taskkill /PID <进程ID> /F

# 或修改端口（server.cjs 中修改 SERVER_PORT）
```

### 7.2 数据库连接失败

**现象**：`ECONNREFUSED` 或 `Access denied`

**排查步骤**：

1. 确认 MySQL 服务已启动
2. 检查 `.env` 中的 `DB_HOST`、`DB_PORT`、`DB_USER`、`DB_PASSWORD`
3. 确认数据库 `pscg` 已创建
4. 确认用户权限正确

```bash
# 测试数据库连接
mysql -h 127.0.0.1 -u PSCG -p -e "SELECT 1;"
```

### 7.3 依赖安装失败

**现象**：`npm install` 报错

**解决**：

```bash
# 清除缓存重试
npm cache clean --force
rm -rf node_modules package-lock.json
npm install

# 如遇 sharp 安装问题（常见于 Windows）
npm install sharp --ignore-scripts
npm rebuild sharp
```

### 7.4 Vite HMR 不生效

**现象**：修改代码后页面不自动更新

**解决**：
1. 确认终端无报错
2. 尝试重启 `npm run dev`
3. 检查是否有防火墙阻止 WebSocket 连接

### 7.5 后端 API 404

**现象**：前端调用接口返回 404

**解决**：
1. 确认后端服务正在运行（检查 3001 端口）
2. 确认 `VITE_API_URL` 配置正确
3. 查看后端控制台是否有路由注册错误
