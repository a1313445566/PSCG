# 开发环境搭建指南

## 环境要求

### 必需软件
- **Node.js**: 18.x 或更高版本（推荐 v20+）
- **npm**: 8.0 或更高版本
- **MySQL**: 8.0 或更高版本

### 推荐扩展（VS Code）
- Volar (Vue 开发)
- ESLint
- Prettier

## 安装步骤

### 1. 克隆项目
```bash
git clone https://github.com/a1313445566/PSCG.git
cd PSCG
```

### 2. 安装依赖
```bash
# 安装项目依赖
npm install
```

### 3. 配置数据库

#### 创建数据库
```sql
-- 连接 MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE pscg DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户（使用 PSCG 大写）
CREATE USER 'PSCG'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON pscg.* TO 'PSCG'@'localhost';
FLUSH PRIVILEGES;

-- 退出
EXIT;
```

**注意**：数据库用户名必须使用 `PSCG`（大写），与 `config/database.js` 配置保持一致

#### 配置数据库连接
编辑 `config/database.js` 文件：
```javascript
module.exports = {
  host: 'localhost',
  port: 3306,
  user: 'PSCG',
  password: 'your_password',
  database: 'pscg',
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0
}
```

### 4. 初始化数据库

**方法 1：手动导入 SQL 文件**
```bash
mysql -u PSCG -p pscg < scripts/db-migration.sql
```

**方法 2：使用迁移脚本**
```bash
mysql -u PSCG -p pscg < migrations/migrate_to_aittest.sql
```

**注意**：数据库表结构会在后端服务首次启动时自动创建（通过 `services/database.js` 中的初始化逻辑）

### 5. 配置环境变量

项目根目录创建 `.env` 文件：
```env
# 服务器配置
SERVER_PORT=3001

# API 地址配置（开发环境使用 /api 代理）
VITE_API_URL=/api

# 签名密钥（请替换为实际密钥）
SIGNATURE_SECRET=your-secret-key-here
VITE_SIGNATURE_SECRET=your-secret-key-here

# 数据库配置
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=PSCG
DB_PASSWORD=your_password
DB_NAME=pscg
```

**注意**：
- `SIGNATURE_SECRET` 用于 API 签名验证，请替换为安全的随机密钥
- 生成密钥：`node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"`

### 6. 启动开发服务器

#### 启动前端开发服务器
```bash
npm run dev
```
访问：http://localhost:5173

#### 启动后端服务器
```bash
npm run server
```
API 地址：http://localhost:3001/api

## 常见问题

### 1. 端口被占用

**Windows 系统**：
```powershell
# 查看端口占用
netstat -ano | findstr :3001
netstat -ano | findstr :5173

# 结束进程
taskkill /PID <PID> /F
```

**Linux/Mac 系统**：
```bash
# 查看端口占用
lsof -i :3001
lsof -i :5173

# 结束进程
kill -9 <PID>
```

**修改端口**：
- 前端：在 `vite.config.js` 的 `server` 配置中添加 `port: 自定义端口`
- 后端：修改 `.env` 中的 `SERVER_PORT`

### 2. 数据库连接失败

**Windows 系统**：
- 检查 MySQL 服务是否启动（服务管理器中查看）
- 检查数据库配置信息（`config/database.js` 或 `.env`）
- 检查 Windows 防火墙设置
- 验证数据库用户权限：`SHOW GRANTS FOR 'PSCG'@'localhost';`

**Linux/Mac 系统**：
- 检查 MySQL 服务是否启动：`systemctl status mysql`
- 检查数据库配置信息
- 检查防火墙设置：`ufw status`
- 验证数据库用户权限

### 3. 依赖安装失败

**Windows 系统**：
```powershell
# 清除缓存
npm cache clean --force

# 删除 node_modules
rmdir /s /q node_modules
del package-lock.json

# 重新安装
npm install
```

**Linux/Mac 系统**：
```bash
# 清除缓存
npm cache clean --force

# 删除 node_modules
rm -rf node_modules package-lock.json

# 重新安装
npm install
```

### 4. 前端页面空白
- 检查控制台错误信息
- 清除浏览器缓存
- 检查后端服务是否启动
- 检查 API 地址配置

### 5. 热重载不生效
- 检查 Vite 配置
- 重启开发服务器
- 检查文件保存设置

## 开发建议

### 1. 代码规范
- 使用 ESLint + Prettier 保持代码风格一致
- Vue 组件使用 `<script setup>` 语法
- 组件命名使用 PascalCase
- 变量和函数使用 camelCase

### 2. 数据库操作
- 使用参数化查询防止 SQL 注入
- 所有列表接口必须分页
- 高频查询字段添加索引
- 批量操作分批执行

### 3. API 开发
- 统一的响应格式
- 完善的错误处理
- 输入参数验证
- 适当的日志记录

### 4. 前端开发
- 组件化开发
- 使用 composables 复用逻辑
- 响应式设计
- 性能优化（虚拟滚动、懒加载等）
