# 💻 PSCG 开发环境设置指南

本文档详细介绍了如何设置 PSCG 项目的本地开发环境。

## 1. 环境准备

### 1.1 系统要求
| 环境 | 版本要求 | 备注 |
|------|----------|------|
| Node.js | 16.x 或更高 | 建议使用 LTS 版本 |
| NPM | 8.x 或更高 | 随 Node.js 安装 |
| MySQL | 8.0 或更高 | 或 MariaDB 10.5+ |
| Git | 最新版本 | 版本控制工具 |

### 1.2 推荐开发工具
| 工具 | 用途 | 备注 |
|------|------|------|
| VS Code | 代码编辑器 | 推荐安装 Vue、ESLint 扩展 |
| Postman | API 测试 | 或使用 Thunder Client 扩展 |
| MySQL Workbench | 数据库管理 | 可选，Navicat 替代 |
| Chrome DevTools | 浏览器调试 | Firefox DevTools 也可 |

## 2. 项目克隆和初始化

### 2.1 获取项目代码
```bash
# 克隆项目
git clone https://your-repository-url/pscg.git
cd pscg

# 或者如果你已经下载了项目文件
cd /path/to/pscg
```

### 2.2 安装依赖
```bash
# 安装项目依赖
npm install

# 如果需要，全局安装开发工具
npm install -g @vue/cli
npm install -g nodemon
```

### 2.3 配置本地环境
```bash
# 创建本地环境配置文件
cp .env.example .env.local

# 编辑配置文件
nano .env.local
```

`.env.local` 示例：
```env
# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_NAME=pscg_dev
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_CHARSET=utf8mb4

# JWT 配置
JWT_SECRET=dev_jwt_secret_key
SESSION_SECRET=dev_session_secret_key

# 应用配置
NODE_ENV=development
PORT=3001
VITE_API_BASE_URL=http://localhost:3001/api
```

### 11.3 API 请求封装
```javascript
// utils/api.js
import axios from 'axios'

const api = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json'
  }
})

// 请求拦截器
api.interceptors.request.use(
  config => {
    const token = localStorage.getItem('token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    return config
  },
  error => Promise.reject(error)
)

export default api
```

## 12. 后端开发技巧

### 12.1 中间件开发
```javascript
// middleware/auth.js
const jwt = require('jsonwebtoken')

const authMiddleware = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1]
  
  if (!token) {
    return res.status(401).json({
      success: false,
      message: '未提供认证令牌'
    })
  }
  
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET)
    req.user = decoded
    next()
  } catch (error) {
    return res.status(401).json({
      success: false,
      message: '无效的认证令牌'
    })
  }
}

module.exports = authMiddleware
```

### 12.2 数据库查询优化
```javascript
// services/userService.js
const getUsersWithStatistics = async () => {
  const sql = `
    SELECT 
      u.*,
      COUNT(DISTINCT ar.id) as total_attempts,
      AVG(ar.correct) as avg_correct_rate
    FROM users u
    LEFT JOIN answer_records ar ON u.id = ar.user_id
    GROUP BY u.id
    ORDER BY u.created_at DESC
    LIMIT ?
  `
  
  return await db.query(sql, [100])
}
```

## 14. 故障排除

### 14.1 常见问题

#### 问题1：端口被占用
```bash
# 查看端口占用
lsof -i :3001

# 或
netstat -tlnp | grep :3001

# 杀死进程
kill -9 <PID>
```

#### 问题2：MySQL 连接失败
```bash
# 检查 MySQL 服务
sudo systemctl status mysql

# 查看错误日志
sudo tail -f /var/log/mysql/error.log

# 重置密码
sudo mysql_secure_installation
```

#### 问题3：依赖安装失败
```bash
# 清理缓存
npm cache clean --force

# 删除 node_modules
rm -rf node_modules package-lock.json

# 重新安装
npm install
```
