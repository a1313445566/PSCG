# PSCG 安全架构文档

## 概述

本文档描述 PSCG 智能题库系统的安全架构设计，包括管理员认证、数据安全、请求防护等方面。

## 1. 管理员认证系统

### 1.1 认证架构

系统采用 **JWT (JSON Web Token)** 认证机制，管理员凭证存储在数据库中，支持密码修改功能。

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   管理员登录     │───▶│  JWT Token 签发  │───▶│   Token 存储     │
│  (用户名+密码)   │    │   (有效期24h)    │    │  (sessionStorage)│
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

### 1.2 密码存储

密码使用 **PBKDF2** 算法进行哈希存储：

| 参数 | 值 |
|------|-----|
| 算法 | PBKDF2 |
| 迭代次数 | 100,000 |
| 盐值长度 | 64 字节 |
| 派生密钥长度 | 128 字节 |
| 哈希函数 | SHA-512 |

**实现文件**: `services/passwordHash.js`

```javascript
// 密码哈希生成
const hash = await hashPassword(plainPassword);
// 存储: hash.salt:hash.derivedKey

// 密码验证
const isValid = await verifyPassword(plainPassword, storedHash);
```

### 1.3 数据库表结构

```sql
CREATE TABLE admin_credentials (
  id INT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) UNIQUE NOT NULL,
  password_hash VARCHAR(512) NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 1.4 相关 API

| 接口 | 方法 | 说明 | 认证要求 |
|------|------|------|---------|
| `/api/admin/login` | POST | 管理员登录 | 无 |
| `/api/admin/verify` | GET | 验证 Token | 需要 |
| `/api/admin/change-password` | POST | 修改密码 | 需要 |
| `/api/admin/verify-data-management` | POST | 数据管理二次验证 | 需要 |

## 2. 签名验证系统

### 2.1 答题提交签名

为防止答题数据篡改，系统实现了签名验证机制：

```
前端                              后端
  │                                │
  │  1. 生成签名数据               │
  │     - quizId                  │
  │     - answers                 │
  │     - timestamp               │
  │                                │
  │  2. HMAC-SHA256 签名           │
  │     或降级签名(非HTTPS)        │
  │                                │
  │  3. 提交数据 + 签名 ──────────▶│
  │                                │ 4. 验证时间戳
  │                                │    (±5分钟有效期)
  │                                │
  │                                │ 5. 验证签名
  │                                │
  │◀────────── 返回结果 ──────────│
```

### 2.2 时间戳验证规则

- **有效期**: 提交时间与服务器时间差不超过 5 分钟
- **未来时间容忍**: 允许 1 分钟的未来时间偏移
- **过期处理**: 返回 401 错误，要求重新提交

### 2.3 相关配置

```env
# .env 文件
SIGNATURE_SECRET=your-secret-key
VITE_SIGNATURE_SECRET=your-secret-key
```

**注意**: 前后端签名密钥必须一致。

## 3. 请求限流系统

### 3.1 限流中间件

系统实现了基于 IP 的请求限流：

| 功能 | 说明 |
|------|------|
| 请求追踪 | 记录每个 IP 的请求次数 |
| 自动封禁 | 超过阈值自动封禁 IP |
| 自动解封 | 封禁时间到期自动解除 |
| 手动管理 | 支持手动封禁/解封 IP |

### 3.2 配置参数

```javascript
// routes/quiz.js
const submitLimiter = new RateLimiter({
  windowMs: 60 * 1000,      // 时间窗口: 1分钟
  maxRequests: 10,          // 最大请求: 10次
  blockDuration: 5 * 60 * 1000  // 封禁时长: 5分钟
});
```

### 3.3 安全 API

| 接口 | 方法 | 说明 | 认证要求 |
|------|------|------|---------|
| `/api/security/status` | GET | 获取安全状态 | 管理员 |
| `/api/security/blocked-ips` | GET | 获取封禁 IP 列表 | 管理员 |
| `/api/security/unblock-ip` | POST | 解封指定 IP | 管理员 |
| `/api/security/unblock-all` | POST | 解封所有 IP | 管理员 |

## 5. 安全最佳实践

### 5.1 管理员账户

1. **初始化**: 使用 `node scripts/admin-tool.js` 创建管理员账户
2. **密码要求**: 建议使用强密码（8位以上，包含大小写字母和数字）
3. **定期更换**: 建议定期修改管理员密码
4. **账号安全**: 不要在公共环境登录管理员账号

### 5.2 环境配置

1. **密钥管理**: 不要将 `.env` 文件提交到 Git 仓库
2. **生产环境**: 使用强随机的 `SIGNATURE_SECRET`
3. **HTTPS**: 生产环境强烈建议启用 HTTPS

### 5.3 数据安全

1. **定期备份**: 使用后台数据备份功能
2. **访问控制**: 限制管理后台的访问权限
3. **日志监控**: 定期检查安全日志

## 8. 故障排除

### 8.1 登录失败

**症状**: 提示"登录失败"

**排查步骤**:
1. 检查管理员账户是否已初始化
2. 确认用户名和密码正确
3. 查看服务器日志: `pm2 logs PSCG`
4. 尝试重置密码

### 8.2 Token 验证失败

**症状**: 后台操作提示"未授权"

**排查步骤**:
1. 检查 Token 是否过期（24小时有效期）
2. 清除浏览器 sessionStorage 重新登录
3. 确认服务器 JWT_SECRET 配置正确

### 8.3 签名验证失败

**症状**: 答题提交失败

**排查步骤**:
1. 检查前后端 `SIGNATURE_SECRET` 是否一致
2. 检查客户端系统时间是否正确
3. 参考《签名验证调试指南.md》
