## 🐛 发现的 Bug 清单（共 4 类）

### Bug 1️⃣：错误的数据库模块引用

**影响文件**：
- `middleware/adminAuth.js`
- `services/permissionService.js`

**错误代码**：
```javascript
// ❌ 使用纯配置对象（只有配置参数，没有连接池）
const db = require('../config/database')
```

**正确代码**：
```javascript
// ✅ 使用 Database 类实例（有 pool.execute 方法）
const db = require('../services/database')
```

**根因分析**：
- `config/database.js` 导出的是 `{ host, port, user, password, database }` 配置对象
- `services/database.js` 导出的是 Database 单例实例，包含 `pool.execute()` 方法
- 新开发的权限系统代码沿用了错误的引用方式

---

### Bug 2️⃣：错误的数据库调用方法

**影响文件**：
- `middleware/adminAuth.js:65`
- `services/permissionService.js` (13 处)

**错误代码**：
```javascript
// ❌ Database 类没有 execute() 方法
const [rows] = await db.execute(sql, params)
```

**正确代码**：
```javascript
// ✅ 使用连接池的 execute() 方法
const [rows] = await db.pool.execute(sql, params)
```

**技术细节**：
- MySQL2 的 Pool 对象提供 `pool.execute()` 方法
- 返回值是 `[rows, fields]` 结构，需要解构获取行数据

---

### Bug 3️⃣：MySQL LIMIT/OFFSET 参数化限制

**影响文件**：
- `services/permissionService.js:getRoles()`
- `services/permissionService.js:getAdminUsers()`

**错误代码**：
```javascript
// ❌ MySQL 不支持在 LIMIT/OFFSET 中使用参数占位符
const [rows] = await db.pool.execute(
  'SELECT ... LIMIT ? OFFSET ?',
  [limit, offset]
)
-- 报错：ER_WRONG_ARGUMENTS - Incorrect arguments to mysqld_stmt_execute
```

**正确代码**：
```javascript
// ✅ 将 LIMIT/OFFSET 值直接拼接到 SQL 字符串
const [rows] = await db.pool.execute(
  `SELECT ... LIMIT ${parseInt(limit)} OFFSET ${parseInt(offset)}`
)
```

**注意事项**：
- 使用 `parseInt()` 确保值是整数，防止 SQL 注入
- 只对数值型参数使用字符串拼接，其他参数仍用 `?` 占位符

---

### Bug 4️⃣：response 函数调用签名错误（已确认正确）

**正确的函数签名** (`utils/response.js`)：
```javascript
success(res, data, message)   // res 在第一位
error(res, message, status)   // res 在第一位
```

**正确调用方式** (`routes/admin-permissions.js`)：
```javascript
// ✅ 正确
response.success(res, result, '获取角色列表成功')
response.error(res, '获取角色列表失败', 500)
```

**常见错误**（需避免）：
```javascript
// ❌ 错误：把 result 当 res 传进去
res.json(response.success(result, '成功'))

// ❌ 错误：漏传 res 参数
res.status(500).json(response.error('失败'))
```

## 💡 经验教训与最佳实践

### 1. 数据库模块引用规范

| 模块类型 | 引用方式 | 使用场景 |
|----------|----------|----------|
| **Database 类** | `require('./database')` 或 `require('../services/database')` | 业务逻辑层（services、middleware） |
| **配置对象** | `require('../config/database')` | 仅用于读取配置参数 |

### 2. MySQL2 正确用法

```javascript
// ✅ 正确：使用连接池
const db = require('./database')

// 查询（返回 [rows, fields]）
const [rows] = await db.pool.execute('SELECT * FROM table WHERE id = ?', [id])

// 带分页的查询（LIMIT/OFFSET 不能用参数化）
const offset = (page - 1) * limit
const [rows] = await db.pool.execute(
  `SELECT * FROM table ORDER BY id DESC LIMIT ${limit} OFFSET ${offset}`
)

// 插入（返回 result）
const [result] = await db.pool.execute('INSERT INTO table (...) VALUES (?)', [value])
console.log(result.insertId) // 获取插入 ID
```

### 3. response 工具函数正确用法

```javascript
const response = require('../utils/response')

// 成功响应
response.success(res, data, '操作成功消息')
// → { success: true, data: {...}, message: '操作成功消息' }

// 错误响应
response.error(res, '错误消息', 500)
// → 500 { error: '错误消息', success: false }

// 常用快捷方式
response.badRequest(res, '参数错误')     // 400
response.notFound(res, '资源')           // 404
response.unauthorized(res, '未授权')      // 401
```

### 4. 调试技巧：添加分层日志

当遇到 API 错误时，按以下顺序添加日志：

```javascript

console.log('[API Debug] token 长度:', adminToken?.length || 0)

// 2️⃣ 后端中间件：验证 Token 接收和解析

console.log('[AdminAuth Debug] JWT 验证结果:', decoded)

console.log('[Service Debug] 具体数据:', rows[0])
```
