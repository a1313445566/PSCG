# PSCG 全栈智能题库系统 - 项目介绍

## 一、项目概述

**PSCG (Personalized Study & Challenge Generator)** 是一套面向教育场景的全栈智能题库管理系统，采用前后端分离架构，集成了题目管理、智能答题、AI 辅助教学、数据分析等核心功能模块。系统支持学生在线练习、错题巩固、学习进度追踪，同时为管理员提供完善的后台管理功能和 AI 智能分析能力。

### 核心价值
- **个性化学习**：根据学生答题情况智能推荐题目，支持难度自适应
- **智能化管理**：集成 19 个 AI 工具的助手系统，辅助教学决策
- **数据驱动**：全方位的学习数据分析，帮助教师精准掌握学情
- **安全可靠**：11 项安全防护措施，保障系统和数据安全

## 二、功能模块详解

### 1. 用户系统

#### 学生端
- **登录方式**：学号自动识别 + 年级班级选择
- **认证机制**：JWT Token 认证，30 分钟会话超时
- **用户信息**：姓名、年级、班级、积分（points）存储于 `users` 表
- **唯一性约束**：`student_id + grade + class` 联合唯一索引

#### 管理员端
- **独立认证**：用户名密码登录，与学生系统完全隔离
- **密码安全**：bcrypt 哈希加密存储（12 轮 salt rounds）
- **凭证存储**：`admin_credentials` 表，支持多管理员
- **权限控制**：通过 `middleware/adminAuth.js` 统一鉴权

#### 技术实现
```javascript
// JWT Token 生成（后端）
const token = jwt.sign(
  { studentId: user.student_id, userId: user.id },
  process.env.JWT_SECRET,
  { expiresIn: '30m' }
)

// 密码哈希（管理员）
const hashedPassword = await bcrypt.hash(password, 12)
```

### 3. 题目系统

#### 题型支持（3 种）

| 类型标识 | 名称 | 选项格式 | 特殊处理 |
|----------|------|----------|----------|
| `single` | 单选题 | A/B/C/D 多选项 | 支持乱序 |
| `multiple` | 多选题 | A/B/C/D 多选项 | 支持乱序 |
| `judgment` | 判断题 | 固定 ["对", "错"] | **禁止乱序** |

> ⚠️ **判断题特殊规范**：
> - 选项固定为 `["对", "错"]`
> - 前后端都不进行打乱处理
> - 使用 `shuffleOptions()` 前必须检查题型：`if (q.type !== 'judgment')`
> - 答案为 `A`（对）或 `B`（错）

#### 题目数据结构（questions 表）
```sql
CREATE TABLE questions (
  id INT PRIMARY KEY AUTO_INCREMENT,
  subject_id INT NOT NULL,           -- 所属学科
  subcategory_id INT NOT NULL,       -- 所属子分类
  content TEXT NOT NULL,             -- 题目内容（支持富文本）
  type VARCHAR(20) DEFAULT 'single', -- 题型枚举
  options TEXT NOT NULL,             -- JSON 格式选项数组
  correct_answer TEXT NOT NULL,      -- 正确答案
  explanation TEXT,                  -- 解析说明
  difficulty INT DEFAULT 1,          -- 难度等级 (1-5)
  audio_url VARCHAR(255),            -- 音频附件 URL
  image_url VARCHAR(255),            -- 图片附件 URL
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  -- 外键和索引...
);
```

#### 核心功能

##### 3.1 批量导入
- **入口**：`BatchAddQuestion.vue` 组件
- **支持格式**：Excel / JSON 批量导入
- **数据校验**：使用 Zod schema 验证每条记录
- **事务保证**：批量操作使用数据库事务，确保原子性

##### 3.2 分屏编辑
- **组件**：`SplitEditPanel.vue`
- **布局**：左侧题目列表 + 右侧编辑面板
- **特性**：实时预览、草稿自动保存（draftStorage.js）

##### 3.3 文件去重机制
- **存储表**：`file_hashes`
- **工作流程**：
  ```
  上传文件 → 计算 SHA256 哈希 → 查询 file_hashes 表
      ↓ 存在          ↓ 不存在
  返回已有路径     保存文件 + 写入记录
  ref_count++       
  ```
- **字段设计**：
  ```sql
  file_hash VARCHAR(64) NOT NULL UNIQUE,  -- 文件哈希值
  file_path VARCHAR(255) NOT NULL,         -- 实际存储路径
  file_type ENUM('image', 'audio'),        -- 文件类型
  ref_count INT DEFAULT 1,                 -- 引用计数
  ```

##### 3.4 附件支持
- **图片**：JPEG/PNG/GIF/WebP，最大 10MB
- **音频**：MP3/WAV/OGG，最大 10MB
- **上传中间件**：Multer 配置（server.cjs:24-65）
- **图片优化**：Sharp 自动压缩和格式转换

### 4. 答题系统

#### 4.1 答题流程

```
选择学科 → 选择子分类 → 创建答题会话(quiz_sessions)
    → 随机抽题(N道) → 选项乱序(shuffleOptions) 
    → 用户答题 → 实时计时 → 行为追踪(answer_behavior)
    → 提交批改 → 记录结果(answer_records + quiz_attempts)
    → 错题收集(error_collection) → 显示成绩单(ResultView)
```

#### 4.2 核心机制

##### 随机抽题
- **接口**：`POST /api/quiz/start`
- **参数**：subjectId, subcategoryId, questionCount
- **逻辑**：
  - 从指定子分类随机抽取 N 道题
  - 支持固定数量或随机范围（可配置）
  - 返回完整的答题会话对象

##### 选项乱序（shuffleOptions）
- **工具函数**：`src/utils/shuffleOptions.js`
- **算法**：Fisher-Yates 洗牌算法
- **映射保存**：生成 `shuffleMapping` 用于还原正确答案
- **特殊处理**：
  ```javascript
  // 判断题不进行乱序
  if (q.type === 'judgment') {
    return { ...q, options: ['对', '错'], shuffleMapping: null }
  }
  
  // 其他题型正常打乱
  const shuffled = fisherYatesShuffle(q.options)
  const mapping = generateMapping(originalOptions, shuffled)
  return { ...q, options: shuffled, shuffleMapping: mapping }
  ```

##### 答题计时
- **组件**：`QuizView.vue` 内置计时器
- **记录字段**：`time_spent`（秒）
- **超时处理**：会话过期时间（expires_at），默认 2 小时

##### 行为追踪（answer_behavior）
- **存储表**：`answer_behavior`
- **追踪内容**：
  - 每次点击选项的时间戳
  - 修改答案的次数和顺序
  - 最终提交前的停留时间
  - 页面滚动行为
- **用途**：AI 分析学生的学习习惯和薄弱点

##### 错题收集（error_collection）
- **触发条件**：答错的题目自动加入错题本
- **存储规则**：
  - 按 `user_id + question_id` 唯一索引
  - 记录 `correct_count`（连续答对次数）
  - 达到 3 次正确后自动移除
- **巩固模式**：
  - 进入错题巩固模式时只显示错题
  - 再次答错重置 `correct_count = 0`
  - 答对则 `correct_count++`

##### 阅读理解支持
- **特殊题型**：阅读理解包含文章段落 + 多个小题
- **数据结构**：
  ```javascript
  {
    type: 'reading',
    content: '阅读材料...',
    passage: '完整的文章内容',
    questions: [
      { id: 1, type: 'single', options: [...], answer: 'A' },
      { id: 2, type: 'multiple', options: [...], answer: ['A', 'C'] }
    ]
  }
  ```
- **答题方式**：按小题逐一作答，最终统一批改

### 5. 统计分析系统

#### 5.1 学习进度（learning_progress）
- **存储表**：`learning_progress`
- **统计维度**：
  - 各学科完成率
  - 各题库掌握程度
  - 每日/每周学习时长
  - 连续学习天数
- **可视化**：进度条、环形图、趋势折线图

#### 5.2 学习报告（LearningReportView）
- **生成时机**：每次答题结束后自动生成
- **报告内容**：
  - 本次答题的正确率
  - 与历史平均对比
  - 薄弱点分析
  - 下一步学习建议
- **技术栈**：VChart 图表库 + VMind 智能图表推荐

#### 5.3 答题历史（AnswerHistoryView）
- **数据源**：`answer_records` + `question_attempts` 表
- **查询维度**：
  - 按时间范围筛选
  - 按学科/题库筛选
  - 按正确率排序
- **分页支持**：使用 `usePagination` composable

#### 5.4 排行榜（leaderboard）
- **存储表**：`leaderboard`（或从 users.points 聚合计算）
- **排名规则**：
  - 主要依据：总积分（points）
  - 辅助指标：答题总数、正确率
- **展示形式**：`LeaderboardTable.vue` 组件
- **刷新频率**：实时计算 + 缓存（5 分钟）

#### 5.5 难度自适应（difficulty/question_difficulty_history）
- **自适应算法**：
  ```
  初始难度 = 子分类默认难度
  ↓
  连续答对 3 题 → 难度 +1（最高 5）
  连续答错 2 题 → 难度 -1（最低 1）
  ```
- **记录表**：`question_difficulty_history`
  - 记录每次答题后的难度调整
  - 支持按时间段回溯
- **服务层**：`services/difficultyService.js`

#### 5.6 用户统计（user-stats）
- **API 端点**：`/api/user-stats/:userId`
- **统计内容**：
  - 总答题数、总正确率
  - 各学科分布
  - 活跃时段分析
  - 错题率 TOP10
- **组件**：`UserStatsView.vue`, `RecentRecordsView.vue`

#### 5.7 仪表盘（dashboard）
- **页面**：`DashboardView.vue`（管理员首页）
- **数据卡片**：
  - 总用户数、今日新增
  - 总题目数、今日新增
  - 今日答题次数、平均正确率
  - 系统负载、缓存命中率
- **图表区域**：
  - 近 7 天活跃用户趋势
  - 学科热度排行
  - 答题时段分布

#### 5.8 数据分析（analysis）
- **页面**：`DataAnalysis.vue` + `AnalysisCharts.vue`
- **分析维度**：
  - **整体概览**（AnalysisOverview）：关键指标汇总
  - **错误分析**（ErrorAnalysis）：高频错误知识点
  - **多维度交叉分析**：
    - 年级 × 学科 × 正确率
    - 时间段 × 答题量 × 正确率
    - 题型分布 × 掌握情况
- **图表类型**：
  - 柱状图、折线图、饼图、散点图、热力图
  - 使用 VChart 渲染，VMind 自动推荐最佳图表

#### 6.3 聊天功能

  id VARCHAR(36) PRIMARY KEY,   -- UUID

### 7. 安全系统

#### 7.1 认证与授权

##### JWT 双认证机制
- **学生认证**：
  - 登录成功后签发 JWT Token
  - Token 存储：localStorage（持久化）+ sessionStorage（活动状态）
  - 有效期：30 分钟
  - 续期机制：每次请求检查有效期，即将过期时自动刷新

- **管理员认证**：
  - 独立的认证体系（admin_credentials 表）
  - 使用 bcrypt 哈希密码（12 轮 salt）
  - 通过 `middleware/adminAuth.js` 鉴权
  - 所有 `/api/admin/*` 路由强制要求管理员权限

#### 7.2 CSRF 防护（csrf.js）
- **实现原理**：
  1. 服务端生成随机 CSRF Token（同步令牌模式）
  2. 通过 `/api/csrf-token` 接口下发
  3. 前端在所有 POST/PUT/DELETE 请求中携带
  4. 服务端验证 Token 有效性
- **Token 存储**：Session 或 Cookie（HttpOnly）
- **防护范围**：所有状态改变的操作

#### 7.3 三级限流（rateLimit.js）

| 级别 | 中间件名称 | 限制策略 | 适用场景 |
|------|------------|----------|----------|
| 全局限流 | globalLimiter | IP 维度，1000 次/分钟 | 所有 API 请求 |
| API 限流 | apiLimiter | IP 维度，200 次/分钟 | 普通 API 调用 |
| 提交限流 | submitLimiter | IP+用户维度，10 次/分钟 | 表单提交（答题、登录） |

- **持久化**：封禁数据存储到 MySQL（ip_blocks 表）
- **自动解封**：超时后自动解除封禁
