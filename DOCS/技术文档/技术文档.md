# PSCG 项目技术文档（重构版）

**文档版本**: v3.0.0  
**最后更新**: 2026-03-28  
**适用范围**: 开发/运维/安全/架构

## 1. 项目概览
- 前后端分离：Vue 3 + Vite 5 + Pinia + Vue Router 4；Express 4 + MySQL 8。
- 核心能力：题库管理、智能答题、难度自动/手动调整、答题记录与分析、排行榜、AI 自然语言与题目语义分析、学习风格/行为分析。
- 安全能力：JWT 管理员认证、答题签名+防重放、CSRF、防刷限流、AI 输入校验、文件引用计数、数据库性能监控。

## 2. 技术栈
- 前端：Vue 3.5.25、Pinia 3.0.4、Vue Router 4.6.4、Element Plus 2.13.3、VisActor (VChart/VMind) 2.x、Quill 2.0.3。
- 后端：Node.js 16+、Express 4.18.2、mysql2 3.20.0、multer 2.1.1、jsonwebtoken 9.0.3、memory-cache 0.2.0、compression/cors/dotenv。
- 工具/封装：`src/utils/api.js`（重试/防抖/超时/CSRF），`services/cache.js`（TTL 缓存），`middleware/rateLimit.js`、`middleware/signatureCache.js`。

## 4. 核心功能模块（对齐最新实现）
- **用户认证**：JWT，路由守卫；登录视图 `LoginView.vue`。
- **题库/题目管理**：`routes/subjects.js`、`routes/questions.js`，支持导入/文件引用计数。
- **智能答题**：`QuizView` + `questionStore`；后端 `quiz.js` 生成会话/签名；`answer-records.js` 存储记录并防重复提交。
- **难度调整**：`difficultyService` 加权错误率 + 阈值；`routes/difficulty.js` 自动/批量/手动、分布查询。
- **排行榜**：`leaderboard.js` + `LeaderboardView.vue`。
- **数据分析**：`analysis.js` 综合统计（用户/年级/班级/学科/时间区间），缓存与 XLSX 导出；前端分析组件/图表。
- **学习进度与报告**：`learning-progress.js`、`user-stats.js`；前端 `LearningProgressView`、`LearningReportView`、`AnswerHistoryView`、`RecentRecordsView`、`AnswerBehaviorTracker`。
- **AI 模块**：
  - 自然语言分析/历史：`ai.js` + `AIAnalysisView`。
  - 题目语义分析与相似题：`question-semantic.js` + `QuestionSemanticAnalysis.vue`，通过 AI 队列处理。
  - 学习风格/行为/错题原因分析：`answer-behavior.js`（任务入队），`user-stats.js` 提供报告数据。
- **安全**：JWT、答题签名+时间戳±5 分钟、防重放签名缓存、CSRF、限流、AI 输入校验、文件引用计数、数据库性能监控。

## 5. API 分层概览（详见 API 文档）
- 用户/学科/题目：`/api/users`、`/api/subjects`、`/api/questions`。
- 答题：`/api/quiz/start`、`/api/quiz/submit`、`/api/answer-records`。
- 难度：`/api/difficulty/*`（题目/题库自动/手动/批量/分布）。
- 分析/统计：`/api/analysis/*`（综合）、`/api/user-stats/*`（风格/行为/偏好/时段/错题倾向）。
- AI：`/api/ai/*`（自然语言/学习风格/数据增强/历史）、`/api/question-semantic/*`（语义分析队列/相似题/标签统计）。
- 安全与运维：`/api/security/*`（限流状态/封禁）、`/api/csrf-token`、`/api/admin/*`。

## 6. 数据库要点
- 使用连接池；参数化查询；避免 SELECT *。
- 文件引用：`fileReferenceService` 维护引用计数，清理孤儿文件。
- 学习/行为相关表：答题记录、学习进度、行为统计依赖队列/缓存，注意索引和批处理。

## 7. 性能与缓存
- 前端：路由懒加载、组件按需、图表按需、选项打乱、防抖 API 封装。
- 后端：`services/cache.js` 对 subjects/analysis/leaderboard/user-stats 等设 TTL；`dbPerformance.js` 监控慢查询；`compression` + 静态资源缓存；提交/分析接口开启限流。

## 8. 安全实践（摘要）
- 签名：HMAC-SHA256，时间戳 ±5 分钟，签名缓存防重放。
- CSRF：`middleware/csrf.js` + `api.js` 自动带 `X-CSRF-Token`。
- 限流：提交/全局/接口级限流，支持封禁与解封。
- AI 输入校验：限制批量大小/优先级，XSS 清洗。
- 上传：图片/音频白名单、大小限制，引用计数防孤儿。

## 9. 部署与维护（摘要）
- 环境变量：DB_*、JWT_SECRET、SESSION_SECRET、SIGNATURE_SECRET、PORT 等；前端 VITE_* 对齐签名密钥。
- 启动：`npm run build`（前端），`npm run server` 或 PM2 进程（server.cjs）。
- 反代：Nginx 转发 `/api` 到 3001，前端静态文件指向 `dist/`，开启缓存与 gzip，HTTPS 建议启用。
- 备份：保留数据库、配置、上传文件；定期清理旧备份。
