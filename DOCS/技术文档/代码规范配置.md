## 🚀 使用方法

### 检查代码规范
```bash
npm run lint:check
```

### 自动修复问题
```bash
npm run lint
```

### 格式化代码
```bash
npm run format
```

## 📝 主要规则

### Vue 规则
- ✅ 使用 `<script setup>` 语法
- ⚠️ 警告 `v-html` 使用（XSS 风险）
- ✅ 建议显式声明 `emits`

### TypeScript 规则
- ⚠️ 警告 `any` 类型使用
- ⚠️ 警告未使用的变量（`_` 开头的变量除外）
- ✅ 允许 `require`（后端代码）

### JavaScript 规则
- ✅ 使用 `const` 而非 `let`（如果变量不会重新赋值）
- ❌ 禁止使用 `var`
- ⚠️ 生产环境警告 `console.log`

### Prettier 格式化规则
- 不使用分号
- 使用单引号
- 缩进 2 个空格
- 行宽 100 字符
- 不使用尾逗号

## 🎯 最佳实践

1. **提交代码前运行**:
   ```bash
   npm run lint
   ```

2. **IDE 设置**:
   - 启用保存时自动格式化
   - 启用 ESLint 自动修复

3. **命名规范**:
   - 未使用的变量使用 `_` 前缀
   - 未使用的参数使用 `_` 前缀
   - catch 块中的 error 变量会被自动忽略

4. **后端代码**:
   - 使用 `.cjs` 扩展名
   - 可以使用 `require` 和 `module.exports`

5. **前端代码**:
   - 使用 ES Modules (`import`/`export`)
   - Vue 组件使用 `<script setup>`
