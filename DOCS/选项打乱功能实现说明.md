# 选项打乱功能实现说明

## 概述

为了防止学生作弊和记忆答案位置，实现了选项打乱功能。每次答题时，题目的选项顺序会被随机打乱，同时系统会记录打乱映射，确保答案验证的正确性。

## 实现方案

### 方案 2：正确实现选项打乱（已实现）

#### 核心原理
1. **前端打乱选项**：在加载题目时，前端对每道题的选项进行随机打乱
2. **存储打乱映射**：前端保存打乱映射关系（打乱后位置 → 原始位置）
3. **后端验证考虑映射**：提交答案时，后端根据映射将用户答案还原到原始位置进行验证

#### 1.2 QuizView.vue 修改

在 `onMounted` 钩子中，加载题目后立即打乱选项：

```javascript
const questionsWithShuffled = data.questions.map(q => {
  const options = typeof q.options === 'string' ? JSON.parse(q.options) : q.options;
  const { shuffledOptions, reverseMapping } = shuffleOptions(options);
  
  return {
    ...q,
    options: shuffledOptions,      // 打乱后的选项
    shuffleMapping: reverseMapping, // 打乱映射
    type: q.type || 'single'
  };
});
```

提交答案时，将打乱映射一起提交：


## 数据流程

### 答题流程

```
1. 前端请求题目
   ↓
2. 后端返回题目（原始选项顺序）
   ↓
3. 前端打乱选项，保存映射
   {
     options: ['C', 'A', 'D', 'B'],  // 打乱后
     shuffleMapping: {0: 2, 1: 0, 2: 3, 3: 1}
   }
   ↓
4. 用户选择答案（例如选择 A）
   ↓
5. 前端提交：
   - 用户答案: 'A'（打乱后的位置）
   - 打乱映射: {0: 2, 1: 0, 2: 3, 3: 1}
   ↓
6. 后端验证：
   - 将 'A' 映射回原始位置 'B'
   - 验证 'B' 是否正确
   ↓
7. 后端返回结果：
   - 正确答案: 'A'（映射到打乱后位置）
   - 选项: ['C', 'A', 'D', 'B']（打乱后）
   ↓
8. 前端显示结果（正确标记）
```

### 映射关系说明

**示例场景**：
- 原始选项: `['苹果', '香蕉', '橙子', '葡萄']` (A, B, C, D)
- 正确答案: B (香蕉)
- 打乱后: `['橙子', '苹果', '葡萄', '香蕉']` (A, B, C, D)

**映射关系**:
```javascript
shuffleMapping = {
  0: 2,  // 原始位置 0 (苹果) -> 打乱后位置 2
  1: 3,  // 原始位置 1 (香蕉) -> 打乱后位置 3
  2: 0,  // 原始位置 2 (橙子) -> 打乱后位置 0
  3: 1   // 原始位置 3 (葡萄) -> 打乱后位置 1
}

reverseMapping = {
  0: 2,  // 打乱后位置 0 (橙子) -> 原始位置 2
  1: 3,  // 打乱后位置 1 (葡萄) -> 原始位置 3
  2: 0,  // 打乱后位置 2 (苹果) -> 原始位置 0
  3: 1   // 打乱后位置 3 (香蕉) -> 原始位置 1
}
```

**用户选择示例**:
- 用户在打乱后的位置 D 选择 (实际选择了 '香蕉')
- 用户答案: 'D'
- 映射回原始位置: reverseMapping[3] = 1 -> 'B'
- 验证: 'B' == 'B' ✓ 正确

**返回结果示例**:
- 原始正确答案: 'B'
- 映射到打乱后位置: shuffleMapping[1] = 3 -> 'D'
- 返回给前端: 'D'
- 前端显示: 在位置 D 标记正确答案

## 注意事项

1. **安全性**：
   - 打乱映射由前端生成并提交，后端需要验证映射的合法性
   - 可以考虑在后端也生成一份映射进行对比

2. **兼容性**：
   - 如果前端没有提交打乱映射，后端使用原始选项顺序
   - 旧数据不受影响

3. **性能**：
   - 打乱算法使用 Fisher-Yates，时间复杂度 O(n)
   - 映射操作简单，不影响性能

4. **多选题**：
   - 多选题的答案也需要正确映射
   - 支持数组格式和字符串格式（如 "ABC"）
