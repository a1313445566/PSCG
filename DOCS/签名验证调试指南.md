# 签名验证调试指南

## 概述

签名验证系统提供了详细的日志输出，帮助你快速定位问题。所有日志都经过安全处理，不会泄露敏感信息。

## 日志安全机制

### 信息脱敏

所有敏感信息在日志中都会被脱敏处理：

```
原始签名: adefab16ed768a6be0dba85714e235735fd02bf0385a38e5d26384e1723a2065...
脱敏后:   adefab16***1723a2065
```

**脱敏规则**：
- ✅ 显示签名的前 8 位和后 8 位
- ✅ 中间部分用 `***` 替代
- ❌ 不显示完整签名
- ❌ 不显示签名密钥
- ❌ 不显示用户答案内容

### 环境区分

日志输出根据环境自动调整详细程度：

**开发环境**：输出详细信息
```javascript
{
  quizId: "xxx",
  userId: "123",
  answerCount: 10,
  signature: "adefab16***1723a2065",
  method: "HMAC-SHA256"
}
```

**生产环境**：只输出关键信息
```javascript
{
  quizId: "xxx",
  userId: "123",
  answerCount: 10,
  method: "HMAC-SHA256"
}
```

## 常见问题排查

### 问题 1: 签名不匹配

**前端日志**：
```
[签名生成] ✅ HMAC-SHA256 签名生成成功
  signature: "adefab16***1723a2065"
```

**后端日志**：
```
[签名验证] ❌ 验证失败: 签名不匹配
  received: "adefab16***1723a2065"
  expectedHmac: "adefab16***different"
```

**原因**：前后端签名密钥不一致

**解决方案**：
1. 检查 `.env` 文件中 `SIGNATURE_SECRET` 和 `VITE_SIGNATURE_SECRET` 是否相同
2. 重新构建前端：`npm run build`
3. 重启后端：`pm2 restart PSCG`

### 问题 2: 时间戳过期

**后端日志**：
```
[签名验证] ❌ 验证失败: 时间戳超出范围
  timeDiff: '600秒'
  maxAllowed: '300秒'
```

**原因**：客户端时间与服务器时间相差超过 5 分钟

**解决方案**：
1. 检查客户端系统时间是否正确
2. 检查时区设置是否正确
3. 确保使用的是 `Date.now()` 而不是其他时间戳

### 问题 3: 使用不同的签名方法

**前端日志**：
```
[签名生成] ✅ HMAC-SHA256 签名生成成功
  method: "HMAC-SHA256"
```

**后端日志**：
```
[签名验证] ✅ 签名验证成功
  method: '降级签名'
```

**原因**：
- 前端：HTTPS 环境，使用 HMAC-SHA256
- 后端：验证时同时检查两种方法

**说明**：这是正常的，后端会同时验证两种签名方法

### 问题 4: 缺少必要参数

**后端日志**：
```
[签名验证] ❌ 验证失败: 缺少必要参数 {
  hasData: true,
  hasTimestamp: true,
  hasSignature: false,
  hasUserId: true
}
```

**原因**：请求中缺少 `signature` 字段

**解决方案**：
1. 检查前端是否正确生成签名
2. 检查网络请求是否包含 `signature` 字段
3. 查看前端控制台的错误信息

## 调试步骤

### 步骤 1: 查看前端日志

1. 打开浏览器开发者工具（F12）
2. 切换到 Console 标签
3. 清除现有日志
4. 提交答案
5. 查找 `[签名生成]` 开头的日志

### 步骤 2: 查看后端日志

```bash
# 实时查看日志
pm2 logs PSCG --lines 100

# 或查看文件
tail -f /root/.pm2/logs/PSCG-out.log
```

### 步骤 3: 对比前后端日志

**前端签名**：
```
signature: "adefab16***1723a2065"
method: "HMAC-SHA256"
```

**后端验证**：
```
received: "adefab16***1723a2065"
method: "HMAC-SHA256"
```

如果签名脱敏后的前后部分相同，说明签名一致。

### 步骤 4: 检查环境变量

```bash
# 查看当前环境变量
cat .env | grep SIGNATURE_SECRET

# 应该看到两个相同的密钥
SIGNATURE_SECRET=adefab16...
VITE_SIGNATURE_SECRET=adefab16...
```

## 安全建议

### ✅ 已实现的安全措施

1. **签名脱敏**：只显示前后各 8 位
2. **密钥不输出**：签名密钥永远不会出现在日志中
3. **环境区分**：生产环境减少日志输出
4. **答案不输出**：用户答案内容不会出现在日志中

### ⚠️ 注意事项

1. **不要在日志中添加完整签名**
2. **不要输出签名密钥**
3. **不要记录用户的答案内容**
4. **定期清理旧日志文件**
