# TRAE 后端开发规则
## 1. 架构规范
- 路由(routes) → 中间件 → 服务(services) → 数据库
- 所有接口统一异常捕获，返回标准格式
- 文件上传用Multer，图片压缩用Sharp

## 2. 数据库规则
- 语法：MySQL标准（AUTO_INCREMENT、VARCHAR、INT）
- 所有写操作加事务，大数据批量操作分批执行
- 禁止联表过深，禁止无索引条件查询
- 连接池复用，操作后正确关闭连接

## 3. 安全&大数据规则
- 管理员接口必须走adminAuth中间件
- 所有入参校验，防SQL注入/XSS
- AI接口启用速率限制，防止过载
- 日志/性能监控必须集成
