# 部署工具使用说明

## 📁 文件清单

```
scripts/
├── pre-deploy-check.ps1    # 本地部署前检查脚本 (Windows PowerShell)
├── deploy-aitest.sh         # 服务器部署脚本 (Ubuntu)
└── rollback-to-master.sh    # 服务器回滚脚本 (Ubuntu)

DOCS/
└── 部署-AItest分支上线指南.md  # 完整部署文档
```

---

## 🚀 快速开始

### 第一步：本地检查（Windows）

在本地开发环境执行部署前检查：

```powershell
# 在项目根目录执行
.\scripts\pre-deploy-check.ps1
```

**检查内容**：
- ✅ Git 分支状态
- ✅ 未提交的代码
- ✅ 依赖包变更
- ✅ 数据库结构变更
- ✅ 环境变量变更
- ✅ 代码质量（ESLint）
- ✅ 前端构建测试
- ✅ 关键文件完整性
- ✅ 敏感信息检查

**检查结果**：
- ✅ 全部通过 → 可以安全部署
- ⚠️ 有警告 → 建议处理后再部署
- ❌ 有失败 → 必须修复后才能部署

---

### 第二步：上传脚本到服务器

将部署脚本上传到 Ubuntu 服务器：

```bash
# 方法1: 使用 SCP 上传
scp scripts/deploy-aitest.sh user@your-server:/tmp/
scp scripts/rollback-to-master.sh user@your-server:/tmp/

# 方法2: 使用 Git Clone（推荐）
# 在服务器上执行
git clone https://github.com/your-repo/PSCG.git
cd PSCG
```

---

### 第三步：修改脚本配置

编辑部署脚本，修改配置项：

```bash
# 编辑 deploy-aitest.sh
nano deploy-aitest.sh

# 修改以下配置：
PROJECT_DIR="/var/www/PSCG"    # 改为你的实际项目路径
PROJECT_NAME="PSCG"             # PM2 应用名称
DB_NAME="pscg_db"               # 数据库名称
BACKUP_DIR="$HOME/backups"      # 备份目录
```

同样修改 `rollback-to-master.sh`。

---

### 第四步：执行部署

#### 方式1：一键部署（推荐）

```bash
# SSH 登录服务器
ssh user@your-server

# 进入项目目录
cd /var/www/PSCG

# 赋予执行权限
chmod +x scripts/deploy-aitest.sh
chmod +x scripts/rollback-to-master.sh

# 执行部署
sudo bash scripts/deploy-aitest.sh
```

#### 方式2：手动部署

如果需要手动控制每一步，参考详细文档：

```bash
# 查看完整部署文档
cat DOCS/部署-AItest分支上线指南.md
```

---

### 第五步：部署验证

部署完成后，验证服务状态：

```bash
# 检查 PM2 状态
pm2 list

# 查看实时日志
pm2 logs PSCG

# 健康检查
curl http://localhost:3001/api/health

# 性能监控
pm2 monit
```

在浏览器中访问：
- 首页：http://your-server:3001/
- 健康检查：http://your-server:3001/api/health
- API 工具：http://your-server:3001/api/tools

## 🛠️ 常见问题

### 1. 本地检查脚本报错

**问题**：执行 `pre-deploy-check.ps1` 时报错"无法加载文件"

**解决**：
```powershell
# 临时允许脚本执行
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

# 然后执行脚本
.\scripts\pre-deploy-check.ps1
```

---

### 2. 服务器部署脚本报错"Permission denied"

**问题**：执行脚本时提示权限不足

**解决**：
```bash
# 赋予执行权限
chmod +x scripts/deploy-aitest.sh

# 使用 sudo 执行
sudo bash scripts/deploy-aitest.sh
```

---

### 3. 数据库备份失败

**问题**：mysqldump 提示权限不足

**解决**：
```bash
# 使用 root 用户备份
mysqldump -u root -p --single-transaction pscg_db > backup.sql

# 或在脚本中使用 sudo
sudo mysqldump -u root --single-transaction pscg_db > backup.sql
```

---

### 4. PM2 启动失败

**问题**：服务启动后立即退出

**解决**：
```bash
# 查看 PM2 日志
pm2 logs PSCG --lines 100

# 查看 PM2 配置
pm2 show PSCG

# 手动启动测试
node server.cjs

# 如果是端口占用
netstat -tlnp | grep 3001
kill -9 <PID>
pm2 restart PSCG
```

---

### 5. 前端构建失败

**问题**：`npm run build` 报错

**解决**：
```bash
# 清除缓存
rm -rf node_modules/.vite
rm -rf dist

# 重新安装依赖
npm install

# 重新构建
npm run build
```
