# 环境配置说明

## 配置文件

项目使用单一的 `.env` 文件管理所有环境变量。

### 文件位置

```
PSCG/
└── .env          # 环境配置文件（不提交到 Git）
```

### 配置项说明

```bash
# 服务器配置
SERVER_PORT=3001                          # 后端服务器端口

# API 地址（前端使用）
VITE_API_URL=http://localhost:3001/api     # 前端 API 请求地址

# 签名密钥（用于答题提交验证）
SIGNATURE_SECRET=your-secret-key-here     # 后端签名密钥
VITE_SIGNATURE_SECRET=your-secret-key-here # 前端签名密钥（与后端相同）

# 数据库配置
DB_HOST=127.0.0.1                         # 数据库主机
DB_PORT=3306                              # 数据库端口
DB_USER=pscg                              # 数据库用户名
DB_PASSWORD=your-password                 # 数据库密码
DB_NAME=pscg                              # 数据库名称
```

## 使用方法

### 开发环境

1. 复制示例配置：
```bash
cp .env.example .env
```

2. 修改 `.env` 中的配置项：
```bash
VITE_API_URL=http://localhost:3001/api
```

3. 启动开发服务器：
```bash
npm run dev      # 前端开发服务器
npm run server   # 后端开发服务器
```

### 生产环境

1. 修改 `.env` 中的生产配置：
```bash
VITE_API_URL=http://your-server-ip:3001/api
```

2. 构建并部署：
```bash
npm run build          # 构建前端
pm2 restart PSCG       # 重启后端服务
```

## 安全注意事项

### ⚠️ 重要安全规则

1. **永远不要提交 `.env` 到 Git**
   - `.env` 已添加到 `.gitignore`
   - 敏感信息不会被版本控制

2. **签名密钥管理**
   - 使用强密钥（建议 64 字节以上）
   - 定期轮换密钥（每 3-6 个月）
   - 不要在不同环境使用相同密钥

3. **数据库密码**
   - 使用强密码
   - 定期更换密码
   - 限制数据库访问权限

### 生成新的签名密钥

```bash
# 使用 Node.js 生成 64 字节密钥
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```

更新 `.env` 文件中的签名密钥后，需要：

1. **重新构建前端**：
```bash
npm run build
```

2. **重启后端服务**：
```bash
pm2 restart PSCG
```

## 环境变量加载机制

### 前端（Vite）

- 开发模式（`npm run dev`）：自动加载 `.env`
- 生产构建（`npm run build`）：自动加载 `.env`
- 访问方式：`import.meta.env.VITE_*`

### 后端（Node.js）

- 使用 `dotenv` 加载 `.env`
- 访问方式：`process.env.*`

## 故障排查

### 签名验证失败

**症状**：提交答案时返回 401 错误

**原因**：前后端签名密钥不一致

**解决方案**：
1. 检查 `.env` 文件中 `SIGNATURE_SECRET` 和 `VITE_SIGNATURE_SECRET` 是否相同
2. 重新构建前端：`npm run build`
3. 重启后端：`pm2 restart PSCG`

### API 请求失败

**症状**：前端无法连接到后端

**原因**：`VITE_API_URL` 配置错误

**解决方案**：
1. 检查 `.env` 中的 `VITE_API_URL` 是否正确
2. 确保后端服务正在运行：`pm2 status`
3. 重新构建前端：`npm run build`

## 配置最佳实践

1. **开发与生产分离**
   - 开发时使用 `localhost`
   - 生产时使用实际服务器 IP 或域名

2. **密钥管理**
   - 使用环境变量注入（CI/CD）
   - 使用密钥管理服务（如 Vault）
   - 避免在代码中硬编码

3. **配置验证**
   - 后端启动时检查必要的环境变量
   - 前端构建时验证 `VITE_*` 变量

## 示例配置文件

创建 `.env.example` 作为配置模板：

```bash
# 服务器配置
SERVER_PORT=3001

# API 地址（前端使用）
VITE_API_URL=http://localhost:3001/api

# 签名密钥（请替换为实际密钥）
SIGNATURE_SECRET=your-secret-key-here
VITE_SIGNATURE_SECRET=your-secret-key-here

# 数据库配置
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=pscg
DB_PASSWORD=your-password
DB_NAME=pscg
```

团队成员可以复制此文件并修改：
```bash
cp .env.example .env
```
