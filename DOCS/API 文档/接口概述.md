# PSCG 教育系统 API 接口概述

> 文档版本：v1.0 | 最后更新：2026-04-05
> 基础 URL：`http://localhost:3001`

## 一、基础信息

### 1.1 API 前缀

所有业务接口均以 `/api` 为前缀，例如：

```
http://localhost:3001/api/questions
http://localhost:3001/api/users/login
http://localhost:3001/admin/login
```

### 1.2 技术栈

| 组件 | 技术选型 |
|------|----------|
| 后端框架 | Express.js |
| 数据库 | MySQL (mysql2) |
| 认证方式 | JWT (jsonwebtoken) |
| 文件上传 | Multer + Sharp |
| 数据分析 | ExcelJS |
| AI 集成 | OpenAI API (流式 SSE) |
| 缓存服务 | 自研内存缓存 |

## 三、中间件链

所有 API 请求按以下顺序经过中间件处理：

```
请求 → cors → compression → json/urlencoded → responseTime → apiLimiter(全局限流) → csrfVerify(CSRF验证) → 业务路由
```

### 3.1 各中间件说明

| 中间件 | 作用域 | 说明 |
|--------|--------|------|
| **cors** | 全局 | 允许跨域，支持 GET/POST/PUT/DELETE/OPTIONS |
| **compression** | 全局 | 响应压缩，减少传输体积 |
| **express.json** | 全局 | JSON 请求体解析，限制 10MB |
| **responseTime** | 全局 | 响应时间监控中间件 |
| **apiLimiter** | `/api/*` | 全局 API 速率限制 |
| **csrfVerify** | `/api/*` POST/PUT/DELETE | CSRF 令牌验证 |
| **adminAuth** | 特定路由 | 管理员 JWT 验证中间件 |
| **submitLimiter** | `/api/quiz/submit` | 提交答案专用限流器 |

## 四、认证机制

### 4.1 双体系 JWT 认证

系统采用 **学生 JWT + 管理员 JWT** 双轨制：

#### 学生认证（users.js）

```javascript
// JWT 密钥
const JWT_SECRET = 'your-secret-key'
const JWT_EXPIRES_IN = '24h' // 24 小时过期

// Token Payload 结构
{
  userId: number,      // 用户 ID
  studentId: string    // 学号
}
```

**登录接口：** `POST /api/users/login`

**使用方式：**
- 学生登录后获得 Token
- 后续请求通过 `Authorization: Bearer <token>` 头部传递（部分接口无需 Token）

#### 管理员认证（admin.js）

```javascript
// JWT 密钥（独立于学生密钥）
const JWT_SECRET = process.env.JWT_SECRET || 'pscg-admin-secret-key-change-in-production'
const JWT_EXPIRES_IN = '24h'

// Token Payload 结构
{
  id: number,          // 管理员 ID
  username: string,    // 用户名
  isAdmin: true        // 标识为管理员
}
```

**登录接口：** `POST /api/admin/login`

**使用方式：**
- 管理员登录后获得 Token
- 所有需要管理员权限的接口必须携带此 Token
- 通过 `Authorization: Bearer <token>` 头部传递

### 4.2 adminAuth 中间件

用于保护管理员专用接口的路由守卫：

```javascript
// 使用示例
router.get('/models', adminAuth, async (req, res) => {
  // req.admin 包含解码后的管理员信息
  const adminId = req.admin.id
})
```

**保护的接口包括：**
- `/api/chat/*` - 所有聊天相关接口
- `/api/dashboard/*` - 数据看板接口
- `/api/security/*` - 安全管理接口
- `/api/answer-behavior/statistics/:userId` - 行为统计
- `/api/answer-behavior/user-style/:userId` - 学习风格

## 五、统一响应格式

### 5.1 成功响应

```json
{
  "success": true,
  "data": { ... },
  "message": "操作成功"
}
```

或直接返回数据：

```json
{
  "id": 1,
  "name": "张三",
  "grade": "3年级"
}
```

### 5.2 分页响应

```json
{
  "data": [ ... ],
  "total": 100,
  "page": 1,
  "limit": 20,
  "totalPages": 5
}
```

### 5.3 错误响应

```json
{
  "error": "错误描述信息",
  "details": "详细错误信息（可选）"
}
```

### 5.4 HTTP 状态码规范

| 状态码 | 含义 | 使用场景 |
|--------|------|----------|
| 200 | 成功 | 正常响应 |
| 400 | 请求错误 | 参数缺失、格式错误、验证失败 |
| 401 | 未认证 | Token 无效/过期、签名验证失败 |
| 403 | 无权限 | 权限不足 |
| 404 | 未找到 | 资源不存在 |
| 500 | 服务器错误 | 内部异常 |

## 六、CSRF 防护流程

系统对所有 POST/PUT/DELETE 请求实施 CSRF 保护：

### 6.1 获取 Token

```
GET /api/csrf-token
→ Response: { success: true, csrfToken: "xxxxx" }
```

### 6.2 提交请求时携带

前端需在请求头中添加：

```
X-CSRF-Token: <csrf_token>
```

或在请求体中包含 `csrfToken` 字段。

## 七、速率限制说明

系统实现三级速率限制：

| 限流器 | 作用域 | 默认限制 | 说明 |
|--------|--------|----------|------|
| **globalLimiter** | 全局 | 按 IP 限制 | 防止 DDoS 攻击 |
| **apiLimiter** | `/api/*` | API 级别限制 | 保护所有 API 接口 |
| **submitLimiter** | `/api/quiz/submit` | 答题提交专用 | 最严格的限制，防止刷分 |

### 限流状态查询

管理员可通过以下接口查看限流状态：

```
GET /api/security/rate-limit   (需管理员认证)
```

### IP 封禁功能

支持手动封禁/解封 IP：

```
POST /api/security/block-ip     # 封禁单个 IP
POST /api/security/unblock-ip   # 解封单个 IP
POST /api/security/unblock-all  # 解除所有封禁
POST /api/security/batch-block  # 批量封禁
```

## 八、签名验证机制

答题提交等敏感操作采用 HMAC-SHA256 签名验证：

### 8.1 签名生成规则

```javascript
const dataStr = JSON.stringify(data) + timestamp + userId
const signature = crypto.createHmac('sha256', secret).update(dataStr).digest('hex')
```

### 8.2 时间戳有效性

- 签名时间戳必须在 **5 分钟内** 有效
- 超出范围视为过期请求

### 8.3 降级方案

非 HTTPS 环境下自动降级为多轮哈希算法，确保安全性。

### 8.4 重放攻击防护

- 签名使用后立即标记为已用
- 同一签名无法重复提交

## 九、缓存策略

系统在多处使用内存缓存提升性能：

| 缓存键 | 过期时间 | 说明 |
|--------|----------|------|
| subjects | 默认 | 学科列表（含子分类） |
| subjects_stats | 1 小时 | 学科题目数量统计 |
| grades | 默认 | 年级列表 |
| classes | 默认 | 班级列表 |
| leaderboard_* | 可配置 | 排行榜数据（滑动过期） |
| analysis_* | 可配置 | 分析数据 |
| error_prone_* | 可配置 | 错题率高的题目 |

### 缓存管理接口

```
GET  /api/cache/stats   # 查看缓存统计
POST /api/cache/clear   # 清空所有缓存
```

## 十一、开发规范提醒

1. **参数校验**：所有入参必须后端二次校验，防 SQL 注入/XSS
2. **分页限制**：单页最大 100 条记录
3. **批量操作**：单次最多 100 条
4. **文件上传**：图片 2MB，音频 10MB
5. **XSS 防护**：题目内容自动过滤危险标签
6. **Base64 图片**：自动转换为文件存储，防止数据库膨胀
7. **事务安全**：关键操作使用数据库事务保证一致性
