# 认证 API 文档

## 基础信息

- **基础路径**: `/api/admin` (管理员) 和 `/api/users` (学生)
- **认证方式**: JWT Token
- **响应格式**: JSON

---

## 管理员认证

### 1. 检查管理员初始化状态

**请求**
```
GET /api/admin/status
```

**响应示例**
```json
{
  "initialized": true,
  "username": "admin"
}
```

### 2. 初始化管理员（首次使用）

**请求**
```
POST /api/admin/init
```

**请求体**
```json
{
  "username": "admin",
  "password": "123456"
}
```

**响应示例**
```json
{
  "success": true,
  "message": "管理员初始化成功"
}
```

**错误响应**
```json
{
  "error": "管理员已存在，无法重复初始化"
}
```

### 3. 管理员登录

**请求**
```
POST /api/admin/login
```

**请求体**
```json
{
  "username": "admin",
  "password": "123456"
}
```

**响应示例**
```json
{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "admin": {
    "id": 1,
    "username": "admin"
  }
}
```

**错误响应**
```json
{
  "error": "用户名或密码错误"
}
```

### 4. 修改管理员密码

**请求**
```
PUT /api/admin/password
```

**认证**: 需要管理员 Token

**请求体**
```json
{
  "oldPassword": "123456",
  "newPassword": "newpassword123"
}
```

**响应示例**
```json
{
  "success": true,
  "message": "密码修改成功"
}
```

## 学生用户 API

### 1. 获取用户列表（管理员）

**请求**
```
GET /api/users
```

**认证**: 需要管理员 Token

**查询参数**
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| grade | string | 否 | - | 年级 |
| class | string | 否 | - | 班级 |
| student_id | string | 否 | - | 学号（模糊搜索） |
| name | string | 否 | - | 姓名（模糊搜索） |
| page | number | 否 | 1 | 页码 |
| limit | number | 否 | 20 | 每页数量 |
| withStats | boolean | 否 | false | 是否包含统计数据 |

**响应示例**
```json
{
  "data": [
    {
      "id": 1,
      "student_id": "2024001",
      "name": "张三",
      "grade": "六年级",
      "class": "1 班",
      "points": 100,
      "created_at": "2024-01-01T00:00:00.000Z",
      "total_sessions": 10,
      "avg_accuracy": 85.5
    }
  ],
  "total": 100,
  "page": 1,
  "limit": 20,
  "totalPages": 5
}
```

## 学生登录

学生登录通过 `/api/users` 接口实现，使用学号、年级、班级进行验证。

**实际登录逻辑**:
- 学生在前端输入学号、年级、班级
- 后端查询 `users` 表匹配记录
- 生成 JWT Token 返回给前端
- Token 存储在 localStorage 中，有效期 24 小时

**Token 使用**:
- 前端请求时在 Header 中添加：`Authorization: Bearer {token}`
- 管理员 Token 存储在 sessionStorage 中
- 学生 Token 存储在 localStorage 中

---

## JWT Token 说明

### Token 结构
```javascript
{
  "id": 1,              // 用户/管理员 ID
  "username": "admin",  // 用户名
  "isAdmin": true,      // 是否为管理员
  "iat": 1234567890,    // 签发时间
  "exp": 1234654290     // 过期时间
}
```

### Token 配置
- **密钥**: `JWT_SECRET` (环境变量或默认值)
- **有效期**: 24 小时 (`JWT_EXPIRES_IN = '24h'`)
- **算法**: HS256

### 使用示例
```javascript
// 前端添加 Token 到请求头
const token = localStorage.getItem('token')
fetch('/api/questions', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
})
```

## 注意事项

1. **管理员初始化**: 首次使用需要通过 `/api/admin/init` 创建管理员账号
2. **密码安全**: 密码使用 bcrypt 加密存储，不保存明文
3. **Token 安全**: Token 存储在客户端，注意 XSS 防护
4. **会话超时**: 前端需处理 Token 过期，跳转到登录页
5. **权限控制**: 管理员接口需要 `adminAuth` 中间件验证
