## 一、学生认证接口

### 1.1 学生登录

- **方法**: POST
- **路径**: `/api/users/login`
- **认证**: 无需
- **说明**: 学生登录或自动注册。如果用户不存在则自动创建新用户，已存在则更新姓名信息。

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| studentId | string | 是 | 学号（2位数字） |
| name | string | 否 | 姓名（1-4个中文字符） |
| grade | string | 是 | 年级（如"3年级"） |
| class | string | 是 | 班级（如"2班"） |

**请求示例**:
```json
{
  "studentId": "01",
  "name": "张三",
  "grade": "3年级",
  "class": "2班"
}
```

**响应示例** (200):
```json
{
  "userId": 1,
  "studentId": "01",
  "name": "张三",
  "grade": "3年级",
  "class": "2班",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expiresIn": "24h"
}
```

**错误响应** (400):
```json
{ "error": "学号不能为空" }
{ "error": "学号只能输入2位数字" }
{ "error": "姓名只能输入1-4个中文字符" }
```

---

### 1.2 JWT Token 结构

学生登录成功后返回的 JWT Token 包含以下信息：

```javascript
// Payload
{
  userId: number,      // 用户 ID（数据库主键）
  studentId: string    // 学号
}

// Header
{
  alg: "HS256",
  typ: "JWT"
}
```

**Token 配置**:
- 密钥: `your-secret-key`
- 过期时间: `24h`（24小时）
- 签名算法: HS256

## 二、管理员认证接口

### 2.1 检查初始化状态

- **方法**: GET
- **路径**: `/api/admin/status`
- **认证**: 无需
- **说明**: 检查系统是否已完成管理员初始化

**响应示例** (200):
```json
{
  "initialized": true,
  "username": "admin"
}
```

或未初始化时：
```json
{
  "initialized": false,
  "username": null
}
```

---

### 2.2 初始化管理员（首次设置）

- **方法**: POST
- **路径**: `/api/admin/init`
- **认证**: 无需
- **说明**: 首次使用时创建管理员账户，仅当不存在管理员时可调用

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码（至少6位） |

**请求示例**:
```json
{
  "username": "admin",
  "password": "123456"
}
```

**响应示例** (200):
```json
{ "success": true, "message": "管理员初始化成功" }
```

**错误响应**:
```json
{ "error": "管理员已存在，无法重复初始化" }   // 400
{ "error": "用户名和密码不能为空" }             // 400
{ "error": "密码长度不能少于6位" }               // 400
```

---

### 2.3 管理员登录

- **方法**: POST
- **路径**: `/api/admin/login`
- **认证**: 无需
- **说明**: 管理员登录验证身份密码

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

**请求示例**:
```json
{
  "username": "admin",
  "password": "123456"
}
```

**响应示例** (200):
```json
{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "username": "admin",
  "expiresIn": "24h"
}
```

**错误响应** (401):
```json
{ "error": "用户名或密码错误" }
```

---

### 2.4 管理员 Token 结构

管理员登录成功后返回的 JWT Token 包含以下信息：

```javascript
// Payload
{
  id: number,          // 管理员 ID
  username: string,    // 用户名
  isAdmin: true        // 管理员标识
}

// 配置
密钥: process.env.JWT_SECRET || 'pscg-admin-secret-key-change-in-production'
过期时间: '24h'
签名算法: HS256
```

---

### 2.5 验证 Token 有效性

- **方法**: GET
- **路径**: `/api/admin/verify`
- **认证**: 需要 Bearer Token
- **说明**: 前端用于检查登录状态是否有效

**请求头**:
```
Authorization: Bearer <admin_token>
```

**响应示例** (200):
```json
{
  "valid": true,
  "username": "admin",
  "isAdmin": true
}
```

**错误响应** (401):
```json
{ "valid": false, "error": "未登录" }
{ "valid": false, "error": "登录已过期" }
```

---

### 2.6 修改密码

- **方法**: POST
- **路径**: `/api/admin/change-password`
- **认证**: 需要 Bearer Token
- **说明**: 管理员修改自己的登录密码

**请求头**:
```
Authorization: Bearer <admin_token>
```

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| oldPassword | string | 是 | 当前密码 |
| newPassword | string | 是 | 新密码（至少6位） |

**请求示例**:
```json
{
  "oldPassword": "123456",
  "newPassword": "newpass123"
}
```

**响应示例** (200):
```json
{ "success": true, "message": "密码修改成功" }
```

**错误响应**:
```json
{ "error": "未登录" }                              // 401
{ "error": "登录已过期，请重新登录" }                // 401
{ "error": "旧密码和新密码不能为空" }                 // 400
{ "error": "新密码长度不能少于6位" }                  // 400
{ "error": "旧密码错误" }                            // 400
{ "error": "管理员不存在" }                          // 404
```

---

### 2.7 数据库管理二次验证

- **方法**: POST
- **路径**: `/api/admin/verify-data-management`
- **认证**: 需要 Bearer Token
- **说明**: 执行危险操作（如数据清空、恢复）前的二次密码确认

**请求头**:
```
Authorization: Bearer <admin_token>
```

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| password | string | 是 | 当前密码 |

**响应示例** (200):
```json
{ "valid": true, "message": "验证成功" }
```

**错误响应**:
```json
{ "valid": false, "error": "未登录" }       // 401
{ "valid": false, "error": "请输入密码" }     // 400
{ "valid": false, "error": "密码错误" }      // 400
{ "valid": false, "error": "管理员不存在" }  // 404
```

## 三、认证流程图

### 3.1 学生登录流程

```
前端 → POST /api/users/login {studentId, name, grade, class}
     ↓
后端验证学号格式（2位数字）
     ↓
查询数据库：WHERE student_id=? AND grade=? AND class=?
     ↓
┌─ 用户不存在 → 自动创建 → 返回 userId + token
└─ 用户已存在 → 更新姓名(如有) → 返回 userId + token
```

### 3.2 管理员登录流程

```
前端 → POST /api/admin/login {username, password}
     ↓
后端查询 admin_credentials 表
     ↓
bcrypt 验证密码哈希
     ↓
生成 JWT Token（含 id, username, isAdmin）
     ↓
返回 token + expiresIn
```

---

## 四、安全注意事项

1. **密码存储**: 使用 bcrypt 哈希存储，明文永不落库
2. **JWT 密钥**: 生产环境必须通过环境变量 `JWT_SECRET` 配置
3. **Token 过期**: 固定 24 小时有效期，过期后需重新登录
4. **敏感操作**: 数据库管理操作需要二次密码验证
5. **初始化保护**: 管理员初始化仅可执行一次，防止覆盖
