## 一、答题流程接口 (quiz.js)

### 1.1 开始答题

- **方法**: POST
- **路径**: `/api/quiz/start`
- **认证**: 无需（通过 studentId + grade + class 标识用户）
- **说明**: 创建答题会话，随机抽取题目。支持普通题库和错题巩固题库两种模式。

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| subjectId | number | 是 | 学科 ID |
| subcategoryId | string/number | 是 | 子分类 ID，或 `"error-collection"` 表示错题巩固 |
| questionCount | number | 否 | 题目数量（默认10） |
| studentId | string | 是 | 学号 |
| grade | string | 是 | 年级 |
| class | string | 是 | 班级 |

**请求示例**:
```json
{
  "subjectId": 1,
  "subcategoryId": 1,
  "questionCount": 10,
  "studentId": "01",
  "grade": "3年级",
  "class": "2班"
}
```

**响应示例** (200):
```json
{
  "quizId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "questions": [
    {
      "id": 101,
      "subject_id": 1,
      "subcategory_id": 1,
      "content": "<p>1+1=?</p>",
      "type": "single",
      "options": ["A. 1", "B. 2", "C. 3", "D. 4"],
      "explanation": "基础加法",
      "audio_url": null,
      "image_url": null,
      "difficulty": 1
    }
  ],
  "expiresAt": "2026-04-05T01:30:00.000Z"
}
```

**错题巩固模式额外字段** (`subcategoryId: "error-collection"`):
```json
{
  "quizId": "...",
  "questions": [...],
  "expiresAt": "...",
  "stats": {
    "101": { "correctCount": 1 },
    "102": { "correctCount": 0 }
  }
}
```

**错误响应**:
```json
{ "error": "缺少必填参数" }                          // 400
{ "error": "学科ID格式错误" }                         // 400
{ "error": "用户不存在" }                             // 404
{ "error": "该题库暂无题目" }                           // 404
{ "error": "错题巩固题库为空" }                         // 404
```

### 1.2 提交答案

- **方法**: POST
- **路径**: `/api/quiz/submit`
- **认证**: 无需（需签名验证）
- **说明**: 提交答题结果。包含完整的判题逻辑、积分计算、错题处理、学习进度更新。

**安全特性**:
- ✅ 签名验证（HMAC-SHA256 或降级算法）
- ✅ 时间戳有效性检查（5分钟内）
- ✅ 防重放攻击（签名使用后标记）
- ✅ 严格限流保护（submitLimiter）

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| quizId | string | 是 | 答题会话 ID（由 start 接口返回） |
| answers | object | 是 | 用户答案对象 `{questionId: answer}` |
| shuffleMappings | object | 是 | 选项打乱映射 `{questionId: reverseMapping}` |
| timestamp | number | 是 | 当前时间戳（毫秒） |
| signature | string | 是 | HMAC-SHA256 签名 |
| timeSpent | number | 否 | 总用时（秒） |

**answers 格式示例**:
```json
{
  "101": "B",
  "102": ["A", "C"],
  "103": {
    "0": "A",
    "1": "B",
    "2": "C"
  }
}
```

**shuffleMappings 格式**:
```json
{
  "101": { "0": 2, "1": 0, "2": 3, "3": 1 },
  "102": { "0": 1, "1": 3, "2": 0, "3": 2 }
}
```

**响应示例** (200):
```json
{
  "score": 8,
  "correctCount": 8,
  "totalQuestions": 10,
  "points": 6,
  "results": [
    {
      "questionId": 101,
      "userAnswer": "B",
      "mappedUserAnswer": "B",
      "correctAnswer": "B",
      "isCorrect": true,
      "explanation": "基础加法运算",
      "content": "<p>1+1=?</p>",
      "options": ["A. 1", "B. 2", "C. 3", "D. 4"],
      "type": "single",
      "audio_url": null,
      "image_url": null,
      "shuffleMapping": { "0": 2, "1": 0, "2": 3, "3": 1 },
      "readingResult": null
    }
  ]
}
```

**积分规则**:
- 普通题库：答对 +1 分，答错 -1 分，全对翻倍
- 错题巩固：答对累计 3 次 +1 分，答错重置计数

**错误响应**:
```json
{ "error": "缺少必填参数" }                           // 400
{ "error": "请求已过期，请重新提交" }                   // 401
{ "error": "答题会话不存在或已过期" }                  // 404
{ "error": "请求已被处理，请勿重复提交" }                // 401
{ "error": "签名验证失败，提交无效" }                    // 401
{ "error": "已经提交过答案" }                           // 400
{ "error": "提交过于频繁，请稍后再试" }                  // 400
```

## 四、数据流图

### 4.1 完整答题流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ POST /start │ ──→ │  学生答题   │ ──→ │ POST /submit│
│ 创建会话     │     │ 前端渲染   │     │ 提交答案     │
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │                     │
       ▼                   ▼                     ▼
  返回 quizId         收集行为            1. 签名验证
  + questions        (可选)               2. 判题逻辑
  (无答案)                                  3. 计算积分
                                           4. 保存记录
                                           5. 处理错题
                                           6. 更新进度
                                           7. 返回结果
```

## 五、安全机制详解

### 5.1 签名验证流程

```
前端生成签名：
  data = { quizId, answers, timestamp }
  dataStr = JSON.stringify(data) + timestamp + userId
  signature = HMAC-SHA256(dataStr, SECRET_KEY)

后端验证签名：
  1. 检查时间戳是否在5分钟内
  2. 使用相同算法重新生成签名
  3. 对比传入签名与生成签名
  4. 标记签名为已用（防重放）
```

### 5.2 限流策略

| 操作 | 限流器 | 说明 |
|------|--------|------|
| 开始答题 | apiLimiter | 一般限制 |
| 提交答案 | submitLimiter | 最严格限制 |
| 保存记录 | apiLimiter | 一般限制 |

### 5.3 防刷分机制

- 同一用户 5 秒内不可重复提交
- 签名唯一性保证（标记已用）
- 会话只能提交一次
