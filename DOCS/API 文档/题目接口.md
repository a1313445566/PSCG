## 一、题目列表

### 1.1 获取子分类列表

- **方法**: GET
- **路径**: `/api/questions/subcategories`
- **认证**: 无需
- **说明**: 获取所有子分类及其关联的学科信息和题目数量，用于前端筛选器

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| subjectId | number | 否 | 按学科 ID 筛选 |

**响应示例** (200):
```json
[
  {
    "id": 1,
    "name": "第一章 基础知识",
    "subjectId": 1,
    "subjectName": "数学",
    "questionCount": 25
  },
  {
    "id": 2,
    "name": "第二章 进阶应用",
    "subjectId": 1,
    "subjectName": "数学",
    "questionCount": 18
  }
]
```

### 1.3 获取题目列表（分页+筛选）

- **方法**: GET
- **路径**: `/api/questions`
- **认证**: 无需
- **说明**: 获取题目列表，支持多维度筛选和分页。支持排除完整内容模式以提升性能。

**请求参数**:

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| subjectId | number | 否 | - | 学科 ID |
| subcategoryId | number | 否 | - | 子分类 ID |
| type | string | 否 | - | 题型：single/multiple/judgment/reading |
| difficulty | number | 否 | - | 难度等级 (1-5) |
| keyword | string | 否 | - | 关键词搜索（题干内容模糊匹配） |
| page | number | 否 | 1 | 页码 |
| limit | number | 否 | 20 | 每页数量（最大100） |
| excludeContent | string | 否 | "false" | 是否排除完整内容("true"/"false") |

**响应示例** (200):
```json
{
  "data": [
    {
      "id": 1,
      "subjectId": 1,
      "subcategoryId": 1,
      "subjectName": "数学",
      "subcategoryName": "第一章",
      "content": "<p>1+1=?</p>",
      "type": "single",
      "options": ["A. 1", "B. 2", "C. 3", "D. 4"],
      "answer": "B",
      "explanation": "基础加法运算",
      "image": "/images/xxx.jpg",
      "audio": null,
      "difficulty": 1,
      "createdAt": "2026-01-01T00:00:00.000Z"
    }
  ],
  "total": 100,
  "page": 1,
  "limit": 20
}
```

## 二、单个题目操作

### 2.1 获取单个题目详情

- **方法**: GET
- **路径**: `/api/questions/:id`
- **认证**: 无需
- **说明**: 根据 ID 获取题目完整信息，包含选项、答案、解析等

**路径参数**:

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | number | 题目 ID |

**响应示例** (200):
```json
{
  "id": 1,
  "subjectId": 1,
  "subcategoryId": 1,
  "content": "<p>1+1=?</p>",
  "type": "single",
  "options": ["A. 1", "B. 2", "C. 3", "D. 4"],
  "answer": "B",
  "explanation": "基础加法运算",
  "audio": null,
  "image": "/images/xxx.jpg",
  "difficulty": 1,
  "createdAt": "2026-01-01T00:00:00.000Z"
}
```

**错误响应** (404):
```json
{ "error": "题目不存在" }
```

---

### 2.2 添加题目

- **方法**: POST
- **路径**: `/api/questions`
- **认证**: 无需
- **说明**: 创建新题目。自动进行 XSS 过滤、Base64 图片转换、判断题验证。

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| subjectId | number | 是 | 学科 ID |
| subcategoryId | number | 是 | 子分类 ID |
| content | string | 是 | 题干内容（支持富文本 HTML） |
| type | string | 是 | 题型：single/multiple/judgment/reading |
| options | array | 是 | 选项数组 |
| answer | string | 是 | 正确答案 |
| explanation | string | 否 | 解析内容 |
| audio | string | 否 | 音频 URL |
| image | string | 否 | 图片 URL |

**请求示例**:
```json
{
  "subjectId": 1,
  "subcategoryId": 1,
  "content": "<p>1+1等于多少？</p>",
  "type": "single",
  "options": ["A. 1", "B. 2", "C. 3", "D. 4"],
  "answer": "B",
  "explanation": "这是一道基础加法题"
}
```

**特殊处理**:
- **XSS 过滤**: content 和 explanation 自动过滤危险标签
- **Base64 转换**: 自动检测并转换 Base64 图片为文件存储
- **判断题验证**: 当 `type=judgment` 时强制校验选项为 `["对", "错"]`，答案为 A 或 B

**响应示例** (201):
```json
{
  "id": 101,
  "subjectId": 1,
  "subcategoryId": 1,
  "content": "<p>1+1等于多少？</p>",
  "type": "single",
  "options": ["A. 1", "B. 2", "C. 3", "D. 4"],
  "answer": "B",
  "explanation": "这是一道基础加法题",
  "audio": null,
  "image": null,
  "createdAt": "2026-04-05T00:00:00.000Z"
}
```

**错误响应**:
```json
{ "error": "题目信息不完整" }                    // 400
{ "error": "判断题选项必须为[\"对\", \"错\"]" }     // 400
{ "error": "添加题目失败" }                        // 500
```

| 参数名 | 类型 | 说明 |
|--------|------|------|

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|

| subcategoryId | number | 是 | 子分类 ID |
| content | string | 是 | 题干内容 |
| type | string | 是 | 题型 |

## 三、批量操作

### 3.1 批量操作接口

- **方法**: POST
- **路径**: `/api/questions/batch`
- **认证**: 无需
- **说明**: 支持批量删除、批量修改难度、批量修改题型、批量移动题目

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| action | string | 是 | 操作类型：delete / updateDifficulty / updateType / move |
| ids | array | 是 | 目标题目 ID 数组（最多100个） |
| data | object | 视 action 而定 | 操作数据 |

**action 详细说明**:

#### delete - 批量删除
```json
{
  "action": "delete",
  "ids": [1, 2, 3, 4, 5]
}
```
**响应**:
```json
{ "success": true, "affected": 5 }
```

#### updateDifficulty - 批量修改难度
```json
{
  "action": "updateDifficulty",
  "ids": [1, 2, 3],
  "data": { "difficulty": 3 }
}
```

#### updateType - 批量修改题型
```json
{
  "action": "updateType",
  "ids": [1, 2, 3],
  "data": { "type": "multiple" }
}
```

#### move - 批量移动到其他学科
```json
{
  "action": "move",
  "ids": [1, 2, 3],
  "data": {
    "subjectId": 2,
    "subcategoryId": 5
  }
}
```

**错误响应**:
```json
{ "error": "参数错误" }                          // 400
{ "error": "单次最多操作100条记录" }                // 400
{ "error": "包含无效的题目ID" }                    // 400
{ "error": "缺少难度参数" }                        // 400 (updateDifficulty)
{ "error": "缺少类型参数" }                        // 400 (updateType)
{ "error": "缺少学科参数" }                        // 400 (move)
{ "error": "未知操作类型" }                        // 400
```

## 四、数据模型

### questions 表结构

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT PK AUTO_INCREMENT | 题目 ID |
| subject_id | INT | 学科 ID（外键） |
| subcategory_id | INT | 子分类 ID（外键） |
| content | TEXT | 题干内容（HTML 富文本） |
| type | ENUM | 题型：single/multiple/judgment/reading |
| options | JSON | 选项数组 |
| correct_answer | VARCHAR | 正确答案 |
| explanation | TEXT | 解析内容 |
| audio_url | VARCHAR | 音频文件 URL |
| image_url | VARCHAR | 图片文件 URL |
| difficulty | INT | 难度等级 (1-5) |
| created_at | DATETIME | 创建时间 |

### 支持的题型 (type)

| 类型值 | 名称 | 选项特点 | 答案格式 |
|--------|------|----------|----------|
| single | 单选题 | 4-6 个选项 | 单字母 (A/B/C/D) |
| multiple | 多选题 | 4-6 个选项 | 多字母 (ABC/ABD) |
| judgment | 判断题 | 固定 ["对", "错"] | A(对) 或 B(错) |
| reading | 阅读理解 | 小题嵌套结构 | JSON 对象 {1:"A", 2:"B"} |

## 五、安全特性

1. **XSS 防护**: 所有文本内容经过深度 XSS 过滤
2. **Base64 处理**: 自动检测并转换内嵌 Base64 图片为文件存储
3. **SQL 注入防护**: 使用 MySQL 预编译语句（参数化查询）
4. **LIKE 注入防护**: 关键词搜索时转义 `%` 和 `_` 通配符
5. **长度限制**: 关键词搜索限制最大 100 字符
6. **数量限制**: 批量操作单次最多 100 条
7. **ID 校验**: 批量操作验证所有 ID 为有效正整数
8. **文件引用管理**: 新增/更新/删除时自动维护文件引用计数
