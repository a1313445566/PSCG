## 一、全局排行榜

### 1.1 获取全局排行榜（分页+筛选）

- **方法**: GET
- **路径**: `/api/leaderboard`
- **认证**: 无需
- **说明**: 获取全系统学生积分排名，支持按年级、班级、学科、时间范围筛选。结果自动缓存。

**请求参数**:

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| page | number | 否 | 1 | 页码 |
| limit | number | 否 | 20 | 每页数量（最大100） |
| grade | string | 否 | - | 年级筛选 |
| class | string | 否 | - | 班级筛选 |
| subjectId | number | 否 | - | 学科 ID 筛选 |
| timeRange | string | 否 | all | 时间范围：all/week/month |
| forceRefresh | string | 否 | "false" | 是否强制刷新缓存 |

## 四、缓存策略

### 4.1 缓存机制

排行榜接口采用智能缓存机制提升性能：

| 缓存键格式 | 过期时间 | 刷新策略 |
|------------|----------|----------|
| `leaderboard:{hash}` | 可配置（默认5分钟） | 滑动过期 + 强制刷新 |
| `leaderboard:user:{userId}:{hash}` | 同上 | 同上 |
| `leaderboard:top:{hash}` | 同上 | 同上 |

**Hash 生成规则**:
```
hash = MD5(grade + class + subjectId + timeRange)
```

不同筛选条件的组合会生成不同的缓存键，互不干扰。

### 4.2 强制刷新缓存

所有排行榜接口均支持 `forceRefresh=true` 参数强制刷新：

```
GET /api/leaderboard?forceRefresh=true
GET /api/leaderboard/user/1?forceRefresh=true
GET /api/leaderboard/top?forceRefresh=true
```

### 4.3 自动刷新触发条件

以下操作会自动清除相关缓存：
- 提交答题成绩 (`POST /api/quiz/submit`)
- 手动调整积分（如有）
- 批量数据导入

## 五、积分计算规则

### 5.1 积分来源

| 行为 | 积分变化 | 说明 |
|------|----------|------|
| 普通题库答对一题 | +1 | 基础得分 |
| 普通题库答错一题 | -1 | 惩罚扣分 |
| 全部答对 | 额外翻倍 | 奖励机制 |
| 错题巩固答对（累计3次） | +1 | 巩固奖励 |
| 错题巩固答错 | 重置计数 | 不扣分但清零进度 |

### 5.2 排序规则

1. **主排序**: 总积分降序
2. **次排序**: 正确率降序（积分相同时）
3. **第三排序**: 最后活跃时间升序（都相同时先活跃者优先）

## 六、使用场景

### 6.1 学生端

| 页面 | 推荐接口 | 展示方式 |
|------|----------|----------|
| 首页侧边栏 | `GET /api/leaderboard/top?limit=10` | TOP10 列表 |
| 个人中心 | `GET /api/leaderboard/user/:userId` | 我的排名 + 前后对比 |
| 学科详情 | `GET /api/leaderboard/subject/:subjectId` | 学科内排名 |

### 6.2 教师/管理员端

| 页面 | 推荐接口 | 用途 |
|------|----------|------|
| 班级管理 | `GET /api/leaderboard?grade=3年级&class=2班` | 班级排名 |
| 学科分析 | `GET /api/leaderboard/subject/:id` | 学科竞赛排行 |
| 数据导出 | `GET /api/leaderboard?page=1&limit=100` | 完整榜单导出 |

## 七、性能优化建议

1. **合理设置 limit**: 首页展示用 TOP10 即可，避免请求过多数据
2. **利用缓存**: 相同筛选条件下不要频繁传 `forceRefresh=true`
3. **按需筛选**: 尽量使用 grade/class 缩小查询范围
4. **时间范围选择**: 优先使用 `week` 或 `month` 而非 `all` 减少计算量
