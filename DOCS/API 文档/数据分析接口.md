## 一、综合数据分析 (analysis.js)

### 1.1 多维度数据分析（带缓存）

- **方法**: GET
- **路径**: `/api/analysis`
- **认证**: 无需
- **说明**: 获取多维度数据分析结果，支持学科、年级、班级、时间范围筛选。结果自动缓存 5 分钟。

| subjectId | number | 否 | 学科 ID |
| grade | string | 否 | 年级 |
| class | string | 否 | 班级 |
| startDate | string | 否 | 开始日期 (YYYY-MM-DD) |
| endDate | string | 否 | 结束日期 (YYYY-MM-DD) |
| forceRefresh | string | 否 | "true" 强制刷新缓存 |

**响应示例** (200):

### 1.2 导出分析报告为 Excel

- **方法**: GET
- **路径**: `/api/analysis/export`
- **认证**: 无需
- **说明**: 导出完整数据分析报告为 Excel 文件，包含多个工作表。

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| subjectId | number | 否 | 学科 ID |
| grade | string | 否 | 年级 |
| class | string | 否 | 班级 |
| startDate | string | 否 | 开始日期 |
| endDate | string | 否 | 结束日期 |
| format | string | 否 | 文件格式，默认 xlsx |

**响应**: 返回 Excel 文件流（Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet）

**Excel 工作表结构**:
- Sheet1: 总览数据
- Sheet2: 准确率趋势
- Sheet3: 学科表现
- Sheet4: 年级对比
- Sheet5: 难度分布
- Sheet6: 高错误率题目
- Sheet7: 优秀学生
- Sheet8: 进步学生
- Sheet9: 待关注学生

### 4.3 获取用户学科雷达图数据

- **方法**: GET
- **路径**: `/api/user-stats/:userId/radar`
- **认证**: 无需
- **说明**: 生成用户在各学科的雷达图数据，包括准确率、速度、稳定性、进步幅度等多维指标

**路径参数**:

| 参数名 | 类型 | 说明 |
|--------|------|------|
| userId | number | 用户 ID |

**响应示例** (200):
```json
{
  "success": true,
  "radarData": {
    "subjects": ["数学", "语文", "英语", "科学"],
    "metrics": {
      "accuracy": [82, 75, 68, 78],       // 准确率 (0-100)
      "speed": [75, 80, 70, 72],           // 速度得分 (越高越快)
      "stability": [88, 82, 75, 80],       // 稳定性 (波动越小越高)
      "improvement": [15, 10, 20, 8],      // 近期进步幅度 (%)
      "engagement": [85, 78, 72, 80]       // 参与度 (答题频率)
    },
    "overallScores": {
      "数学": 79.0,
      "语文": 75.0,
      "英语": 67.6,
      "科学": 79.5
    },
    "strengths": ["数学", "科学"],
    "weaknesses": ["英语"],
    "recommendation": "建议重点提升英语学科的准确率和参与度"
  }
}
```

**指标说明**:
- **accuracy**: 各学科答题正确率
- **speed**: 标准化后的答题速度分数
- **stability**: 成绩波动程度（方差越小分越高）
- **improvement**: 最近 7 天相比之前 7 天的进步百分比
- **engagement**: 该学科答题频率占总答题量的比例

### 5.5 自动调整题库难度

- **方法**: POST
- **路径**: `/api/difficulty/auto-adjust/:subcategoryId`
- **认证**: 无需
- **说明**: 基于答题数据自动调整题库内所有题目的难度

**路径参数**:

| 参数名 | 类型 | 说明 |
|--------|------|------|
| subcategoryId | number | 子分类 ID |

**请求参数**:

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| threshold | number | 否 | 0.2 | 触发调整的正确率阈值偏差 |
| minAttempts | number | 否 | 10 | 最少作答次数才参与调整 |

**响应示例** (200):
```json
{
  "success": true,
  "adjustedCount": 8,
  "skippedCount": 17,
  "details": [
    { "questionId": 101, "old": 2, "new": 3, "reason": "正确率过高(85%)，上调难度" },
    { "questionId": 105, "old": 4, "new": 3, "reason": "正确率过低(35%)，下调难度" }
  ]
}
```

**自动调整规则**:
- 正确率 > (当前难度对应基准 + threshold): 难度 +1
- 正确率 < (当前难度对应基准 - threshold): 难度 -1
- 难度范围限制: 1-5

## 六、数据分析使用场景

### 6.1 教师视角

| 场景 | 推荐接口 | 用途 |
|------|----------|------|
| 查看班级整体表现 | `GET /api/analysis` | 了解班级学习状况 |
| 导出成绩报告 | `GET /api/analysis/export` | 生成 Excel 报告 |
| 发现薄弱知识点 | `GET /api/answer-records/error-prone-questions` | 定位高错题 |
| 学生个体分析 | `GET /api/user-stats/:userId/*` | 一对一辅导依据 |

### 6.2 学生视角

| 场景 | 推荐接口 | 用途 |
|------|----------|------|
| 查看个人进度 | `GET /api/learning-progress/:userId` | 了解学习进展 |
| 学科能力画像 | `GET /api/user-stats/:userId/radar` | 雷达图展示 |
| 学习风格分析 | `GET /api/user-stats/:userId/learning-style` | 优化学习方法 |
| 错题巩固入口 | `GET /api/error-collection/:userId` | 针对性练习 |

### 6.3 管理员视角

| 场景 | 推荐接口 | 用途 |
|------|----------|------|
