## 一、系统设置 (settings.js)

### 1.1 获取系统配置

- **方法**: GET
- **路径**: `/api/settings`
- **认证**: 无需
- **说明**: 获取当前系统的所有配置项，包括 UI 配置、功能开关、AI 配置等

**响应示例** (200):
```json
{
  "success": true,
  "settings": {
    "siteName": "PSCG 智能学习系统",
    "siteDescription": "基于 AI 的个性化教育平台",
    "version": "1.0.0",
    "ui": {
      "primaryColor": "#409EFF",
      "sidebarCollapsed": false,
      "theme": "light"
    },
    "features": {
      "aiChatEnabled": true,
      "leaderboardVisible": true,
      "errorCollectionEnabled": true,
      "learningProgressEnabled": true
    },
    "quiz": {
      "defaultQuestionCount": 10,
      "maxQuestionCount": 50,
      "timeLimit": 0,
      "showExplanationAfterAnswer": true
    },
    "ai": {
      "defaultModelId": 1,
      "maxTokensPerRequest": 4096,
      "temperature": 0.7
    },
    "security": {
      "sessionTimeout": 24,
      "maxLoginAttempts": 5,
      "enableCaptcha": false
    }
  }
}
```

---

### 1.2 更新系统配置

- **方法**: PUT
- **路径**: `/api/settings`
- **认证**: 管理员 JWT
- **说明**: 批量更新系统配置项。仅更新传入的字段，未传入字段保持不变。

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| settings | object | 是 | 要更新的配置对象（支持嵌套更新） |

**请求示例**:
```json
{
  "settings": {
    "siteName": "PSCG 教育平台 v2",
    "features": {
      "aiChatEnabled": false
    },
    "quiz": {
      "defaultQuestionCount": 15
    }
  }
}
```

**响应示例** (200):
```json
{
  "success": true,
  "message": "配置已更新",
  "updatedSettings": { ... }   // 返回更新后的完整配置
}
```

### 2.2 添加学科

- **方法**: POST
- **路径**: `/api/subjects`
- **认证**: 管理员 JWT
- **说明**: 创建新学科

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| name | string | 是 | 学科名称 |
| icon | string | 否 | 图标标识 |
| color | string | 否 | 主题色（HEX） |
| description | string | 否 | 学科描述 |
| sortOrder | number | 否 | 排序序号 |

**响应示例** (201):
```json
{
  "success": true,
  "subject": {
    "id": 9,
    "name": "英语",
    "icon": "language",
    "color": "#E6A23C",
    "description": "英语学科",
    "sortOrder": 9,
    "createdAt": "2026-04-05T00:00:00.000Z"
  }
```

---

### 2.3 更新学科

- **方法**: PUT
- **路径**: `/api/subjects/:id`
- **认证**: 管理员 JWT
- **说明**: 更新学科信息

**路径参数**:

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | number | 学科 ID |

**请求参数**: 同 2.2（所有字段可选）

**响应示例** (200): 返回更新后的学科对象

---

### 2.4 删除学科

- **方法**: DELETE
- **路径**: `/api/subjects/:id`
- **认证**: 管理员 JWT
- **说明**: 删除学科及其所有子分类。如果学科下有题目则拒绝删除。

**路径参数**:

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | number | 学科 ID |

**响应示例** (200):
```json
{ "success": true, "message": "学科已删除" }
```

**错误响应**:
```json
{ "error": "该学科下还有题目，无法删除" }   // 400
```

---

### 2.5 调整学科排序

- **方法**: POST
- **路径**: `/api/subjects/reorder`
- **认证**: 管理员 JWT
- **说明**: 批量调整学科的显示顺序

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| orders | array | 是 | 排序数组 `[{id: 1, sortOrder: 2}, ...]` |

**响应示例** (200):
```json
{ "success": true, "message": "排序已更新" }
```

---

### 2.6 - 2.11 子分类管理接口

以下接口用于管理子分类（挂载在 `/api/subjects/:subjectId/subcategories` 下）：

#### 2.6 添加子分类
- **POST** `/api/subjects/:subjectId/subcategories`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| name | string | 是 | 子分类名称 |
| sortOrder | number | 否 | 排序序号 |

**响应示例** (201): 返回新创建的子分类对象

---

#### 2.7 更新子分类
- **PUT** `/api/subjects/:subjectId/subcategories/:subcategoryId`

**请求参数**: name, sortOrder（均为可选）

---

#### 2.8 删除子分类
- **DELETE** `/api/subjects/:subjectId/subcategories/:subcategoryId`

**约束**: 如果子分类下有题目则拒绝删除

---

#### 2.9 调整子分类排序
- **POST** `/api/subjects/:subjectId/subcategories/reorder`

**请求参数**: orders 数组

---

#### 2.10 获取单个学科详情
- **GET** `/api/subjects/:id`

返回完整学科信息 + 所有子分类 + 题目统计

---

#### 2.11 获取学科题目数量统计
- **GET** `/api/subjects/stats`

**响应示例** (200):
```json
{
  "success": true,
  "stats": [
    { "subjectId": 1, "subjectName": "数学", "questionCount": 400 },
    { "subjectId": 2, "subjectName": "语文", "questionCount": 350 }
  ],
  "totalQuestions": 1200
}
```

| tables | array | 否 | 指定要备份的表（默认全部） |

## 五、文件上传 (upload.js)

### 5.1 上传图片

- **方法**: POST
- **路径**: `/api/upload/image`
- **认证**: 无需
- **说明**: 上传图片文件（JPEG/PNG/GIF/WebP），自动压缩和格式转换。

**请求格式**: multipart/form-data

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| image | File | 是 | 图片文件（最大 2MB） |

**支持的格式**: JPEG, PNG, GIF, WebP, SVG（转PNG）
**大小限制**: 2MB
**自动处理**:
- 压缩到最大 1920x1920
- JPEG 质量 85%
- 自动转换为 PNG 格式存储

**响应示例** (200):
```json
{
  "success": true,
  "url": "/uploads/images/2026/04/05/abc123.png",
  "filename": "abc123.png",
  "size": 152000,
  "width": 800,
  "height": 600,
  "format": "png",
  "uploadedAt": "2026-04-05T09:30:00.000Z"
}
```

**错误响应**:
```json
{ "error": "请选择要上传的图片" }           // 400
{ "error": "不支持的图片格式" }               // 400
{ "error": "图片大小不能超过 2MB" }            // 400
```

---

### 5.2 上传音频

- **方法**: POST
- **路径**: `/api/upload/audio`
- **认证**: 无需
- **说明**: 上传音频文件（MP3/WAV/OGG），用于题目听力材料。

**请求格式**: multipart/form-data

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| audio | File | 是 | 音频文件（最大 10MB） |

**支持的格式**: MP3, WAV, OGG, M4A, FLAC
**大小限制**: 10MB

**响应示例** (200):
```json
{
  "success": true,
  "url": "/uploads/audio/2026/04/05/def456.mp3",
  "filename": "def456.mp3",
  "size": 2500000,
  "duration": 120,         // 秒
  "format": "mp3",
  "uploadedAt": "2026-04-05T09:30:00.000Z"
}
```

---

### 5.3 ⚠️ 批量导入接口（不存在！）

> **重要说明**：本系统 **没有** Excel/CSV 批量上传导入功能！
>
> 以下信息是常见 AI 幻觉，**切勿使用**：
>
> - ❌ `/api/upload/import-file` — **此端点不存在**
> - ❌ `/api/upload/excel` — **此端点不存在**
> - ❌ Excel (.xlsx) 批量导入 — **此功能不存在**
>
> **实际的批量添加方式**：
> 1. 在管理后台 → 题目管理 → 点击「批量添加」按钮
> 2. 粘贴符合格式的文本内容（参见 [批量添加题目示例](../示例/批量添加题目示例.md)）
> 3. 点击「解析题目」预览结果
> 4. 点击「批量添加」逐条创建

---

### 5.4 ⚠️ 其他上传相关接口（不存在！）

> 以下接口均为 AI 幻觉，**实际不存在**：
>
> - ❌ `/api/upload/security-check` — 文件安全扫描（不存在）
> - ❌ `/api/upload/cleanup-orphaned` — 清理孤儿文件（不存在）
> - ❌ `/api/upload/stats` — 上传统计信息（不存在）
>
> **实际上传模块仅有 2 个端点**：
> - `POST /api/upload/image` — 上传图片
> - `POST /api/upload/audio` — 上传音频
>
> 文件安全由 Multer 配置（类型+大小限制）和 Sharp 处理（图片）保障，无独立的安全扫描端点。

## 六、错题巩固 (error-collection.js)

| page | number | 否 | 1 | 页码 |

### 6.2 更新错题巩固状态

- **方法**: POST
- **路径**: `/api/error-collection/:userId/update`

**请求参数**:

| 参数名 | type | 必填 | 说明 |
|--------|------|------|------|

**响应示例** (200):
