# 选项打乱功能控制说明

## 功能概述

选项打乱功能可以通过**后台管理系统**的答题设置进行控制，实现开启/关闭选项随机排序。

## 后台设置界面

### 访问路径
1. 登录后台管理系统
2. 进入"系统设置"
3. 找到"答题设置"卡片

### 设置选项

**答案随机排序**
- **开启**：每次答题时，题目选项会被随机打乱
- **关闭**：选项保持原始顺序

界面显示：
```
答案随机排序  [开启] [关闭]  当前状态：开启/关闭
```

## 实现原理

### 1. 后台设置存储

设置存储在 `settingsStore` 中：
```javascript
{
  randomizeAnswers: true,  // 是否随机打乱答案选项
  // ... 其他设置
}
```

### 2. 前端加载设置

在 `QuizView.vue` 的 `onMounted` 钩子中：
```javascript
const { randomizeAnswers } = settingsStore.settings
```

### 3. 根据设置决定是否打乱

```javascript
const questionsWithShuffled = data.questions.map(q => {
  const options = typeof q.options === 'string' ? JSON.parse(q.options) : q.options;
  
  // 根据设置决定是否打乱选项
  if (randomizeAnswers) {
    const { shuffledOptions, reverseMapping } = shuffleOptions(options);
    return {
      ...q,
      options: shuffledOptions,
      shuffleMapping: reverseMapping,
      type: q.type || 'single'
    };
  } else {
    // 不打乱选项
    return {
      ...q,
      options: options,
      shuffleMapping: null,
      type: q.type || 'single'
    };
  }
});
```

### 4. 后端验证

后端会根据提交的 `shuffleMapping` 进行判断：
- 如果 `shuffleMapping` 为 `null`，则使用原始答案验证
- 如果 `shuffleMapping` 存在，则使用映射验证

## 使用场景

### 开启场景（推荐）
- 正式考试/测验
- 防止学生记忆答案位置
- 提高答题公平性

### 关闭场景
- 调试模式
- 特定题型要求（如听力题选项需要按顺序播放）
- 教师演示

## 测试方法

### 1. 开启选项打乱
1. 后台设置"答案随机排序"为"开启"
2. 开始答题
3. 观察选项顺序是否随机打乱
4. 提交答案，验证结果正确

### 2. 关闭选项打乱
1. 后台设置"答案随机排序"为"关闭"
2. 开始答题
3. 观察选项顺序是否保持原始顺序
4. 提交答案，验证结果正确

### 3. 多次答题测试
开启状态下，多次答题观察同一题目的选项顺序是否每次都不同。

## 注意事项

1. **设置生效范围**
   - 设置全局生效
   - 影响所有学科和题库
   - 不区分普通题库和错题巩固题库

2. **兼容性**
   - 完全向后兼容
   - 关闭功能时，行为与旧版本一致
   - 不影响现有数据和功能

3. **性能**
   - 打乱算法效率高（O(n)）
   - 不影响答题性能

4. **数据存储**
   - 开启打乱：`shuffled_options` 字段保存打乱后的选项
   - 关闭打乱：`shuffled_options` 字段保存原始选项

## 配置文件

设置保存在数据库中，也可以在配置文件中设置默认值：

```javascript
// 默认设置
const defaultSettings = {
  randomizeAnswers: true,  // 默认开启
  fixedQuestionCount: false,
  minQuestionCount: 5,
  maxQuestionCount: 10,
  fixedQuestionCountValue: 10
}
```

## 常见问题

### Q1: 为什么选项顺序没有变化？
**A:** 检查后台设置中的"答案随机排序"是否已开启。

### Q2: 开启打乱后，答题记录会受影响吗？
**A:** 不会。系统会正确保存打乱后的选项顺序，历史记录会保持答题时的状态。

### Q3: 可以针对特定题型关闭打乱吗？
**A:** 目前是全局设置。如需针对特定题型，可以扩展代码逻辑：
```javascript
// 示例：听力题不打乱
if (q.type === 'listening') {
  return { ...q, options, shuffleMapping: null };
}
```

### Q4: 打乱会影响正确答案的显示吗？
**A:** 不会。系统会自动处理映射关系，确保正确答案显示在正确的位置。
