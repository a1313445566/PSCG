# PSCG 文档中心（重构版）

**文档版本**: v3.1.0  
**最后更新**: 2026-03-28  
**维护人**: 项目组

## 重构背景与目标
- 项目近期改动较大，现有文档需一次性全面校准。
- 目标：统一结构、版本与术语；对齐最新架构、API、部署、安全与使用指南；明确归档与缺口。

## 文档导航（按类型）
- 技术文档：`技术文档/`（架构、安全、故障排除、性能/样式/AI 集成）
- API 文档：`API文档/`（REST/AI/统计等接口）
- 部署文档：`部署文档/`（系统部署、cloudflared 隧道）
- 使用指南：`使用指南/`、`用户指南/`（API 使用、后台管理、学生端、系统管理、AI 数据增强）
- 开发文档：`开发文档/`（编码规范、环境、测试指南）
- BUG 记录：`BUG记录/`（缺陷与修复记录）
- 模板：`模板/需求文档模板.md`
- 归档：`已归档/`（过期或已完成文档）

## 当前优先级（建议执行顺序）
1) 技术文档：更新架构、安全、中间件/服务、性能优化，覆盖 AI 队列、签名/限流/防重放、文件引用计数、难度服务等。
2) API 文档：同步新增/变更/废弃接口，补齐请求/响应/错误码示例，尤其 AI、user-stats、analysis、difficulty、question-semantic、answer-behavior。
3) 部署文档：校准 Node/Vite/PM2/Nginx/SSL/备份与环境变量。
4) 使用指南：更新后台/学生端/系统管理及 AI 功能操作流程与关键入口组件。
5) 开发文档：保持编码规范、环境、测试指南与 lint/构建要求同步。
6) BUG 记录/归档：整理历史问题与已归档内容，确保引用有效。

## 版本与日期规范
- 版本号：`v主.次.修`，主版本对应重大结构或架构变更；次版本对应功能/接口更新；修订版用于小修。  
- 日期格式：`YYYY-MM-DD` 或 `2026年3月28日`，在同一文档内保持一致。  
- 头部元信息统一包含：文档版本、最后更新、适用范围/角色。

## 更新与贡献流程
1. 在对应目录创建/修改 Markdown（中文命名，描述清晰）。  
2. 更新 `DOCS/README.md` 导航与 `DOCS/目录结构说明.md`。  
3. 如为替换性改动，将旧版移入 `已归档/` 保留记录。  
4. 涉及接口或部署变更时，同步调整 API/部署文档，确保示例与配置可直接执行。  
5. 保存后进行一次自查：链接可用、示例可运行、术语一致。
