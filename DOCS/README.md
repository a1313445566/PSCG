## 📁 文档分类说明

### 1. 技术文档 (`/技术文档/`)
存放系统架构、技术实现、问题排查等技术性文档。

#### 主要文档：
- **技术文档.md** - 系统整体技术架构和设计说明
- **性能监控功能集成指南.md** - 性能监控功能的详细技术文档
- **时间问题修复记录.md** - 时间相关问题的详细修复记录
- **测试性能监控API.sh** - 自动化测试脚本

#### 适用场景：
- 系统维护和故障排查
- 功能扩展和技术选型
- 性能优化和问题修复

### 2. API文档 (`/API文档/`)
存放API接口的详细文档。

#### 主要文档：
- **API.md** - 完整的API接口文档，包含所有端点的详细说明

#### 适用场景：
- 前后端接口联调
- API接口开发和测试
- 第三方系统集成

### 3. 使用指南 (`/使用指南/`)
存放用户和开发者快速上手指南。

#### 主要文档：
- **性能监控功能快速指南.md** - 性能监控功能的快速使用指南

#### 适用场景：
- 新功能快速上手
- 日常使用参考
- 功能操作指导

### 4. 需求文档 (`/需求文档/`)
存放项目需求分析和功能规划文档。

#### 主要文档：
- **后台数据加载优化需求文档.md** - 数据加载优化功能需求
- **学科自动错题巩固题库需求.md** - 错题巩固功能需求
- **VMind 自然语言交互失败（PSCG 项目适配）实例代码+排查方案 (1).md** - 自然语言交互相关需求

#### 适用场景：
- 功能规划和需求分析
- 项目进度跟踪
- 功能验收测试

### 5. 部署文档 (`/部署文档/`)
存放系统部署、运维和配置文档。

#### 主要文档：
- **cloudflared隧道设置指南.md** - 内网穿透隧道设置指南

#### 适用场景：
- 系统部署和安装
- 环境配置和运维
- 网络和安全设置

### 6. 开发文档 (`/开发文档/`)
预留目录，用于存放开发规范、代码标准等文档。

## 📝 文档更新规范

### 1. 新增文档
1. 根据文档类型选择对应目录
2. 创建新的Markdown文档
3. 更新本文档的目录结构说明
4. 在README.md中添加引用

### 2. 更新文档
1. 修改对应文档内容
2. 更新文档版本或修改日期
3. 如有重大变更，在CHANGELOG中记录

### 3. 文档命名规范
- 使用中文命名，便于理解
- 文件名应清晰描述文档内容
- 避免使用特殊字符和空格
- 技术文档使用`.md`扩展名
- 脚本文件使用`.sh`扩展名

### 4. 文档格式规范
- 使用Markdown格式编写
- 添加适当的标题层级
- 使用代码块展示示例代码
- 添加必要的注释和说明
- 保持统一的格式风格
