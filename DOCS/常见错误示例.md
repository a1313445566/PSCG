## 1. 数据库语法错误

### ❌ 错误：使用 SQLite 语法

项目使用 MySQL，但错误地使用了 SQLite 语法。

```sql
-- ❌ 错误示例
CREATE TABLE security_logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,  -- SQLite 语法
  name TEXT,                              -- SQLite 类型
  timestamp DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### ✅ 正确：使用 MySQL 语法

```sql
-- ✅ 正确示例
CREATE TABLE IF NOT EXISTS security_logs (
  id INT AUTO_INCREMENT PRIMARY KEY,      -- MySQL 语法
  name VARCHAR(100) NOT NULL,             -- MySQL 类型
  timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_name (name)                   -- MySQL 索引
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='安全监控操作日志';
```

### 📝 检查清单
- [ ] 检查 `package.json` 中的数据库依赖（mysql2/sqlite3）
- [ ] 检查 `services/database.js` 的实现方式
- [ ] 使用对应的数据库语法

## 2. Composables 使用错误

### ❌ 错误：错误理解 usePagination

错误地认为 `total` 是 `usePagination` 的返回值。

```javascript
// ❌ 错误示例
const logPagination = usePagination(20, ref(0))

// 错误：total 不是返回值！
logPagination.total.value = result.total

// 错误：total 不是返回值！
console.log(logPagination.total)
```

### ✅ 正确：total 作为参数传入

```javascript
// ✅ 正确示例
import { usePagination } from '@/composables/usePagination'
import { ref } from 'vue'

// 先创建 total ref
const logTotal = ref(0)

// 传入 total ref
const logPagination = usePagination(20, logTotal)

// 正确：直接修改 total ref
logTotal.value = result.total

// 正确：获取分页参数
const params = logPagination.getServerParams()
```

### 📝 usePagination 实现说明

```javascript
// src/composables/usePagination.js
export function usePagination(defaultPageSize = 10, total = ref(0)) {
  const currentPage = ref(1)
  const pageSize = ref(defaultPageSize)
  
  // total 是参数，不是返回值！
  
  return {
    currentPage,
    pageSize,
    // ... 其他返回值
    // 注意：不包含 total！
  }
}
```

## 4. API 封装遗漏

### ❌ 错误：使用原生 fetch

绕过项目的 API 封装，直接使用原生 fetch。

```javascript
// ❌ 错误示例
const response = await fetch('/api/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(data)
})
const result = await response.json()
```

### ✅ 正确：使用项目 API 封装

```javascript
// ✅ 正确示例
import { api } from '@/utils/api'

// GET 请求
const result = await api.get('/api/users', { page: 1, limit: 20 })

// POST 请求
const result = await api.post('/api/users', data)

// DELETE 请求
const result = await api.delete('/api/users/1')

// 带配置的请求
const result = await api.get('/api/users', {}, {
  retries: 3,        // 重试次数
  showError: false,  // 不显示错误
  timeout: 30000     // 超时时间
})
```

### 📝 API 封装功能

项目的 `api` 封装提供以下功能：
- ✅ 自动重试（网络错误自动重试 2 次）
- ✅ 请求防抖（防止重复请求）
- ✅ 请求超时处理（默认 30 秒）
- ✅ CSRF Token 自动添加
- ✅ 统一错误处理

## 6. 权限验证遗漏

### ❌ 错误：忘记使用 adminAuth

管理后台 API 忘记添加权限验证中间件。

```javascript
// ❌ 错误示例
router.post('/admin/users', async (req, res) => {
  // 错误：没有使用 adminAuth 中间件
  // 任何人都可以访问此接口！
  const result = await db.query('SELECT * FROM users')
  res.json(result)
})
```

### ✅ 正确：使用 adminAuth 中间件

```javascript
// ✅ 正确示例
const adminAuth = require('../middleware/adminAuth')

// 所有管理后台 API 必须使用 adminAuth
router.post('/admin/users', adminAuth, async (req, res) => {
  // adminAuth 已验证 JWT Token
  // req.admin 包含当前管理员信息
  const operator = req.admin.username
  
  const result = await db.query('SELECT * FROM users')
  res.json(result)
})
```

### 📝 adminAuth 功能

`adminAuth` 中间件提供以下功能：
- ✅ 验证 JWT Token
- ✅ 检查管理员权限
- ✅ 解析用户信息到 `req.admin`
- ✅ 自动返回 401/403 错误

## 7. 错误处理遗漏

### ❌ 错误：忘记 try-catch

异步操作忘记使用 try-catch 捕获错误。

#### 前端错误

```javascript
// ❌ 错误示例
const loadData = async () => {
  const result = await api.get('/api/users')
  data.value = result.data
}
```

#### 后端错误

```javascript
// ❌ 错误示例
router.get('/users', async (req, res) => {
  const result = await db.query('SELECT * FROM users')
  res.json(result)
})
```

### ✅ 正确：使用 try-catch

#### 前端正确

```javascript
// ✅ 正确示例
const loadData = async () => {
  try {
    const result = await api.get('/api/users')
    data.value = result.data
  } catch (error) {
    // api 已自动显示错误，无需额外处理
    // 如需自定义错误处理：
    // message.error(`加载数据失败: ${error.message}`)
  }
}
```

#### 后端正确

```javascript
// ✅ 正确示例
router.get('/users', adminAuth, async (req, res) => {
  try {
    const result = await db.query('SELECT * FROM users')
    res.json({ success: true, data: result })
  } catch (error) {
    console.error('[用户管理] 获取列表失败:', error)
    res.status(500).json({ error: '获取列表失败，请稍后重试' })
  }
})
```

## 8. XSS 防护遗漏

### ❌ 错误：未转义用户输入

直接使用用户输入，未进行 XSS 防护。

```javascript
// ❌ 错误示例
const handleBlock = async () => {
  const reason = formData.reason  // 用户输入，可能包含恶意代码
  
  await api.post('/api/block-ip', {
    ip: formData.ip,
    reason: reason  // 错误：未转义！
  })
}
```

### ✅ 正确：使用 escapeHtml

```javascript
// ✅ 正确示例
import { escapeHtml } from '@/utils/xss-filter'

const handleBlock = async () => {
  const reason = formData.reason
  
  // 转义用户输入
  const safeReason = escapeHtml(reason.trim())
  
  await api.post('/api/block-ip', {
    ip: formData.ip,
    reason: safeReason  // 正确：已转义
  })
}
```

### 📝 XSS 防护位置

**文件**: `src/utils/xss-filter.js`

```javascript
import { escapeHtml } from '@/utils/xss-filter'

// 转义 HTML 特殊字符
```

## 9. 数据库实例使用错误

### ❌ 错误：错误的数据库操作方式

使用错误的数据库操作方法。

```javascript
// ❌ 错误示例1：直接使用回调
db.query('SELECT * FROM users', (err, rows) => {
  if (err) throw err
  res.json(rows)
})

// ❌ 错误示例2：使用不存在的方法
const result = await db.execute('SELECT * FROM users')

// ❌ 错误示例3：错误的事务处理
await db.beginTransaction()
```

### ✅ 正确：使用项目的 Database 实例

```javascript
// ✅ 正确示例
const db = require('./services/database')

// 查询单条记录
const row = await db.get('SELECT * FROM users WHERE id = ?', [1])

// 查询多条记录
const rows = await db.query('SELECT * FROM users LIMIT ?', [10])

// 执行更新/插入/删除
await db.run('INSERT INTO users SET ?', { name: '张三', age: 20 })

// 更新操作
await db.run('UPDATE users SET name = ? WHERE id = ?', ['李四', 1])

// 删除操作
await db.run('DELETE FROM users WHERE id = ?', [1])
```

### 📝 Database 实例方法

**文件**: `services/database.js`

| 方法 | 说明 | 返回值 |
|------|------|--------|
| `db.get(sql, params)` | 查询单条记录 | 单个对象 |
| `db.query(sql, params)` | 查询多条记录 | 数组 |
| `db.run(sql, params)` | 执行更新/插入/删除 | ResultSet |

## 10. 生命周期清理遗漏

### ❌ 错误：忘记清理资源

使用 Hook 或定时器，但忘记在组件卸载时清理。

```vue
<script setup>
import { useLoading } from '@/composables/useLoading'

const { showLoading, hideLoading } = useLoading()

// ❌ 错误：忘记清理
// 组件卸载时，Loading 实例不会被清理，导致内存泄漏
</script>
```

### ✅ 正确：组件卸载时清理

```vue
<script setup>
import { useLoading } from '@/composables/useLoading'
import { onUnmounted } from 'vue'

const { showLoading, hideLoading, cleanup } = useLoading()

// ✅ 正确：组件卸载时清理
onUnmounted(cleanup)
</script>
```

### 📝 需要清理的资源

| 资源类型 | 清理方法 | 说明 |
|---------|---------|------|
| **useLoading** | `cleanup()` | 清理 Loading 实例 |
| **定时器** | `clearTimeout()` | 清理 setTimeout |
| **定时器** | `clearInterval()` | 清理 setInterval |
| **事件监听** | `removeEventListener()` | 清理事件监听器 |
