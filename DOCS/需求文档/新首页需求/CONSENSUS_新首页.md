## 零、关键需求变更记录

### 🚨 v2.0 重大变更（2026-04-08）

**用户明确要求**：
> "目前用户端的登录不要影响新首页，新首页在右上角建立登录入口，使用后台管理员账户登录"

**变更内容**：

| 变更项 | 原方案 | 新方案 | 影响范围 |
|--------|--------|--------|----------|
| **访问权限** | 需要学生登录 | **公开访问**（无需登录） | 路由守卫、页面逻辑 |
| **登录入口** | 无独立入口 | **右上角"管理员登录"按钮** | UI 设计、组件结构 |
| **登录账户** | 学生账户 (`studentId`) | **管理员账户** (`adminToken`) | 认证机制、数据源 |
| **登录后跳转** | 进入功能首页 | **进入管理后台** (`/admin`) | 路由逻辑 |
| **数据来源** | 读取学生信息 | **独立数据源**（Mock/API） | 数据层设计 |

**核心定位**：
```
新首页 = 独立门户页面 (Portal Page)
├── 公开访问（无需登录）
├── 右上角管理员登录入口
├── 内容展示 + 导航聚合
└── 完全独立于学生端系统
```

### 1.2 访问权限与登录机制

#### 公开访问设计

| 维度 | 说明 |
|------|------|
| **访问权限** | 公开访问，无需任何登录 |
| **路由守卫** | 不受 `router.beforeEach` 拦截 |
| **数据来源** | 独立 Mock 数据或公开 API |
| **用户信息** | 不读取 `studentId` 或 `adminToken` |

#### 管理员登录入口

| 维度 | 说明 |
|------|------|
| **位置** | 页面右上角 |
| **按钮文案** | "管理员登录" 或 "Admin Login" |
| **点击行为** | `router.push('/admin')` |
| **登录机制** | 复用现有 `/admin` 路由的 PasswordDialog |
| **登录后** | 进入管理后台（AdminView） |

#### 与现有系统的关系

```
┌────────────────────────────────────────────────────┐
│              PSCG 智能学习系统                      │
├────────────────────────────────────────────────────┤
│                                                    │
│  【学生端】                【管理员端】             │
│  /login → /home           /admin (PasswordDialog) │
│  需要学生登录              需要管理员登录           │
│                                                    │
│  【新首页 - 独立门户】 ⭐                           │

│  右上角: 管理员登录按钮 → /admin                   │
│                                                    │
└────────────────────────────────────────────────────┘
```

### 2.3 路由调整方案

**修改前**:
```javascript
{ path: '/home', name: 'Home', component: lazyLoad('HomeView') }
```

**修改后**:
```javascript
// 新首页成为默认首页
{ path: '/home', name: 'NewHome', component: lazyLoad('NewHomeView') }

// 传统首页通过独立路径访问
{ path: '/home/legacy', name: 'HomeLegacy', component: lazyLoad('HomeView') }
```

**影响范围**:
- 登录成功后跳转目标：`/home`（不变，但指向新组件）
- 所有引用 `/home` 的地方：自动切换到新首页
- 需要访问旧首页的地方：改为 `/home/legacy`

## 三、视觉设计共识

### 3.1 设计语言定义

**设计风格**: 现代极简风 (Modern Minimalism)

**参考对象**:
- Notion 文档界面
- 飞书文档
- Linear 产品设计
- Vercel 官网

**核心设计原则**:

| 原则 | 描述 | 应用示例 |
|------|------|----------|
| **留白至上** | 充足的负空间，让内容呼吸 | 卡片间距 ≥ 24px，section间距 ≥ 48px |
| **层级清晰** | 通过字号、字重、颜色建立信息层级 | 标题 24px Bold，正文 14px Regular |
| **克制装饰** | 去除不必要的装饰元素 | 无渐变边框、无 emoji 爆炸、无复杂动画 |
| **一致性强** | 统一的间距、圆角、阴影系统 | 全局使用 8px 网格系统 |
| **微交互** | 细腻的状态反馈 | hover 时 200ms 过渡，translateY(-2px) |

### 3.2 配色方案

**主色调**: 蓝色系 (Blue Palette)

```scss
// ============================================
// 新首页专属配色变量（扩展 _variables.scss）
// ============================================

// 主色系 - 蓝色
$new-primary: #3b82f6;           // 主蓝色 (Blue-500)
$new-primary-light: #60a5fa;     // 浅蓝 (Blue-400)
$new-primary-dark: #2563eb;      // 深蓝 (Blue-600)
$new-primary-bg: #eff6ff;        // 蓝色背景 (Blue-50)

// 中性色系 - 灰度
$new-text-primary: #1e293b;      // 主文字 (Slate-800)
$new-text-secondary: #64748b;    // 次要文字 (Slate-500)
$new-text-tertiary: #94a3b8;     // 辅助文字 (Slate-400)
$new-border: #e2e8f0;            // 边框 (Slate-200)
$new-bg-page: #f8fafc;           // 页面背景 (Slate-50)
$new-bg-card: #ffffff;           // 卡片背景 (纯白)

// 功能色系
$new-success: #10b981;           // 成功 (Emerald-500)
$new-warning: #f59e0b;           // 警告 (Amber-500)
$new-danger: #ef4444;            // 错误 (Red-500)

// 渐变（仅在必要时使用，保持克制）
$new-gradient-subtle: linear-gradient(135deg, #f0f9ff 0%, #e0f2fe 100%); // 微妙蓝渐变
```

**配色应用规范**:

| 元素 | 颜色 | 用途 |
|------|------|------|
| 页面背景 | `$new-bg-page: #f8fafc` | 整体底色 |
| 卡片背景 | `$new-bg-card: #ffffff` | 内容容器 |
| 主标题 | `$new-text-primary: #1e293b` | H1/H2 标题 |
| 正文 | `$new-text-secondary: #64748b` | 段落、描述 |
| 辅助文字 | `$new-text-tertiary: #94a3b8` | 时间戳、标签 |
| 主操作按钮 | `$new-primary: #3b82f6` | CTA 按钮 |
| 链接/强调 | `$new-primary: #3b82f6` | 可点击文本 |
| 分割线 | `$new-border: #e2e8f0` | 分隔线、边框 |

### 3.3 排版系统

**字体栈**:
```scss
$new-font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC',
                 'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue',
                 Helvetica, Arial, sans-serif;
```

**字号层级**:

| 层级 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| Display | 32px | 700 (Bold) | 1.2 | 页面大标题 |
| H1 | 24px | 600 (Semibold) | 1.3 | Section 标题 |
| H2 | 20px | 600 (Semibold) | 1.4 | 卡片标题 |
| H3 | 16px | 500 (Medium) | 1.5 | 子标题 |
| Body | 14px | 400 (Regular) | 1.6 | 正文内容 |
| Caption | 12px | 400 (Regular) | 1.5 | 辅助文字 |
| Overline | 11px | 500 (Medium) | 1.4 | 标签、时间戳 |

### 3.4 间距与网格系统

**基础单位**: `8px` (基于 8pt 网格系统)

| Token | 值 | 用途 |
|-------|-----|------|
| `$new-space-xs` | 4px | 紧凑元素间距 |
| `$new-space-sm` | 8px | 图标与文字间距 |
| `$new-space-md` | 16px | 卡片内边距、小元素间距 |
| `$new-space-lg` | 24px | 卡片间距、section 内边距 |
| `$new-space-xl` | 32px | section 之间间距 |
| `$new-space-2xl` | 48px | 主要区块间距 |
| `$new-space-3xl` | 64px | 页面顶部/底部留白 |

### 3.5 圆角与阴影

**圆角系统**:

| Token | 值 | 用途 |
|-------|-----|------|
| `$new-radius-sm` | 6px | 小按钮、标签 |
| `$new-radius-md` | 8px | 输入框、小型卡片 |
| `$new-radius-lg` | 12px | 中型卡片、下拉菜单 |
| `$new-radius-xl` | 16px | 大型卡片、模态框 |

**阴影系统** (扁平化设计，仅使用微妙阴影):

| Token | 值 | 用途 |
|-------|-----|------|
| `$new-shadow-sm` | `0 1px 2px rgba(0,0,0,0.05)` | 默认卡片 |
| `$new-shadow-md` | `0 4px 6px -1px rgba(0,0,0,0.07)` | hover 状态 |
| `$new-shadow-lg` | `0 10px 15px -3px rgba(0,0,0,0.08)` | 弹出层、模态框 |

### 3.6 动效规范

**过渡动画** (保持克制):

| 属性 | 值 | 应用场景 |
|------|-----|----------|
| Duration Fast | 150ms | 颜色变化、hover 出场 |
| Duration Normal | 200ms | 尺寸变换、位移 |
| Duration Slow | 300ms | 页面切换、复杂动画 |
| Easing | `cubic-bezier(0.4, 0, 0.2, 1)` | 标准 Material 曲线 |

**禁止使用的动效**:
- ❌ 弹性动画 (`cubic-bezier(0.34, 1.56, 0.64, 1)`)
- ❌ 旋转、缩放超过 1.05x
- ❌ 彩虹渐变、闪烁效果
- ❌ 复杂粒子效果

**允许使用的动效**:
- ✅ 微位移 (translateY: -2px ~ -4px)
- ✅ 阴影过渡
- ✅ 透明度淡入淡出
- ✅ 颜色渐变 (200ms 内)
