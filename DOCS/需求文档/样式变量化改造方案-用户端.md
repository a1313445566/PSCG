## 一、用户端改造概述

### 1.1 目标范围

| 域 | 文件范围 | 硬编码估算 |
|----|---------|-----------|
| 用户端主题 | `_game.scss`（996 行） | ~120+ 处 |
| 用户端 Views | `src/views/`（排除 admin 子目录）共 12 个文件 | ~390 处 |
| 用户端 Components | `common/` `leaderboard/` `quiz/` `student/` `subject/` 共 ~18 个组件 | ~650 处 |
| **合计** | | **~1160 处** |

### 1.2 根因（两端共有）

| 原因 | 影响 |
|------|------|
| Vite 未配置 `additionalData` | `.vue` 文件无法直接使用 `$primary-color` 等 SCSS 变量，被迫硬编码 |
| 缺渐变/按钮阴影等组合变量 | 同一值反复手写，~100 处渐变 / ~251 处阴影全部硬写 |
| `_game.scss` 自身含 120+ 处硬编码 | 主题文件带头破坏规范，新代码跟着抄 |
| 无防回归机制 | 技术债持续累积，改了又长回来 |

### 1.3 原则

1. **先配根因**：Phase 0 完成 `additionalData` 配置后再动任何文件
2. **复用现有架构**：在 `_variables.scss` 追加，不新建文件
3. **零视觉变化**：每文件改完即视觉快照验证，有偏差立即回滚
4. **变量化 + 样式抽离同步完成**：超标文件一次改完，不二次打开
5. **渐进可回滚**：每文件一次独立 Git commit

#### 如需补充更多变量

在实施过程中，若遇到未定义的变量：

1. **先检查 `_variables.scss`** —— 确认是否已存在
2. **若不存在，按命名规范追加**：
   ```scss
   // 在对应分类下追加
   $new-variable-name: #color-value;
   ```
3. **同步 CSS 变量**（用于内联样式）：
   ```scss
   :root {
     --new-variable-name: #{$new-variable-name};
   }
   ```

**命名规范**：
- 渐变：`$gradient-{name}` / `--gradient-{name}`
- 背景：`$bg-{level}-{hue}` / `--bg-{level}-{hue}`
- 边框：`$border-{context}` / `--border-{context}`
- 功能：`$form-{type}-{state}` / `--form-{type}-{state}`
- SVG/图形：`$kawaii-{part}`, `$icon-{type}` 等

**验证清单**：

- [ ] `npm run dev` 编译通过

## 四、样式文件抽离规则（与变量化同步推进）

### 4.1 触发条件

满足**任意一条**即必须在本次改造时抽离：

| 条件 | 阈值 |
|------|------|
| `<style>` 行数 | > 200 行 且占比 > 30% |
| 文件总行数 | > 600 行 |

### 4.2 抽离方式

```
变量化替换 → 抽离 <style> 为同名 .scss → .vue 改用 src 引用 → 视觉验证 → 单次 commit
```

```vue
<!-- 改造后的 .vue 文件 -->
<style lang="scss" src="./styles/LeaderboardView.scss" scoped></style>
```

```
src/views/
  styles/                     ← views 专属样式目录
    LeaderboardView.scss
    HomeView.scss
    DocsView.scss
    LearningProgressView.scss
  LeaderboardView.vue          ← 只剩 template + script
  HomeView.vue
  ...
```

**`.vue` 中保留**（不抽离）：少于 20 行的极简样式 / 组件级 CSS 变量 / `:deep()` 内联覆盖。

### 4.3 防反弹

```bash
npm run monitor:style-size        # 检查所有 .vue 样式行数
npm run monitor:style-size:check  # CI 模式，样式 > 400 行时退出码 1
```

## 九、风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| 实际硬编码量超统计（Views 内 JS `:style` 中） | 中 | `:style` 绑定不在改造范围；纯 `<style>` 块内改造 |
| `set-alpha()` 在独立 `.scss` 中编译失败 | 中 | 独立文件需显式 `@use "_functions" as *`；或改为预计算 CSS 变量 |
| `additionalData` 注入后编译变慢 | 低 | 注入纯变量文件（<150 行），无副作用；Vite HMR 增量编译 |
| 样式抽离后 scoped hash 变化 | 极低 | `src` attribute 方式 scoped hash 不变，与内联 `<style>` 行为一致 |
| 视觉偏差未及时发现 | 中 | 每文件改完立即运行 `npm run test:visual`，不攒到最后 |
