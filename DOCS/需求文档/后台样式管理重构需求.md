## 📋 需求概述

### 背景
当前项目样式管理存在以下问题：
1. **重复定义**：`global.css` 和 `common.css` 存在大量重复的工具类和动画
2. **使用率低**：`common.css` (566行) 只在少数文件中引入
3. **命名混乱**：`global.css` 实际是前台游戏风格，`common.css` 是后台样式
4. **缺乏复用**：后台组件各自写 scoped 样式，相同样式重复多次
5. **引入方式不统一**：有些组件用 `@import`，有些直接写 scoped

### 目标
- 统一样式管理架构，采用 ITCSS 最佳实践
- 减少重复代码，提高样式复用率
- 规范前后台样式分离，避免样式冲突
- 提高代码可维护性，建立清晰的样式规范
- 与 Element Plus 无缝集成，保持设计一致性


## 🔧 技术约束

### 必须遵循
1. **CSS 变量优先**：所有颜色、间距、圆角、字体等使用 CSS 变量
2. **scoped 样式**：组件特定样式必须使用 `<style scoped>`
3. **统一引入**：全局样式在 `main.js` 引入一次
4. **前后台分离**：前台游戏风格样式与后台管理样式严格分离
5. **BEM 命名规范**：后台样式使用 `admin-` 前缀，前台使用 `game-` 前缀

### 禁止事项
1. ❌ 禁止在多个文件重复定义相同的工具类
2. ❌ 禁止使用硬编码颜色值（应使用 CSS 变量）
3. ❌ 禁止在组件 scoped 样式中定义全局样式
4. ❌ 禁止删除现有 `global.css` 中的前台游戏样式（先迁移再删除）
5. ❌ 禁止使用 !important（除非特殊情况，需注释说明）
6. ❌ 禁止嵌套超过3层选择器
