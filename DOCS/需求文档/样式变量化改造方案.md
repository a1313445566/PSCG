## 一、改造概述

### 1.1 目标

将项目约 **1500 处**硬编码替换为 SCSS/CSS 变量，消除技术债，为主题切换奠基。

### 1.2 根因

| 原因 | 影响 |
|------|------|
| Vite 未配置 `additionalData` | `.vue` 文件无法直接用 `$primary-color` 等 SCSS 变量，被迫硬编码 |
| 缺渐变/按钮阴影等组合变量 | 同一值反复手写 |
| `_game.scss`(996行) + `_admin.scss`(577行) 自身含 170+ 处硬编码 | 榜样效应差 |
| 无防回归机制 | 技术债持续累积 |

### 1.3 原则

1. **先修根因再动手术** — 先配 `additionalData`，阻止新硬编码
2. **复用现有架构** — 在 `_variables.scss` 追加，不新建文件
3. **完全兼容** — 零视觉变化
4. **渐进式** — 6 批独立可回滚
5. **可维护** — 建多层防回归机制

## 三、变量补齐（写入 `_variables.scss`）

### 3.1 追加位置

在 `_variables.scss` 第 65 行后（现有间距变量之后）、Element Plus 覆盖区之前插入。

### 3.2 追加内容摘要

```scss
// ===== 渐变 =====
$bg-gradient-page: linear-gradient(135deg, #f8f9fa 0%, #e3f2fd 100%);
$header-gradient-scss: linear-gradient(90deg, #7dd3f8 0%, #a8e6cf 50%, #ffd88b 100%);
$bg-gradient-primary: linear-gradient(135deg, $primary-color 0%, lighten($primary-color, 12%) 100%);
$bg-gradient-success: linear-gradient(135deg, $success-color 0%, lighten($success-color, 8%) 100%);
// + card/tip/error/notFound/firstPlace/userStats 共 9 个

// ===== 按钮 3D 阴影（含 hover/active 变体）=====
$shadow-btn-primary: 0 4px 0 shade($primary-color, 15%);
$shadow-btn-primary-hover: 0 8px 0 shade($primary-color, 15%), 0 12px 20px set-alpha($primary-color, 30);
// + secondary/accent/success/danger 及其 hover/active 共 14 个

// ===== 文字阴影 =====
$text-shadow-base: 2px 2px 4px rgba(0, 0, 0, 0.1);
$text-shadow-primary: 2px 2px 4px set-alpha($primary-color, 30);

// ===== 卡片/交互状态 =====
$card-background-hover/input-background/skeleton-base/shimmer/progress-bg/loading-border/divider-color

// ===== Admin 专用（12 个）=====
$admin-bg: #f5f7fa;
// $admin-text-primary/regular/secondary/placeholder
// $admin-border-base/light/lighter
// $admin-data-card-shadow/hover + $admin-table-shadow

// ===== 功能色深变体（10 个）=====
$danger-dark/success-dark/info-light/secondary-light/dark/accent-light/dark/primary-light/dark

// ===== 边框扩展（4 个）=====
$border-color-light/admin/scrollbar-track/thumb/hover
```

### 3.3 同步更新 `:root`

在 `:root {}` 块末尾追加对应 CSS 变量映射（`--xxx: #{$xxx}`），覆盖高频使用的 ~25 个。

> **完整变量列表和精确值见第三章完整代码块，此处仅为摘要。**

## 四、实施步骤

### Phase 0：Vite 根因修复（30 min）⭐ 最重要

修改 `vite.config.js`，在 scss 配置中添加：

```js
additionalData: `@use "@/styles/scss/abstracts/_variables.scss" as *;\n`,
```

验证：`npm run dev` → 编译无报错 → 随机抽查 3 页样式不变 → DevTools `:root` 可见全部变量。

### Phase 1：变量补齐（40 min）

按第三章完整内容修改 `_variables.scss`，验证编译通过 + `:root` 新值正确。

### Phase 1.5：创建变量使用指南（1 h，同步任务）

输出到 `DOCS/开发规范/样式变量使用指南.md`（含所有变量速查表、正误对比、最佳实践）。后续每 Phase 完成后同步更新该文档。

### Phase 2：主题文件去硬编码（2-3 h）

**_game.scss**（~120+ 处）：按钮 3D 阴影→`$shadow-btn-*`；表单背景→`$input-background`；focus 光晕→`set-alpha()`；骨架屏→`$skeleton-*`；页面/卡片渐变→`$bg-gradient-*`；文字阴影→`$text-shadow-*`；边框灰色→`$border-color-light/$admin-border-*`。

**_admin.scss**（~50+ 处）：背景→`$admin-bg`；三级文字→`$admin-text-primary/regular/secondary`；容器阴影→`$admin-data-card-shadow/table-shadow`；分隔线→`$admin-border-lighter`；滚动条→`$scrollbar-*`。

> **保留例外**：`.status-tag.success/warning/danger/info` 的功能性语义色不动。

### Phase 3：Views 高优页面（4-5 h）

按以下顺序逐文件改造：

| 序 | 文件 | 估算量 |
|----|------|--------|
| 1 | LeaderboardView.vue | ~122 |
| 2 | HomeView.vue | ~83 |
| 3 | DocsView.vue | ~67 |
| 4 | LearningProgressView.vue | ~55 |
| 5 | QuizView.vue | ~39 |
| 6 | ProfileView.vue | ~46 |
| 7 | SubcategoryView.vue | ~28 |
| 8 | LearningReportView.vue | ~26 |
| 9 | AnswerHistoryView.vue | ~18 |
| 10 | ResultView/ErrorBook/Admin/Dashboard | ~52 |

### Phase 4：Components 高优组件（6-8 h）

分三类处理：

- **A 类（正常改造）**：LeaderboardTable, LoginForm, SubjectCard, QuestionCard 等 16 个通用组件 — 按规则逐处替换
- **B 类（例外）**：KawaiiCharacter.vue（~116 SVG 颜色）— **保持原样**
- **C 类（Admin 组件）**：BackupRestore, DataCleanup, UserManagement 等 ~35 个管理组件 — 使用 `$admin-*` 变量

### Phase 5：扫尾 + 自动化基础设施（3-4 h）

#### 5.1 视觉回归测试

使用 Playwright `toHaveScreenshot` 进行像素级 diff：

- **基线截图**：Phase 0 前对 10 个页面 × 3 个视口（desktop/tablet/mobile）拍摄基线
- **容差策略**：全局默认 1% 像素差异；图表区域 5%；文字区域 0.1%
- **工作流**：每 Phase 结束后运行 `npm run test:visual`，自动检测视觉偏移
- **CI 集成**：GitHub Actions push 时自动触发（可选）

> 完整测试文件：`tests/visual/snapshot.spec.js`

#### 5.2 硬编码监控脚本

提供 4 种模式的持续监控工具：

| 命令 | 用途 |
|------|------|
| `npm run monitor:scan` | 全量扫描 + TOP15 排行榜 |
| `npm run monitor:baseline` | 记录当前快照为基线 |
| `npm run monitor:diff` | 与基线对比，显示恶化/改善趋势 |
| `npm run monitor:check` | CI 模式（退出码 0/1）|

支持 pre-commit hook 和 GitHub Actions 集成。扫描范围：`src/views/` + `src/components/` 中 `.vue` 文件的 `<style>` 块，检测 hex/rgba/gradient/shadow/named-color 六类模式，加权评分，分级告警。

> 完整脚本：`scripts/hardcoded-monitor.js`

#### 5.3 可接受例外

| 类型 | 示例 |
|------|------|
| SVG 内联 fill/stroke | KawaiiCharacter.vue |
| Element Plus 深度覆盖 | 必须用库规定格式 |
| 功能性语义色 | status-tag success/warning 等 |
| 透明度黑/白 rgba | 过于通用 |
| template 中动态 :style | JS 表达式计算的颜色 |

#### 5.4 防回归体系（6 层）

| 层级 | 工具 | 时机 |
|------|------|------|
| L1 | Stylelint | 编码时 |
| L2 | Pre-commit Hook (`monitor:check`) | 每次 commit |
| L3 | Playwright 截图对比 | Push / Phase 结束 |
| L4 | CI Pipeline | PR |
| L5 | Cron 定期巡检 | 每日/周 |
| L6 | Code Review Checklist | CR 时 |

**至少部署 L1+L2+L3 三层。**

## 八、风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| 实际工作量超统计 | 高 | 每批次预留 20% 缓冲；低优先级留下一轮 |
| `set-alpha()/tint()/shade()` 编译报错 | 中 | Vue style 中显式 `@use _functions`; 或改用预计算的 CSS 变量 |
| `additionalData` 拖慢编译 | 低 | 注入纯变量文件 <150 行无副作用；Vite HMR 增量编译 |
| 新变量意外覆盖局部样式 | 低 | 有语义命名 + 每文件立即视觉验证 + 分批提交可回滚 |
| 浏览器兼容 | 极低 | 目标浏览器均原生支持 CSS 变量；SCSS 编译时替换零风险 |
