## 一、改造概述

### 1.1 改造目标

将项目中的310处硬编码样式替换为CSS变量，为主题系统奠定基础。

### 1.2 改造原则

1. **完全兼容**：不改变现有功能
2. **渐进式**：分批改造，逐步测试
3. **可逆性**：保留回退机制
4. **可维护**：建立清晰的变量命名规范

### 1.3 改造范围

| 类型 | 数量 | 优先级 |
|------|------|--------|
| 硬编码背景色 | 218处 | 🔴 高 |
| 硬编码颜色值 | 80处 | 🔴 高 |
| 硬编码阴影 | 12处 | 🟡 中 |
| **总计** | **310处** | - |

## 六、风险与应对

### 6.1 风险1：样式遗漏

**风险**：可能有硬编码样式遗漏

**应对**：
1. 使用grep命令全面搜索
2. 建立检查清单
3. 全面测试所有页面

### 6.2 风险2：样式冲突

**风险**：CSS变量可能与现有样式冲突

**应对**：
1. 使用唯一的变量前缀 `--theme-`
2. 测试所有页面
3. 保留回退机制

### 6.3 风险3：浏览器兼容

**风险**：CSS变量在旧浏览器不兼容

**应对**：
1. 使用autoprefixer添加前缀
2. 考虑使用CSS变量polyfill
3. 最低支持Chrome 80
