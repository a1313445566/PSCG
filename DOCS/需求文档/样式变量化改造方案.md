## 核心原则

1. **先修根因再动手术** — 先配 `additionalData`，阻止新硬编码
2. **两域分治** — 用户端与后台端独立推进，互不阻塞，可并行
3. **变量风格隔离** — 后台端统一使用 `$admin-*` 前缀，避免命名冲突
4. **瘦身先行** — 样式超标文件在变量化时同步抽离，不单独开专项
5. **渐进可回滚** — 每文件/每批一次独立 Git commit
6. **零视觉变化** — 每次改完立即运行视觉快照验证
7. **公共优先** — 公共组件样式文件是基础设施，优先于业务文件清理
8. **全尺寸覆盖** — 不限于颜色，间距/字号/圆角/边框宽度统一变量化

## AI 执行策略

### 执行模式：「AI 改 + 人验」

```
每文件改造流程：
  ① AI 扫描硬编码（颜色 + 间距/字号/圆角）→ 输出替换计划
  ② AI 执行替换 + 样式抽离（如需）
  ③ 编译验证（npm run dev）
  ④ 人工浏览器视觉验证 ← 关键步骤
  ⑤ 确认 → Git 提交
```

### AI 信任等级分批

| 批次 | 内容 | AI 角色 | 人工角色 |
|------|------|---------|---------|
| 第 1 批 | Phase 0 + 1.2 + 1.5 | 全自动执行 | 仅验证编译 |
| 第 2 批 | Phase 2A/2B 主题文件 | 执行 + 编译验证 | 逐文件浏览器验证 |
| 第 3 批 | Phase 3A/3B Views | 执行 + 编译验证 | 逐文件浏览器验证 |
| 第 4 批 | Phase 4A/4B Components | 辅助执行 | 人工主导验证 |
| 第 5 批 | Phase 5 监控/CI | 辅助 | 人工主导 |

### AI Guard Rails

1. **单文件单次提交** —— 每改完一个文件立即编译验证，失败就回滚
2. **严格替换表** —— 不自行判断颜色/尺寸语义，按方案速查表精确映射
3. **先扫描再替换** —— 每个文件先输出替换计划，确认后再执行
4. **异常暂停** —— 遇到方案中未定义的变量，先暂停询问，不自行猜测
5. **尺寸变量保守原则** —— 优先替换高频值（8/12/16/20/24px），低频值（6/10/14px）可保留
6. **保留例外** —— 以下类型不替换：
   - `rgba(0,0,0,0.x)` 纯透明黑（过于通用）
   - SVG `fill`/`stroke` 图形颜色数据（非 CSS 样式）
   - `:deep(.el-*)` 内部 Element Plus 深度覆盖
   - 功能性语义色（`.status-tag.success` 等）
   - template 中 `:style` 动态绑定
   - 固定尺寸（`width: 40px`、`height: 100px` 等）—— 组件特有尺寸不宜变量化
   - `transform: translateX/Y(px)` 动画偏移量—— 动画特有参数不宜变量化
   - `box-shadow` 中的非标准偏移量—— 保留，仅替换颜色部分

## 风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| 实际工作量超统计 | 高 | 每批次预留 20% 缓冲；低优先级留下一轮 |
| `_question-management.scss` 未纳入统计 | 中 | v5.0 已补充至 Phase 1.2，~60 处硬编码 + 间距未变量化 |
| `set-alpha()/tint()/shade()` 编译报错 | 中 | 独立 `.scss` 文件需显式 `@use "_functions" as *` |
| `additionalData` 拖慢编译 | 低 | 注入纯变量文件 <150 行无副作用；Vite HMR 增量编译 |
| 新变量意外覆盖局部样式 | 低 | 语义命名 + 每文件立即视觉验证 + 分批提交可回滚 |
| Stage A/B 并行引发变量命名冲突 | 低 | `$admin-*` 前缀隔离；两路改 `_variables.scss` 需合并时注意顺序 |
| Element Plus `:deep()` 覆盖被意外修改 | 中 | 改造前建立快照基线；`:deep()` 内部原则上不动 |
| 样式抽离后 scoped hash 变化 | 极低 | `src` attribute 方式 scoped hash 不变，与内联 `<style>` 行为一致 |
| AI 颜色语义误判 | 中 | 同一色值在不同上下文含义不同（如 `#ffffff` 可能是卡片背景或文字白）；严格按替换表执行，AI 不自行判断 |
| AI 无法验证视觉效果 | 高 | 每 Phase 改完后必须人工浏览器验证，AI 只负责编译通过检查 |
| 浏览器兼容 | 极低 | 目标浏览器均原生支持 CSS 变量；SCSS 编译时替换零风险 |

## 风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| 实际工作量超统计 | 高 | 每批次预留 20% 缓冲；低优先级留下一轮 |
| 尺寸变量化引入布局偏移 | 高 | `$spacing-md`(16px) 保持不变（76+ 处引用）；新变量仅用于增量替换 |
| `_question-management.scss` 未纳入统计 | 中 | v5.0 已补充至 Phase 1.2，~60 处硬编码 + 间距未变量化 |
| `set-alpha()/tint()/shade()` 编译报错 | 中 | 独立 `.scss` 文件需显式 `@use "_functions" as *` |
| `additionalData` 拖慢编译 | 低 | 注入纯变量文件 <150 行无副作用；Vite HMR 增量编译 |
| 新变量意外覆盖局部样式 | 低 | 语义命名 + 每文件立即视觉验证 + 分批提交可回滚 |
| Stage A/B 并行引发变量命名冲突 | 低 | `$admin-*` 前缀隔离；两路改 `_variables.scss` 需合并时注意顺序 |
| Element Plus `:deep()` 覆盖被意外修改 | 中 | 改造前建立快照基线；`:deep()` 内部原则上不动 |
| 样式抽离后 scoped hash 变化 | 极低 | `src` attribute 方式 scoped hash 不变，与内联 `<style>` 行为一致 |
| AI 颜色语义误判 | 中 | 同一色值在不同上下文含义不同；严格按替换表执行，AI 不自行判断 |
| AI 尺寸替换破坏布局 | 中 | 固定尺寸/动画偏移量不替换；低频尺寸值（6/10/14px）保守处理 |
| AI 无法验证视觉效果 | 高 | 每 Phase 改完后必须人工浏览器验证，AI 只负责编译通过检查 |
| 浏览器兼容 | 极低 | 目标浏览器均原生支持 CSS 变量；SCSS 编译时替换零风险 |
