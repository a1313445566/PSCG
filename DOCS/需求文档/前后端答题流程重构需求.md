# 前后端答题流程重构需求

## 一、需求概述

为了提高系统安全性和减轻前端负担，需要对现有答题流程进行重构，将题目生成和答案验证逻辑从前端移至后端，前端不再保存题库数据，只负责展示和交互。

## 二、核心需求

### 2.1 后端API重构

#### 2.1.1 开始答题API
- **路径**：`POST /quiz/start`
- **参数**：
  - `subjectId`：学科ID（必填）
  - `subcategoryId`：题库ID（必填）
  - `questionCount`：题目数量（可选）
  - `studentId`：学生ID（必填）
  - `grade`：年级（必填）
  - `class`：班级（必填）
- **返回**：
  - `quizId`：答题会话ID
  - `questions`：题目列表（不含正确答案）
  - `expiresAt`：会话过期时间

#### 2.1.2 提交答案API
- **路径**：`POST /quiz/submit`
- **参数**：
  - `quizId`：答题会话ID（必填）
  - `answers`：用户答案（格式：`{questionId: answer}`）（必填）
- **返回**：
  - `score`：得分
  - `correctCount`：正确题目数
  - `totalQuestions`：总题目数
  - `results`：每道题的详细结果（含正确答案和解析）
  - `points`：获得的积分

#### 2.1.3 获取历史答题记录API
- **路径**：`GET /quiz/history`
- **参数**：
  - `studentId`：学生ID（必填）
  - `limit`：返回数量（可选，默认20）
- **返回**：
  - 历史答题记录列表

### 2.2 数据库调整

#### 2.2.1 新增表
- **quiz_sessions**：存储答题会话信息
  - `id`：会话ID（主键）
  - `user_id`：用户ID（外键）
  - `subject_id`：学科ID（外键）
  - `subcategory_id`：题库ID（外键）
  - `questions`：题目列表（JSON格式）
  - `created_at`：创建时间
  - `expires_at`：过期时间

- **quiz_attempts**：存储每道题的答题情况
  - `id`：记录ID（主键）
  - `quiz_session_id`：答题会话ID（外键）
  - `question_id`：题目ID（外键）
  - `user_answer`：用户答案
  - `is_correct`：是否正确
  - `created_at`：创建时间

#### 2.2.2 现有表调整
- 保留 `answer_records` 和 `question_attempts` 表，用于历史记录
- 确保与新表的数据一致性

### 2.3 前端流程调整

#### 2.3.1 开始答题流程
1. 用户选择题库后，调用 `start` API
2. 接收返回的题目列表，显示在页面上
3. 保存 `quizId` 用于后续提交

#### 2.3.2 提交答案流程
1. 用户完成答题后，调用 `submit` API
2. 传递 `quizId` 和用户答案
3. 接收返回的结果，显示得分和错题分析

#### 2.3.3 状态管理调整
- 移除 `questionStore` 中的题目存储逻辑
- 简化 `quizStore`，只存储当前答题会话信息
- 不再使用本地存储保存题目数据

### 2.4 安全性要求

#### 2.4.1 会话管理
- 为每个答题会话设置过期时间（如30分钟）
- 定期清理过期会话

#### 2.4.2 数据验证
- 验证 `quizId` 的有效性
- 防止重复提交
- 验证用户身份

### 2.5 性能要求

#### 2.5.1 缓存策略
- 缓存常用题库的题目列表
- 缓存用户答题历史

#### 2.5.2 响应时间
- `start` API响应时间不超过1秒
- `submit` API响应时间不超过2秒

4. 创建新的数据库表

### 3.2 前端调整
1. 修改 `QuizView.vue`，使用新的API
2. 简化 `questionStore.js` 和 `quizStore.js`
3. 移除本地存储逻辑
4. 测试前端流程

### 3.3 测试验证
1. 功能测试：验证答题流程正常
2. 性能测试：验证API响应时间
3. 安全测试：验证数据安全性
4. 兼容性测试：确保后台管理功能正常

## 四、影响范围

### 4.1 影响的功能
- 答题流程
- 题目生成逻辑
- 答案验证逻辑
- 前端状态管理

### 4.2 不影响的功能
- 题目管理
- 题库管理
- 学科管理
- 用户管理
- 排行榜管理
- 错误率分析

## 五、预期效果

1. **提高安全性**：正确答案不暴露给前端
2. **减轻前端负担**：前端不再需要保存题库数据
3. **数据一致性**：所有验证和计算都在后端完成
4. **扩展性**：更容易添加新的答题规则和功能
5. **性能优化**：减少前端内存使用

## 六、风险评估

### 6.1 潜在风险
- **网络依赖**：答题过程需要网络连接
- **后端负载**：增加了后端的计算负担
- **会话管理**：需要妥善管理答题会话的生命周期

### 6.2 缓解措施
- 实现会话过期机制
- 优化数据库查询
- 添加缓存层
- 监控系统性能

## 七、验收标准

1. 答题流程正常，能够正确生成题目和验证答案
2. 前端不再保存题库数据
3. API响应时间符合要求
4. 后台管理功能正常
5. 数据安全性得到保障
