## 一、功能概述

### 1.1 需求背景
教育系统需要支持判断题（True/False）题型，用于测试学生对概念性知识的理解能力。判断题是一种特殊的客观题，选项固定为"对"和"错"。

### 1.2 功能描述
- **新增题型**: `judgment`（判断题）
- **选项定义**: 固定为 `["对", "错"]`，不可自定义
- **答案映射**: A = 对（正确），B = 错（错误）
- **展示形式**: 前端以对/错按钮组呈现，而非传统的选项列表

### 1.3 核心特性
| 特性 | 说明 |
|------|------|
| 选项固定 | 前后端强制为 ["对", "错"] |
| 不打乱选项 | 禁止对判断题执行 shuffleOptions 操作 |
| 答案标准化 | 统一使用 A/B 映射 |
| UI 特殊化 | 使用按钮组件而非列表组件 |

## 二、技术约束

### 2.1 选项处理约束
```javascript
// ⚠️ 强制约束：shuffleOptions 必须检查题型
if (q.type !== 'judgment') {
  // 仅对非判断题执行打乱操作
  return shuffleOptions(q.options)
}
// 判断题直接返回原始选项，不做任何处理
return { ...q, options: ['对', '错'], shuffleMapping: null }
```

### 2.2 前后端分工

#### 后端约束
- ✅ 存储时 options 字段固定为 `["对", "错"]`
- ✅ 返回数据时不执行打乱逻辑
- ❌ **禁止**在 API 层对判断题选项进行任何排序/打乱

#### 前端约束
- ✅ 接收数据后检查 `type === 'judgment'`
- ✅ 使用专用 UI 组件渲染（按钮组）
- ❌ **禁止**调用 shuffleOptions 处理判断题

### 2.3 答案映射规则
```
正确答案 = "A" → 表示"对"
正确答案 = "B" → 表示"错"
```

// src/utils/shuffleOptions.js
export function shuffleOptions(question) {
  // 🔒 关键：判断题跳过打乱
  if (question.type === 'judgment') {
    return {
      ...question,
      options: question.options || ['对', '错'],
      shuffleMapping: null // 无需映射
    }
  }

  // 其他题型正常打乱...
}

## 五、后端变更

### 5.1 API 接口说明
无特殊后端处理需求，仅需：
1. **验证**: 接收请求时校验 `type === 'judgment'` 时 options 必须为 `["对", "错"]`
2. **存储**: 直接存储，无需额外转换
3. **返回**: 原样返回数据库记录

### 5.2 数据验证中间件
```javascript
// routes/questions.js
function validateJudgmentQuestion(req, res, next) {
  const { type, options } = req.body;

  if (type === 'judgment') {
    const validOptions = JSON.stringify(['对', '错']);
    const currentOptions = JSON.stringify(options.sort());

    if (currentOptions !== validOptions) {
      return res.status(400).json({
        code: 400,
        msg: '判断题选项必须为 ["对", "错"]',
        data: null
      });
    }
  }

  next();
}
```
