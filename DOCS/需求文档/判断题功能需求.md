## 4. 数据结构设计

### 4.1 数据库表修改

#### 4.1.1 questions 表 type 字段修改

**现有结构**：
```sql
type ENUM('single', 'multiple') DEFAULT 'single'
```

**修改方案**：
```sql
ALTER TABLE questions MODIFY COLUMN type VARCHAR(20) DEFAULT 'single';
```

**修改原因**：
- MySQL ENUM 类型添加新值需要重建表或使用特定语法
- VARCHAR(20) 更灵活，支持未来扩展更多题型
- 现有代码已支持 `reading`、`listening`、`image`、`judgment` 等类型

#### 4.1.2 数据迁移脚本

```sql
-- 1. 备份现有数据
CREATE TABLE questions_backup AS SELECT * FROM questions;

-- 2. 修改字段类型
ALTER TABLE questions MODIFY COLUMN type VARCHAR(20) DEFAULT 'single';

-- 3. 更新现有数据（确保类型值正确）
UPDATE questions SET type = 'single' WHERE type NOT IN ('single', 'multiple', 'reading', 'listening', 'image', 'judgment');
```

### 4.2 判断题数据格式

#### 4.2.1 选项存储格式

**格式定义**：
```json
{
  "options": ["对", "错"]
}
```

**重要说明**：
- 判断题选项固定为两个元素：`["对", "错"]`
- 存储为 JSON 字符串：`'["对", "错"]'`
- **不使用** `{"√":"对", "×":"错"}` 对象格式，避免前端渲染复杂度

#### 4.2.2 答案存储格式

**正确答案格式**：
- 对：存储为 `"A"`（第一个选项）
- 错：存储为 `"B"`（第二个选项）

**示例**：
```json
{
  "type": "judgment",
  "options": ["对", "错"],
  "correct_answer": "A"  // 表示"对"是正确答案
}
```

**设计原因**：
1. 与现有单选题答案格式保持一致（A/B/C/D）
2. 复用现有的答案验证逻辑
3. 前端渲染逻辑统一

### 4.3 答题记录

**保持与现有答题记录结构一致**：
- `user_answer`：存储为 `"A"` 或 `"B"`
- `correct_answer`：存储为 `"A"` 或 `"B"`
- `is_correct`：0 或 1

### 4.4 批量导入格式

#### 4.4.1 支持的格式示例

```
This is an apple. (×)

These are bananas. (√)

1+1=2. (√)

地球是平的. (×)

23.There are 12 coats hanging in the closet. (    × )

24.The weather is cloudy in autumn. (    √  )

25.It's winter, and we like to make a snowman. (  √    )

26.I put my red socks into the blue box. (    × )

27.I take to sports like a duck to water. (  ×   )

28.I can help everyone stand in line. (   √   )

1、1+1=2. (√)

2、地球是平的. (×)

【判断】地球是圆的。(√)
```

#### 4.4.2 解析规则

**自动识别规则**：
1. **优先级1**：包含 `【判断】` 标记的题目
2. **优先级2**：答案为 `(√)` 或 `(×)` 格式的题目（括号内允许任意空格）
3. **优先级3**：答案为 `(A)` 或 `(B)` 且选项为"对/错"的题目

**答案映射**：
- `(√)` → 正确答案为 `"A"`（对）
- `(×)` → 正确答案为 `"B"`（错）

**正则表达式**：
```javascript
// 判断题答案匹配
const judgmentAnswerRegex = /[\(（]\s*[√×]\s*[\)）]/

// 判断题标记匹配
const judgmentTagRegex = /【判断】/

// 编号题目匹配
const numberedQuestionRegex = /^(\d+[.、]\s*)(.+?)([\(（]\s*[√×]\s*[\)）])(.*)$/

// 无编号题目匹配
const unnumberedQuestionRegex = /^(.+?)([\(（]\s*[√×]\s*[\)）])(.*)$/
```

#### 6.1.3 批量导入接口

**前端解析逻辑**：
```javascript
// BatchAddQuestion.vue
const parseBatchQuestions = () => {
  const questions = []
  const lines = batchQuestionText.value.split('\n').map(line => line.trim()).filter(line => line)
  
  let currentQuestion = null
  
  for (const line of lines) {
    // 优先检查【判断】标记
    if (line.includes('【判断】')) {
      if (currentQuestion) questions.push(currentQuestion)
      
      const contentMatch = line.match(/【判断】(.+?)[\(（]\s*[√×]\s*[\)）]/)
      const answerMatch = line.match(/[\(（]\s*([√×])\s*[\)）]/)
      
      if (contentMatch && answerMatch) {
        currentQuestion = {
          content: contentMatch[1].trim(),
          answer: answerMatch[1] === '√' ? 'A' : 'B',
          type: 'judgment',
          options: ['对', '错'],
          explanation: ''
        }
      }
      continue
    }
    
    // 检查判断题答案格式 (√) 或 (×)
    const judgmentMatch = line.match(/^(?:(\d+[.、]\s*))?(.+?)[\(（]\s*([√×])\s*[\)）]/)
    if (judgmentMatch) {
      if (currentQuestion) questions.push(currentQuestion)
      
      currentQuestion = {
        content: judgmentMatch[2].trim(),
        answer: judgmentMatch[3] === '√' ? 'A' : 'B',
        type: 'judgment',
        options: ['对', '错'],
        explanation: ''
      }
      continue
    }
    
    // 现有单选题/多选题解析逻辑
    const singleChoiceMatch = line.match(/^(\d+[.、]\s*)?(.+?)[\(（]\s*([A-Da-d]+)\s*[\)）]/)
    if (singleChoiceMatch) {
      if (currentQuestion) questions.push(currentQuestion)
      
      const answer = singleChoiceMatch[3].toUpperCase()
      currentQuestion = {
        content: singleChoiceMatch[2].trim(),
        answer: answer,
        type: answer.length > 1 ? 'multiple' : 'single',
        options: [],
        explanation: ''
      }
      continue
    }
    
    // 解析选项（单选题/多选题）
    const optionMatch = line.match(/^[A-Da-d][.、．]\s*(.+)$/)
    if (optionMatch && currentQuestion && currentQuestion.type !== 'judgment') {
      currentQuestion.options.push(optionMatch[1].trim())
      continue
    }
    
    // 解析说明
    const explanationMatch = line.match(/^解析[:：]\s*(.+)$/)
    if (explanationMatch && currentQuestion) {
      currentQuestion.explanation = explanationMatch[1].trim()
    }
  }
  
  if (currentQuestion) questions.push(currentQuestion)
  
  parsedQuestions.value = questions
  ElMessage.success(`成功解析 ${questions.length} 个题目`)
}
```
