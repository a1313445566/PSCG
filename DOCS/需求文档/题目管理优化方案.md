# 题目管理优化需求文档

## 需求概述

**需求名称**：题目管理 - 性能与用户体验优化  
**需求类型**：性能优化 + 用户体验优化 + 布局重构  
**优先级**：高  
**创建日期**：2026年3月24日  
**文档版本**：v2.0

### 背景

随着题库数量增长，题目管理模块面临以下问题：

**性能问题：**
- 前端一次性加载所有题目数据，数据量大时页面卡顿
- 筛选和搜索在客户端进行，无法利用数据库索引优化
- 内存占用高，影响用户体验
- 列表渲染性能下降，滚动不流畅

**用户体验问题：**
- 筛选操作繁琐，需手动点击"应用"按钮
- 编辑题目需弹窗，操作步骤多
- 批量操作功能有限
- 无撤销功能，误操作无法恢复

**布局问题：**
- 筛选区和操作按钮都在顶部，占用空间大，导致表格可视区域小
- 筛选条件横向排列，学科和题库多时显得拥挤
- 操作按钮（添加、批量删除、刷新）和筛选按钮混在一起，功能分区不清晰
- 没有利用左侧空间做导航，所有功能都挤在一行
- 统计信息（题目数量）不够突出

采用 **左侧学科树 + 右侧表格** 布局，这是综合评估后的最优解：

## 二、性能优化方案

### 2.1 服务端分页 + 筛选（P0）

**核心思路**：将前端筛选改为服务端筛选，按需分页加载数据。

#### 接口改动

**现有接口**：
```
GET /api/questions?subjectId=1&subcategoryId=2&type=single&page=1&limit=20
```

**新增参数**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| keyword | string | - | 搜索关键词 |
| excludeContent | boolean | false | 是否排除完整内容（列表页建议 true） |
| page | number | 1 | 页码 |
| limit | number | 50 | 每页数量（最大 100） |

**响应格式**：
```json
{
  "data": [...],
  "total": 1234,
  "page": 1,
  "limit": 50
}
```

#### 预期效果

| 指标 | 改造前 | 改造后 | 提升 |
|------|--------|--------|------|
| 首屏加载时间 | 3-5s | < 1s | **80%+** |
| 内存占用 | 全量数据 | 50 条 | **90%+** |
| 筛选响应时间 | 1-2s | < 100ms | **95%+** |

---

### 2.2 数据库索引优化（P0）

#### 索引设计

```sql
-- 常用查询字段索引
ALTER TABLE questions ADD INDEX idx_questions_subject (subject_id);
ALTER TABLE questions ADD INDEX idx_questions_subcategory (subcategory_id);
ALTER TABLE questions ADD INDEX idx_questions_type (type);
ALTER TABLE questions ADD INDEX idx_questions_created (created_at);

-- 复合索引（筛选组合查询）
ALTER TABLE questions ADD INDEX idx_questions_filter (subject_id, subcategory_id, type);
```

#### 预期效果

- 查询速度提升 **10x+**
- 数据库 CPU 占用降低

---

### 2.3 字段精简（P1）

**列表页返回字段**：
```
id, subject_id, subcategory_id, type, content(前100字), difficulty, created_at
```

**详情页返回字段**：
```
* (完整字段)
```

#### 预期效果

- 数据传输量减少 **70%+**

```
当前筛选：[语文 ×] [一年级 ×] [关键词: "计算" ×] [清除全部]
```

#### 3.1.3 搜索结果高亮

搜索结果中高亮显示匹配关键词：

```js
const highlightKeyword = (content, keyword) => {
  if (!keyword) return content
  const regex = new RegExp(`(${keyword})`, 'gi')
  return content.replace(regex, '<mark class="highlight">$1</mark>')
}
```

### 3.2 编辑操作体验

#### 3.2.1 行内快速编辑（P0）

双击单元格直接编辑，无需弹窗：

```vue
<script setup>
const editingId = ref(null)
const editingContent = ref('')

const startInlineEdit = (row) => {
  editingId.value = row.id
  editingContent.value = row.content
}

const saveInlineEdit = async (row) => {
  if (editingContent.value !== row.content) {
    await updateQuestion(row.id, { content: editingContent.value })
    ElMessage.success('修改成功')
  }
  editingId.value = null
}

const cancelInlineEdit = () => {
  editingId.value = null
}
</script>
```

#### 3.2.2 删除撤销功能（P0）

删除后显示 3 秒撤销提示：

```vue
<script setup>
const deleteWithUndo = (row) => {
  // 先从列表移除
  const index = questions.value.findIndex(q => q.id === row.id)
  questions.value.splice(index, 1)
  
  // 显示撤销提示
  ElMessage({
    message: '题目已删除',
    type: 'warning',
    duration: 3000,
    showClose: true,
    action: {
      text: '撤销',
      handler: () => {
        questions.value.splice(index, 0, row)
        ElMessage.success('已撤销删除')
      }
    }
  })
  
  // 3秒后真正删除
  setTimeout(() => {
    if (questions.value.find(q => q.id === row.id)) return // 已撤销
    deleteQuestion(row.id)
  }, 3000)
}
</script>
```
