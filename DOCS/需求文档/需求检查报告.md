# 用户中心功能需求检查报告

## 📋 检查时间
2026-03-28

## ✅ 已修复的问题

### 1. 图表库不一致（已修复）
- **问题**：需求文档写的是 `ECharts 5`，但实际项目使用的是 `VisActor VChart`
- **影响**：代码示例错误，无法直接使用
- **修复**：已修正为 `VisActor VChart（@visactor/vchart）`

### 2. API 路径错误（已修复）
- **问题**：需求文档写的 `GET /api/error-collection/:userId`，但实际 API 是 `GET /api/error-collection/:subjectId?studentId=xxx&grade=xxx&class=xxx`
- **影响**：API 调用失败
- **修复**：已修正 API 路径和参数说明

### 3. localStorage 使用说明不足（已修复）
- **问题**：需求文档只提到 `userId`，但实际存储了多个值
- **影响**：开发者不知道如何获取用户信息
- **修复**：已添加完整的 localStorage 字段说明

## ⚠️ 仍需注意的问题

### 1. API 获取错题总数效率问题
**问题描述**：
- 需求要求显示"错题总数"
- 但 `error-collection` API 需要按学科查询
- 遍历所有学科效率低下

**建议**：
```javascript
// 方案 1：创建新的统计 API
GET /api/error-collection/stats/:userId
返回：{ total: 894, bySubject: [...] }

// 方案 2：在前端缓存学科列表，一次性查询
// 适合学科数量较少的情况
```

### 2. 排行榜排名获取
**问题描述**：
- 需求要求显示"全校排名"
- 但现有 API `/api/leaderboard` 返回的是 Top 10 列表
- 没有直接获取用户排名的 API

**建议**：
```javascript
// 方案：在 users/stats/:userId API 中添加 rank 字段
{
  totalSessions: 10,
  totalQuestions: 50,
  avgAccuracy: 85.5,
  points: 120,
  rank: 15  // 新增：全校排名
}
```

### 3. 图表使用方式
**问题描述**：
- `useChartRenderer.js` 是为 Markdown 中的图表设计的
- 从 Markdown 中提取 `vchart` 代码块
- 不适合直接在 Vue 组件中使用

**建议**：
```vue
<!-- 方案 1：直接使用 VChart 组件 -->
<script setup>
import { VChart } from '@visactor/vchart'
import { lineChartSpec } from '@visactor/vchart-spec'
</script>

<template>
  <VChart :spec="lineChartSpec" />
</template>

<!-- 方案 2：使用 useChartRenderer，但需要准备 Markdown -->
<script setup>
const { renderCharts } = useChartRenderer()
const markdown = `
\`\`\`vchart
{
  "type": "line",
  "data": [...],
  ...
}
\`\`\`
`
</script>

<template>
  <div ref="container" v-html="markdownHtml">
    <div class="chart-placeholder"></div>
  </div>
</template>
```

### 4. users 表字段说明
**问题描述**：
- users 表有 `points` 字段（积分）
- 需求文档中没有详细说明如何使用

**数据库结构**：
```sql
users:
  - id (int, 主键)
  - student_id (varchar, 学号)
  - name (varchar, 姓名)
  - grade (int, 年级)
  - class (int, 班级)
  - points (int, 积分，默认 0)
  - created_at (datetime)
```

**建议**：在需求文档中明确说明 `points` 字段的来源和更新规则。

## 🔍 其他建议

### 1. 添加路由权限说明

```javascript
// 路由守卫检查的是 studentId

localStorage.getItem('userGrade')   // 年级
localStorage.getItem('userClass')   // 班级
```

### 2. 统一数据获取方式
建议在需求文档中提供统一的数据获取方法：
```javascript
// src/composables/useUser.js（建议新建）
export function useUser() {
  const userId = localStorage.getItem('userId')
  const studentId = localStorage.getItem('studentId')
  const userName = localStorage.getItem('userName')
  const userGrade = localStorage.getItem('userGrade')
  const userClass = localStorage.getItem('userClass')
  
  return {
    userId,
    studentId,
    userName,
    userGrade,
    userClass,
    isLoggedIn: !!studentId
  }
}
```
