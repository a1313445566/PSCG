# 后台数据加载优化需求文档

## 1. 需求背景

目前后台管理系统在加载数据时存在以下问题：

1. **API请求过多**：初始加载时发起多个API请求，导致页面加载速度慢
2. **数据加载时机不合理**：所有数据在页面初始化时一次性加载，没有按需加载机制
3. **数据缓存缺失**：缺乏有效的数据缓存策略，相同数据可能重复请求
4. **错误处理不完善**：单个请求失败可能影响整体加载
5. **用户列表加载效率低**：为每个用户单独发起统计数据请求，可能导致大量并发请求

## 2. 需求目标

1. **提升加载速度**：通过减少API请求和实现缓存，显著提升页面加载速度
2. **改善用户体验**：页面响应更快，加载状态更清晰
3. **降低服务器负载**：减少并发请求，降低服务器负担
4. **提高代码可维护性**：更清晰的代码结构，更好的错误处理

## 3. 实施步骤

### 3.1 后端优化

#### 3.1.1 创建综合API端点
- **功能**：创建 `/api/init-data` 端点，一次性返回所有初始化数据
- **返回数据**：包含subjects、questions、grades、classes、userStats、recentRecords
- **验证策略**：
  - 验证API返回状态码为200
  - 验证返回数据包含所有必要字段
  - 验证数据格式正确

#### 3.1.2 优化用户列表API
- **功能**：修改 `/api/users` 端点，添加 `withStats` 参数
- **返回数据**：当 `withStats=true` 时，返回包含统计数据的用户列表
- **验证策略**：
  - 验证API返回状态码为200
  - 验证返回数据包含用户统计字段
  - 验证统计数据计算正确

#### 3.1.3 实现分页功能
- **功能**：为用户列表和题目列表实现分页功能
- **参数**：添加 `page` 和 `limit` 参数
- **验证策略**：
  - 验证分页参数生效
  - 验证返回数据包含分页信息
  - 验证不同页码返回不同数据

### 3.2 前端优化

#### 3.2.1 实现数据缓存机制
- **功能**：在前端实现数据缓存，减少重复请求
- **缓存策略**：
  - 核心数据缓存5分钟
  - 用户数据缓存2分钟
  - 其他数据缓存1分钟
- **验证策略**：
  - 验证相同请求在缓存有效期内不重复发起
  - 验证缓存过期后重新请求数据
  - 验证缓存数据正确更新

#### 3.2.2 优化loadAllUsers方法
- **功能**：修改loadAllUsers方法，使用后端提供的统计数据
- **验证策略**：
  - 验证不再为每个用户单独请求统计数据
  - 验证用户列表包含统计数据
  - 验证加载速度显著提升

#### 3.2.3 实现按需加载
- **功能**：只在需要时加载数据，例如用户管理数据只在切换到用户管理标签时加载
- **验证策略**：
  - 验证页面初始化时不加载用户管理数据
  - 验证切换到用户管理标签时加载数据
  - 验证其他标签数据按需加载

#### 3.2.4 改进错误处理
- **功能**：实现更完善的错误处理机制，单个请求失败不影响其他请求
- **验证策略**：
  - 验证单个API请求失败时其他请求正常完成
  - 验证错误信息正确显示
  - 验证系统稳定性不受影响

```javascript
router.get('/init-data', async (req, res) => {
```

#### 4.1.2 优化用户列表API
```javascript
// routes/users.js
router.get('/', async (req, res) => {
  try {
    const { grade, class: className, page = 1, limit, withStats = false } = req.query;
    
    let query = 'SELECT * FROM users WHERE 1=1';
    const params = [];
    
    if (grade) {
      query += ' AND grade = ?';
      params.push(grade);
    }
    
    if (className) {
      query += ' AND class = ?';
      params.push(className);
    }
    
    const pageNum = parseInt(page) || 1;
    let limitNum = limit === '0' ? 0 : parseInt(limit) || 50;
    
    // 如果limit为0或负数，返回所有用户
    if (limitNum <= 0) {
      query += ' ORDER BY CAST(student_id AS UNSIGNED)';
    } else {
      const offset = (pageNum - 1) * limitNum;
      query += ` ORDER BY CAST(student_id AS UNSIGNED) LIMIT ${limitNum} OFFSET ${offset}`;
    }
    
    const users = await db.all(query, params);
    
    // 如果需要统计数据
    if (withStats === 'true') {
      for (const user of users) {
        try {
          const stats = await db.get('SELECT total_sessions, avg_accuracy FROM leaderboard WHERE user_id = ?', [user.id]);
          if (stats) {
            user.total_sessions = stats.total_sessions || 0;
            user.avg_accuracy = stats.avg_accuracy || 0;
          } else {
            user.total_sessions = 0;
            user.avg_accuracy = 0;
          }
        } catch (error) {
          console.error(`加载用户 ${user.id} 统计数据失败:`, error);
          user.total_sessions = 0;
          user.avg_accuracy = 0;
        }
      }
    }
    
    res.json(users);
  } catch (error) {
    console.error('获取用户失败:', error);
    res.status(500).json({ error: '获取用户失败' });
  }
});
```

```javascript
export const apiCache = {
  cache: new Map(),
  
  set(key, value) {
    this.cache.set(key, value);
  },
  
  get(key) {
    return this.cache.get(key);
  },
  
  delete(key) {
    this.cache.delete(key);
  },
  
  clear() {
    this.cache.clear();
  }
};
```

#### 4.2.2 优化loadData方法
```javascript
// stores/questionStore.js
async loadData() {
  try {
    this.isLoading = true;
    this.error = null;
    
    // 检查缓存
    const cachedData = apiCache.get('allData');
    if (cachedData && (Date.now() - cachedData.timestamp < 5 * 60 * 1000)) { // 5分钟缓存
      this.subjects = cachedData.data.subjects;
      this.questions = cachedData.data.questions;
      this.grades = cachedData.data.grades;
      this.classes = cachedData.data.classes;
      this.userStats = cachedData.data.userStats;
      this.recentRecords = cachedData.data.recentRecords;
      return;
    }
    
    // 使用综合API端点
    const response = await fetch(`${getApiBaseUrl()}/data/init-data`);
    if (response.ok) {
      const data = await response.json();
      
      // 缓存数据
      apiCache.set('allData', {
        data,
        timestamp: Date.now()
      });
      
      this.subjects = data.subjects;
      this.questions = data.questions;
      this.grades = data.grades;
      this.classes = data.classes;
      this.userStats = data.userStats;
      this.recentRecords = data.recentRecords;
    }
  } catch (error) {
    this.error = error.message;
    console.error('加载数据失败:', error);
  } finally {
    this.isLoading = false;
  }
}
```

#### 4.2.3 优化loadAllUsers方法
```javascript
// AdminView.vue
const loadAllUsers = async () => {
  try {
    // 检查缓存
    const cachedUsers = apiCache.get('allUsers');
    if (cachedUsers && (Date.now() - cachedUsers.timestamp < 2 * 60 * 1000)) { // 2分钟缓存
      allUsers.value = cachedUsers.data;
      return;
    }
    
    const response = await fetch(`${getApiBaseUrl()}/users?limit=0&withStats=true`);
    if (response.ok) {
      const users = await response.json();
      
      // 按学号数字排序
      users.sort((a, b) => {
        const studentIdA = parseInt(a.student_id) || 0;
        const studentIdB = parseInt(b.student_id) || 0;
        return studentIdA - studentIdB;
      });
      
      // 缓存数据
      apiCache.set('allUsers', {
        data: users,
        timestamp: Date.now()
      });
      
      allUsers.value = users;
    }
  } catch (error) {
    console.error('加载所有用户失败:', error);
  }
};
```

## 6. 风险评估

1. **API兼容性**：
   - 风险：修改API可能影响其他功能
   - 缓解：保持向后兼容，添加新参数而非修改现有参数

2. **缓存一致性**：
   - 风险：缓存可能导致数据不一致
   - 缓解：设置合理的缓存过期时间，提供手动刷新机制

3. **性能瓶颈**：
   - 风险：后端综合API可能成为性能瓶颈
   - 缓解：优化数据库查询，考虑使用缓存服务器
