# 2026-04-08 工作日志

## 修复：admin-permissions.js response 调用方式错误

**问题**：`utils/response.js` 中 `success(res, data, msg)` 和 `error(res, msg, status)` 第一个参数是 `res`，
但 `routes/admin-permissions.js` 全文把 response 当纯数据生成器用，写成 `res.json(response.success(data, msg))`，导致 `res.json is not a function`。

**修复**：将全文所有调用改为正确形式 `response.success(res, data, msg)` / `response.error(res, msg, code)`，
同时将 catch 参数从 `error` 改为 `err`（避免与全局 `response.error` 遮蔽混淆）。

涉及路由：GET/POST/PUT/DELETE /roles、/users、/permissions、/my-permissions，共 ~25 处调用。
