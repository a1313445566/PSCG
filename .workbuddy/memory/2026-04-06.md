## 变量缺失问题修复（v1.1）

- **问题**：实施过程中发现大量硬编码无对应变量定义（`#667eea`, `#764ba2`, `#409eff`, Kawaii SVG 颜色等）。
- **根因**：原有变量体系只含基础语义变量，缺少场景化变量（渐变、El-蓝系列、背景灰系列、SVG 图形色等）。
- **方案**：
  1. 在 `_variables.scss` 中补充 45+ 高频场景化变量：
     - 渐变色系：`$header-gradient`, `$gradient-purple`, `$gradient-pink`
     - Element Plus 扩展蓝：`$el-blue-base` ~ `$el-blue-light-3`
     - 背景灰色系：`$bg-gray-1` ~ `$bg-gray-6`
     - 边框灰色系：`$border-gray-light`, `$border-gray-lighter`
     - 表单背景色：`$form-bg-primary`, `$form-bg-secondary`, `$form-bg-error`
     - SVG 图形色：`$kawaii-*`, `$icon-*`, `$avatar-*` 系列
  2. 同步 CSS 变量（`--xxx`）供内联样式使用
  3. 后台端专用变量（`$admin-*`）在后台端 Phase 1 中补充
- **文档更新**：两个分文档均已更新 Phase 1 章节，增加变量补齐策略和按需补充流程。
